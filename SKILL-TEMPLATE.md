# SKILL.md 标准模板

新建 skill 时复制以下模板，替换 `{{...}}` 占位符。

---

```markdown
---
name: {{skill-name}}
description: "{{一句话描述触发条件：当用户要 X 时使用。}}"
---

# {{Skill 标题}}

## 适用场景

- {{主要使用场景 1}}
- {{主要使用场景 2}}
- 需要与 [{{关联 skill}}](../{{关联 skill}}/SKILL.md) 联动时。

## 核心约束

- {{不可违反的规则 1}}
- {{不可违反的规则 2}}
- {{边界条件或前置要求}}

## 实施步骤

### 步骤 1：{{动作}}

{{说明或代码示例}}

### 步骤 2：{{动作}}

{{说明或代码示例}}

## 代码模式

### FAIL: {{错误做法的简短描述}}

```{{lang}}
{{错误代码示例}}
```

→ {{为什么这是错的，会导致什么后果。}}

### PASS: {{正确做法的简短描述}}

```{{lang}}
{{正确代码示例}}
```

→ {{为什么这是对的，解决了什么问题。}}

## 验证清单

- [ ] {{可检查的验证项 1}}
- [ ] {{可检查的验证项 2}}
- [ ] {{可检查的验证项 3}}

## 反模式

- {{常见错误做法 1 及其后果}}
- {{常见错误做法 2 及其后果}}
```

---

## 模板使用说明

| 区块 | 必需 | 说明 |
|------|------|------|
| frontmatter (name, description) | 是 | `description` 是路由信号，必须包含触发关键词 |
| 适用场景 | 是 | 列出 2-4 个具体场景，包含关联 skill 的链接 |
| 核心约束 | 是 | 列出不可违反的规则，不超过 5 条 |
| 实施步骤 | 建议 | 分步骤说明操作流程，含 bash/代码示例 |
| 代码模式 (FAIL/PASS) | 建议 | 用 FAIL/PASS 对比让正误一目了然 |
| 验证清单 | 是 | 可勾选的检查项，用于执行后自检 |
| 反模式 | 是 | 列出 2-4 个常见错误做法 |

## Token 预算

- SKILL.md 主体建议控制在 1500 tokens 以内
- 超出部分放入 `references/` 子目录，通过链接引用
- 代码示例精简到能说明问题即可，不要放完整实现
