# 调研结论笔记反模式集

## 1. 单步落盘

### FAIL

```text
"好的，我整理成笔记并写入 notes/2026-04-28.md 了。"
```

没有先与用户对齐主要问题与核心结论，错对齐成本高，
事后改名/重写都比一开始走双步确认贵。

### PASS

```text
Step 1：问题对齐表 + "这些是不是本次主要问题？"
Step 2：结论对齐表 + "这些是不是要落盘的结论？"
Step 3：两次确认通过 → 落盘并回报路径
```

## 2. 流水账叙事

### FAIL

```markdown
## 笔记
我们先讨论了 A，发现了 B 问题，然后试了 C 方案，又改成 D，
最后觉得还是 E 比较合理……
```

没有结论密度、过程冗余，复盘时无法快速回灌上下文。

### PASS

```markdown
## 问题对比表
| 问题 | 现象/信号 | 核心判断 | 影响 |
|---|---|---|---|
| 列表渲染卡顿 | 滑动 FPS<30 | FlatList 跨类型回收开销 | 阻塞产品发布 |

## 关键结论
1. 选 FlashList 而非自写虚拟滚动：社区维护、API 兼容。
```

## 3. 干甩 path:line

### FAIL

```markdown
- 见 src/auth/interceptor.ts:23-45
- 见 components/FeedList.tsx
```

读者不知道为什么要看、看了能得到什么，等于没引用。

### PASS

```markdown
| 位点 | 它做什么 | 为什么支撑结论 |
|---|---|---|
| `src/auth/interceptor.ts:23-45` | 401 拦截后触发 refresh token 重试 | 证明 refresh 路径已闭合，不需要回退到强制登出 |
```

## 4. 把外部输入误触发本 skill

### FAIL

```text
用户：这是刚才会议的飞书转写，帮我整理一下。
助手：（直接进入双步确认 + 输出 research 笔记骨架）
```

输入不是当前会话讨论，结论会缺乏依据，且与
`meeting-notes-and-actions` 抢路由。

### PASS

```text
"这是会议转写，更适合 meeting-notes-and-actions；如果你之后要把
会议结论再做调研对比，可以再回到 research-note-wrap 收尾。"
```

## 5. 把范围悄悄扩到「今天所有会话」

### FAIL

```text
用户："总结一下调研。"
助手：（自行扩到当天所有 session，把不相关讨论也合并进来）
```

默认范围必须是 current-session；自动扩范围会把噪音引入笔记。

### PASS

```text
助手："默认只总结当前会话。如果你要合并今天关于 X 的多次相关讨论，
请明确说『总结今天关于 X 的会话』，并提供要合并的会话片段。"
```
