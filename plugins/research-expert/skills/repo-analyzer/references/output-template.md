# 仓库分析报告输出模板

```markdown
# 仓库分析报告 — <repo-name>

## 概要
- 项目定位：一句话说清楚这个项目是什么
- 语言/框架：[检测到的技术栈]
- 许可证：[类型]
- 最后更新：[日期]
- 维护状态：活跃 / 维护模式 / 停滞

## 架构概览
- 目录结构与模块划分
- 入口文件与核心流程
- 设计模式与架构风格

## 功能清单
- 项目声称的功能
- 代码实际支持的功能
- 差异说明

## 质量评估
| 维度 | 评分 (1-5) | 证据 |
|---|---|---|
| 代码可读性 | ... | ... |
| 测试覆盖 | ... | ... |
| 文档完整度 | ... | ... |
| 维护活跃度 | ... | ... |
| 依赖健康度 | ... | ... |

## 亮点与风险
- 值得借鉴的设计
- 潜在风险或不足

## 采用建议
- 适合场景
- 不适合场景
- 集成注意事项
```
