# 仓库分析持久化模板

当用户指定 `--persist <dir>` 时，在 `<dir>/<repo-name>-report/` 生成以下 SKILL.md：

```yaml
---
name: <repo-name>-report
description: <repo-name> 仓库分析报告 — 涵盖架构概览、技术栈、质量评估和采用建议。分析基于 <ref> 版本，生成于 <date>。
---
```

## 内容结构

```markdown
# <repo-name> 分析报告

> 生成于 YYYY-MM-DD，基于 <ref>，来源 <url-or-path>

## 项目定位
一段话描述项目是什么、解决什么问题。

## 技术栈
- 语言/框架、构建工具、主要依赖

## 架构概览
- 目录结构与模块划分
- 入口文件与核心流程
- 设计模式

## 质量快照
| 维度 | 评分 | 关键证据 |
|------|------|---------|
| 代码可读性 | X/5 | ... |
| 测试覆盖 | X/5 | ... |
| 文档完整度 | X/5 | ... |
| 维护活跃度 | X/5 | ... |
| 依赖健康度 | X/5 | ... |

## 亮点
- 值得借鉴的设计决策

## 风险
- 潜在问题和注意事项

## 采用建议
- 适合/不适合的场景
- 集成注意事项
```

## 注意事项

- SKILL.md 应自包含，不依赖外部文件
- 所有评价必须有代码/git 证据支撑
- 标明分析时间和版本，因为内容会随时间过期
