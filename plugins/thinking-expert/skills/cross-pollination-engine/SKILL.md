# 跨界迁移引擎

## 适用场景

- 用户说“别的行业会怎么做”“能借鉴谁”“跳出当前行业想一想”。
- 现有方案卡在惯性思维里，需要借远场样本打破局限。
- 适合产品、服务、运营、教育、体验设计等需要新灵感的场景。
- 如果核心任务本身还没剥离清楚，先用 [first-principles-decomposer](../first-principles-decomposer/SKILL.md)。
- 如果想把跨界灵感继续发散到研究问题，可接 [scientific-brainstorming](../scientific-brainstorming/SKILL.md)。
- 需要案例时，参考 [跨界示例](references/examples.md)。

## 核心约束

- 先把问题改写成“核心任务”，去掉行业行话和既有解决方案。
- 每次优先找 2-4 个相距较远的行业，而不是同业竞品互抄。
- 提取的是底层机制、激励结构和体验原则，不是表面 UI 或营销语。
- 每个借鉴都要翻译成“在我们这里最小能怎么试”，而不是停在灵感层。
- 迁移时必须保留本地约束：成本、监管、团队能力、用户习惯。
- 最后收敛成一个可验证的小实验，而不是十个脑洞。

## 代码模式

```text
问题：
核心任务：

借鉴源 A（行业）：
- 他们怎么解决：
- 背后的原理：
- 迁移到当前场景：

借鉴源 B（行业）：
- 他们怎么解决：
- 背后的原理：
- 迁移到当前场景：

合成方案：
- 组合后的最小方案

第一步试验：
- 本周能验证的版本
```

## 检查清单

- 是否已经把问题改写成核心任务。
- 借鉴源是否足够“远”，而不是同质化对标。
- 是否提取了原理，而不是表面特征。
- 是否说明了迁移后的约束和修改点。
- 是否收敛成一个最小实验。
- 是否明确了验证成功的信号。

## 反模式

### FAIL: 同业互抄

```
问题：付费转化低
借鉴：竞品 A 用免费试用 → 我们也加免费试用
借鉴：竞品 B 用阶梯定价 → 我们也加阶梯定价
→ 全行业越来越像，没有差异
```

### PASS: 跨界远场

```
问题核心任务：降低用户首次承诺成本
借鉴源 A（健身房）：first session 免费但要预约 → 制造 "我已经投入" 感
借鉴源 B（航空业）：里程积分 → 让"未来折扣"变成留存锚
合成：注册即给 30 天积分（30 天后失效），首次完成任务额外 +50%
最小实验：本周给 100 个新用户测一周
```

### FAIL: 借表面不借机制

```
"星巴克的杯子很可爱，我们也做杯子周边"
→ 没看到背后的"第三空间"机制
→ 我们是 SaaS，做杯子完全无意义
```

### PASS: 提取原理

```
"星巴克 = 第三空间（家与公司之外的归属）"
→ 我们的"第三空间"对应什么？
→ 用户在两个核心工作流之外，需要一个轻量探索区
→ 落地：加一个"工作坊"模式，无 KPI 压力
```
