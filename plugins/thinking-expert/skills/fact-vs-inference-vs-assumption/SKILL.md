# 事实 / 推断 / 假设三态标注

## 适用场景

- 写代码审查、安全审计、性能诊断、事故复盘、研究报告、咨询分析、战略备忘录。
- 多源信息综合后，需要让读者一眼看出哪些是观测、哪些是推理、哪些是猜想。
- 与 [first-principles-decomposer](../first-principles-decomposer/SKILL.md) 的差异：后者是"拆假设回到事实层"的解构方法；本 skill 是"产出阶段的标注纪律"。
- 与 [finding-evidence-binding](../finding-evidence-binding/SKILL.md) 配合：后者绑证据，本 skill 标三态；常一起使用。
- 与 [citation-validator](../../../research-expert/skills/citation-validator/SKILL.md) 的差异：后者管引用源质量，本 skill 管自己写的每个判断属于哪一态。

## 核心约束

- **每段结论显式标注**：用 `[事实]` / `[推断]` / `[假设]` 前缀（或英文 `[FACT]` / `[INFER]` / `[ASSUME]`），不允许沉默。
- **三态定义边界**：
  - **事实**：直接可观测、可被同类工具复现，附定位即可。
  - **推断**：基于事实链得出的合理结论，但未直接验证；必须给推断依据。
  - **假设**：未观测、用于桥接逻辑空缺的命题；必须显式标"待验证"。
- **不允许跨态包装**：把假设写成"我们知道 X"、把推断写成"事实是 X"、把事实写成"我相信 X" 都属于错误标注。
- **升降级触发条件**：
  - 假设 → 事实：找到直接观测，引用证据后改标。
  - 推断 → 假设：发现支撑链断裂，降级。
  - 事实 → 推断：源数据被质疑，降级。
- **关键假设必须可证伪**：每个 `[假设]` 后给「会改变结论的反例」或「最小验证实验」。
- **结论级别匹配最弱证据**：一条由 1 事实 + 2 推断 + 1 假设链出的结论，整体只能标 `[假设]`。

## 代码模式

```text
## 关键判断

[事实] <观测描述> —— 定位：<file:line | metric | log>
[事实] <观测描述> —— 定位：<...>
[推断] <由上述事实推出的结论> —— 推断依据：<事实如何指向结论>
[假设] <用于桥接的命题> —— 待验证：<最小验证实验>

## 综合结论

[推断] <整体判断>，置信度受 <最弱环节> 约束。
```

## 检查清单

- [ ] 每段结论是否显式带 `[事实]` / `[推断]` / `[假设]` 标注？
- [ ] `[事实]` 是否都附了可核验定位？
- [ ] `[推断]` 是否给了推断依据（哪些事实指向它）？
- [ ] `[假设]` 是否给了"待验证"或"会改变结论的反例"？
- [ ] 综合结论的标注级别是否匹配最弱环节？
- [ ] 是否避免了"显然 / 一定 / 毫无疑问"等抹平三态差异的措辞？

## 反模式

| 危险措辞 | 错在哪 | 替代写法 |
|---------|------|---------|
| "我们知道 X" | 把假设包装成事实 | `[假设] X —— 待验证：...` |
| "显然 X" / "毫无疑问 X" | 抹平三态 | 标具体一态并给依据 |
| "一定是 X 导致的" | 推断当事实 | `[推断] X 可能导致 —— 推断依据：...` |
| 多链综合后只给单一结论 | 隐藏最弱环节 | 综合结论标级匹配最弱环节 |
