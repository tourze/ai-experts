# 三态标注：完整示例

## 场景 1：安全审计中的三态混淆

### FAIL: 沉默标注 + 推断当事实

```
登录接口的 SSRF 风险来自参数 url 未做白名单校验。攻击者可以读取云元数据，
进而获取 IAM 凭证。这会导致整个 AWS 账号被接管。
```

问题：
- "可以读取云元数据" 是推断（未实测），标成了事实。
- "整个账号被接管" 是更远的假设，无依据。
- 整段没区分观测、推理与猜想。

### PASS: 三态显式 + 综合结论降级

```
[事实] /api/preview 接收 url 参数后直接调用 Http::get($url)，
       无 scheme / IP 白名单。定位：PreviewController.php:18。

[事实] curl 'http://localhost/api/preview?url=http://169.254.169.254/' 
       返回 EC2 metadata 文本。定位：PoC log #2026-04-25T10:33Z。

[推断] 攻击者可读取 IAM 角色凭证。
       推断依据：EC2 metadata 服务默认暴露 iam/security-credentials 路径，
       未观测到 IMDSv2 限制。

[假设] 攻击者可借此接管整个账号。
       待验证：需要确认拿到的角色权限范围；当前 instance role
       仅有 S3 read 权限的可能性同样存在。

[综合·推断] 该接口存在 SSRF；最坏情况受 IAM 权限范围约束，
            置信度受『假设』层制约。
```

## 场景 2：事故复盘中的混杂判断

### FAIL: 把推断写成结论

```
18:00 P99 飙升的根因是数据库连接池打满。
```

问题：metric 显示连接数高 ≠ 连接池打满直接导致延迟。是推断不是事实。

### PASS: 分层标注

```
[事实] 18:00-18:30 login.p99 从 120ms 升至 600ms。
       定位：Datadog metric login.p99，时间窗 2026-04-25T18:00-18:30。

[事实] 同时段 db.connections.active 从 80 升至 195（上限 200）。
       定位：Datadog metric db.connections.active 同时段。

[推断] 连接池接近饱和导致请求等待获取连接。
       推断依据：连接数曲线与延迟曲线呈正相关；DB 自身慢查询 metric 无尖刺。

[假设] 与 18:00 的批量任务有关。
       待验证：检查批量任务调度记录与该时段 connection 来源 IP。

[综合·推断] 触发因素疑为批量任务挤占连接池；脆弱性在于
            连接池无优先级隔离。修复方向同时覆盖触发因素与脆弱性。
```

## 场景 3：战略分析中的假设伪装事实

### FAIL: 市场判断写成事实

```
我们的目标客户每年愿意为这个功能付 $100。市场规模 1000 万美元。
```

问题：愿付价格、用户数、市场规模均为假设，呈现成事实。

### PASS: 三态拆解

```
[事实] 5 名 beta 用户在访谈中说『愿意付费』，但未实际下单。
       定位：访谈记录 2026-Q1 #1-#5。

[假设] 单用户年付意愿 $100。
       待验证：开放预付页面观察 7 天转化率；当前数据不足以确认。

[假设] 目标客户基数 10 万。
       待验证：参考 SimilarWeb 估计的同类工具流量，但样本未对齐 ICP。

[综合·假设] 市场规模上限 $1000 万为「假设 × 假设」推算，
            置信度低；进入 Series A 故事前需把两个假设至少
            升级到推断（用预付转化数据 + ICP 流量校准）。
```

## 升降级实战

### 假设升事实：拿到直接观测后改标

```
（旧）[假设] 客户偏好按月付费 —— 待验证。
（新）[事实] 53% beta 用户在 onboarding 选择月付。
       定位：mixpanel funnel 2026-04-01..2026-04-25。
```

### 推断降假设：支撑链断裂后改标

```
（旧）[推断] 慢查询是 ORDER BY 缺索引。推断依据：EXPLAIN 显示 filesort。
（新）[假设] 慢查询可能与 ORDER BY 有关 —— 待验证。
       原推断已撤回：进一步查 slow log 发现 80% 慢查询不含 ORDER BY，
       filesort 不是主因。
```

## 与 evidence-quality-framework 的协同

```
## 发现 N：<标题>

[事实 / 推断 / 假设] <一句结论>

- 定位：<file:line | metric | trace_id>     ← evidence-binding
- 证据：<片段>                              ← evidence-binding
- 推断依据 / 待验证：<...>                  ← 三态标注
- 触发条件 / 影响 / 修复方向：<...>          ← evidence-binding
```

两个 skill 各管一面：本 skill 管「这是哪一态」，evidence-binding 管「证据三件套」。
