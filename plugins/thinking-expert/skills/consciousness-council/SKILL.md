# 多视角思辨议会

## 两种模式

| 模式 | 触发信号 | 视角数 | 适用 |
|------|---------|--------|------|
| **角色议会** | "开议会""多视角""换专家看" | 4-6 角色 | 架构/产品/战略等高不确定性决策 |
| **六顶思考帽** | "六顶帽子""思考帽""测试策略多角度" | 6 顶帽 | 测试策略、事故复盘、团队分歧 |

六顶思考帽模式说明见 [references/six-hats-guide.md](./references/six-hats-guide.md)。

## 角色议会模式

- 用户说“换几个专家视角看”“帮我多角度想一遍”“开个议会 / council”。
- 问题存在明显权衡，单一视角容易漏掉关键代价。
- 需要让不同认知角色正面冲突，再把冲突收敛为行动建议。
- 如果重点是未来分支推演，优先配合 [what-if-oracle](../what-if-oracle/SKILL.md)。
- 如果重点是把现有方案问穿，优先配合 [grill-me](../grill-me/SKILL.md)。

## 核心约束

- 每次选 4-6 个角色，重点是制造有效张力，不是凑人数。
- 角色必须有不同的关注点、风险偏好和盲区，不能只是“换个措辞表达同意”。
- 至少保留一个实质性分歧；如果所有角色都在同意，说明角色选择失败。
- 对纯事实问题或低风险琐事，不要强行开完整议会。
- 结尾必须输出：共识、核心张力、共同盲点、建议路径、信心等级。
- 需要行业化配方时，参考 [高级配置示例](references/advanced-configurations.md)。

## 代码模式

```text
候选角色：
- 架构师：先看结构与边界
- 唱反调者：优先挑战默认共识
- 实证派：先问证据和数据
- 伦理派：先看谁受益、谁承担代价
- 未来派：先看长期与二阶影响
- 务实派：先看本周能不能落地
- 历史派：先看先例和模式
- 局外人：先问“为什么大家都默认这点”

角色输出模板：
[角色名]
立场：
推理：
他们看到的关键风险：
他们提出的意外洞见：

综合输出模板：
共识：
核心张力：
共同盲点：
建议路径：
信心等级：
```

## 检查清单

- 问题是否真的值得做多角色议会推演。
- 选出的角色之间是否存在真实冲突。
- 每个角色是否都说清了自己的立场和风险点。
- 综合部分是否提炼出“核心张力”，而不是复读每个人说过的话。
- 输出是否收敛成用户可执行的下一步。

## 反模式

### FAIL: 角色全部点头

```
架构师：建议这么做
未来派：我也建议这么做
务实派：我同意以上观点
→ 4 个角色 4 段附和 = 没有议会
```

### PASS: 制造实质张力

```
务实派：本周必须上线，砍掉 SSO 集成
唱反调者：上线后 30% 的企业客户会流失，谁承担？
伦理派：用户数据收集口径还没法律审核
→ 真实冲突 → 综合时浮现"先上线 + 同步法务审核"折衷
```

### FAIL: 简单事实硬套议会

```
用户："这个变量该叫 user_id 还是 userId？"
AI：召集架构师、未来派、伦理派各发表观点...
→ 用户只想要 30 秒答案
```

### PASS: 议会有阈值

```
召开议会的门槛：
- 决策不可逆 OR
- 影响 > 1 个团队 OR
- 至少 2 种合理方案存在权衡
否则直接给答案
```
