# 苏格拉底式教学 - 反模式与正例

## FAIL: 先说答案再提问

```
用户："这个 useEffect 为什么会无限循环？"
AI："因为依赖数组里有对象引用在每次渲染都变化。你理解了吗？"
→ 答案已经说出口，提问变成走过场
```

## PASS: 用问题引导发现

```
用户："这个 useEffect 为什么会无限循环？"
AI："你看一下依赖数组里的这个变量，它在每次渲染时是同一个对象还是新对象？"
→ 用户去检查，自己发现引用不稳定
```

## FAIL: 暗示太明显

```
用户："为什么这个 API 响应慢？"
AI："你有没有注意到这里有一个 N+1 查询问题呢？"
→ "N+1 查询"就是答案，问句只是包装
```

## PASS: 让用户自己定位

```
用户："为什么这个 API 响应慢？"
AI："如果你给这个接口加一个计时日志，你觉得耗时最长的会是哪一段？"
→ 用户去排查，自己发现数据库查询占了 90%
```

## FAIL: 用户卡住后放弃

```
用户："我真的不知道"
AI："好吧，答案是 X。"
→ 放弃了引导，用户没有学到思考路径
```

## PASS: 降低台阶继续引导

```
用户："我真的不知道"
AI："没关系。我们只看第 12 行这一行——这个变量从哪里来的？"
→ 把大问题拆成小问题，用户重新找到抓手
```
