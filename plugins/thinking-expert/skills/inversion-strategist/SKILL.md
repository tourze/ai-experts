# 逆向思维策略师

## 适用场景

- 用户说“反过来想”“怎么一定会失败”“我们要避免什么坑”。
- 目标清楚，但失败模式、脆弱点和不可承受代价还没被显式写出来。
- 适合上线前、扩张前、关键决策前做“先别死”的思考。
- 如果目标本身还没定义清楚，先用 [first-principles-decomposer](../first-principles-decomposer/SKILL.md) 收敛。
- 如果需要把失败路径展开成未来分支，可接 [what-if-oracle](../what-if-oracle/SKILL.md)。
- 需要更多案例时，参考 [示例集](references/examples.md) 与 [方法框架](references/framework.md)。

## 核心约束

- 先把目标写清楚，再做反转；目标模糊时，失败路径也只会变成废话。
- 逆向问题必须足够尖锐，例如“如何保证 60 天内拿不到第一个客户”，而不是“怎么可能出问题”。
- 至少列 7 个失败路径，并按“致命 / 重伤 / 烦扰”分层。
- 每个失败路径都要配套规避动作，不能只做风险陈列。
- 反清单优先写“绝不能做什么”，再补充正向行动。
- 不要只盯极端黑天鹅，也要覆盖缓慢腐蚀型失败。

## 代码模式

```text
目标：

逆向问题：
“如果我要确保在这个目标上失败，我会怎么做？”

🔴 致命
- 失败路径：
  规避动作：

🟡 重伤
- 失败路径：
  规避动作：

🟢 烦扰
- 失败路径：
  规避动作：

反清单：
□ 绝不……
□ 绝不……

正向动作：
□ 现在先补……
□ 上线前验证……
```

## 检查清单

- 目标是否可观察、可验证。
- 失败路径是否覆盖了单点故障、慢性退化和自我制造的坑。
- 每个失败路径是否都给了规避动作。
- 是否区分了“会致命”和“只是很烦”。
- 是否输出了可执行的反清单。
- 是否指出了最先该补的 1-2 个薄弱点。

## 反模式

- 只列泛泛风险，不给任何规避动作。
- 把所有风险都写成最高级，失去分层意义。
- 只看极端灾难，不看日常腐蚀型失败。
- 反清单写成空话，例如“不要犯错”“注意沟通”。
- 把逆向分析当成恐吓，而不是收敛行动。
