# 劳动合同与 HR 模板

## 适用场景

- 起草或重写 `offer letter`、劳动合同、员工手册、保密协议、远程办公附录等劳动用工文本。
- 统一招聘、入职、转正、调岗、离职文件的字段口径，减少 HR、法务和业务之间的版本漂移。
- 需要评估某个条款偏离标准模板后的风险级别时，联动 [legal-risk-assessment](../legal-risk-assessment/SKILL.md)。
- 文本涉及候选人、员工或离职人员的个人信息处理时，联动 [gdpr-data-handling](../gdpr-data-handling/SKILL.md)。

## 核心约束

- 这是文档与流程框架，不替代执业律师意见；跨法域、群体裁员、股权授予、竞业限制、工会/集体协商事项必须升级到本地律师。
- 模板必须显式区分 `employee` / `contractor`、`at-will` / 固定期限、`exempt` / `non-exempt`、全职 / 兼职 / 远程等劳动关系类型，不能混写。
- 任何薪酬条款都要写清币种、计薪周期、奖金是否酌情、股权授予是否受董事会批准和单独协议约束，避免把“预期待遇”写成“刚性承诺”。
- 保密、知识产权、离职限制、争议解决、通知方式、试用期、休假、法定福利等条款必须按法域本地化；不要把美国 `at-will`、英国 `garden leave`、欧盟强制通知期放进同一份模板里。
- 模板输出前要把所有占位符与假设显式列出，交付对象必须知道哪些字段已经确定、哪些仍待 HR / 财务 / 法务补数。

## 代码模式

```markdown
# 录用通知核心骨架

- 候选人：{{candidate_name}}
- 职位：{{job_title}}
- 汇报线：{{manager_title}}
- 用工类型：{{employment_type}}
- 工作地点：{{location_mode}}
- 入职日期：{{start_date}}
- 薪酬：{{currency}} {{base_salary}} / {{pay_cycle}}
- 奖金：{{bonus_rule}}
- 股权：{{equity_rule}}
- 条件生效项：
  - 背调 / 资质核验
  - 身份与工作资格校验
  - 保密、知识产权、信息安全文件签署
- 法域提醒：{{jurisdiction_notice}}
- 接受期限：{{offer_expiry}}
```

```markdown
# 劳动合同条款矩阵

| 模块 | 必填字段 | 本地化提醒 |
| --- | --- | --- |
| 当事人信息 | 公司法定主体、员工姓名、地址、签署日期 | 避免用品牌名替代签约主体 |
| 岗位与职责 | 职位、汇报对象、工作地点、调岗规则 | 远程/混合办公要写明地点与设备责任 |
| 薪酬与福利 | 基薪、计薪周期、奖金、福利、报销 | 奖金是否酌情、最低工资与强制福利要本地化 |
| 保密与知识产权 | 保密范围、归还义务、职务成果归属 | 发明归属、开源贡献、既有成果保留须单列 |
| 离职限制 | 通知期、竞业、招揽限制、交接 | 先确认法域是否允许、范围是否过宽 |
| 争议解决 | 适用法律、仲裁/诉讼地、通知方式 | 不同国家对仲裁与劳动争议有强制规则 |
```

```markdown
# 员工手册政策块最小骨架

## 平等就业与反骚扰
- 受保护特征
- 举报渠道
- 反报复承诺

## 工时与考勤
- 标准工时
- 加班审批
- 迟到缺勤上报方式

## 休假与福利
- 年假 / 病假 / 法定假
- 结转、清零、折现规则

## 信息安全与设备使用
- 账号、设备、监控告知
- 数据访问与保密义务
```

## 检查清单

- 公司法定主体、签署主体、签署权限、币种和日期格式是否全部一致。
- 是否明确劳动关系类型、试用期、工时分类、通知期和离职交接要求，而不是把雇员与承包商语言混在一起。
- 薪酬、奖金、佣金、股权、报销、福利是否写清触发条件、结算周期、是否酌情以及与单独政策的关系。
- 保密、知识产权、开源贡献、既有成果保留、设备归还、数据删除、客户/员工招揽限制是否覆盖到该岗位风险。
- 员工手册中引用的政策、举报渠道、假期制度、信息安全规则是否真实存在，且与当前组织流程一致。
- 若文档涉及候选人或员工个人信息采集、保留或跨境传输，是否已经同步到隐私告知与数据保留安排。

## 反模式

- 把一份“市场通用模板”直接复制到新法域，只改公司名，不改强制劳动法条款。
- 在录用通知中写死尚未获批的奖金、股权或签字费，导致业务承诺超出公司审批。
- 用“可随时解雇”“自动放弃加班费”等明显不可执行语言来追求所谓强保护。
- 把竞业限制写得过宽、期限过长、范围过大，却没有任何法域可执行性分析。
- 手册里保留已经废弃的举报邮箱、考勤系统、远程办公政策或福利规则，导致文档与实际流程脱节。
