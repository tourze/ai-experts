# 5 条核心约束（详细版）

## 1. 正文为主，彩色 callout 为辅

页面 80% 以上是 `<p>` 段落和 `<h2>`/`<h3>` 标题。`.info`/`.warn`/`.ok`/`.def`/`.formula` 等彩色框整页加起来不超过 3-4 个。定义、公式、性质用 `<p>` + `<b>` 加粗关键词写在正文，**不要**塞进 `.def`/`.formula` 框——彩色框堆砌会让页面变成方块拼图，破坏教学叙事。

参考 `references/heap_overview.html`（444 行，7 章节）：0 个 `.def`，0 个 `.formula`，全页只有 5 个 `.info` + 2 个 `.ok` + 4 个 `.warn`。

## 2. SVG 节点不能重叠，viewBox 必须留够空间

- 圆形节点半径 r=22~24，**相邻节点圆心距 ≥ 56px**（2r + 间隙）。
- 父子层 y 间距 ≥ 60px，给连线和标注留空间。
- **viewBox 高度 = max(节点 y + r) + 40**（标注文字约 20px + 安全边距 20px）。
- 写完坐标后逐个验证：每个 (cx, cy) 周围 24px 半径内不能有其他圆心。

宁可加大 viewBox 也不要压缩节点间距。

## 3. 涉及代码时，动画必须与代码联动

- 动画的每个 step 必须包含 `line` 字段，标注当前执行的代码行号（从 0 开始，-1 表示无）。
- 代码用 `.code-panel` 容器，每行 `<div class="cl">`，当前执行行加 `.cl.on`。
- 代码面板放动画上方或左侧，形成"代码 + 可视化"双面板（HTML 骨架结构 1）。
- `render` 函数末尾调用 `hlLines('codeId', s.line)`。
- 说明文字（`.cap`）同时解释**代码在做什么**和**数据结构如何变化**。
- 代码超过 12 行时，只展示当前动画相关片段，其余用 `// ...` 省略。

不能出现"动画区域和代码区域互不关联"的割裂情况。

## 4. 每个可视化概念配 SVG 图示

- 数据结构（树、图、栈、队列）→ 静态结构图。
- 操作过程（插入、删除、排序步骤）→ 状态图或交互动画。
- 概念对比 → 并排 SVG。
- 节奏：**讲完概念 → 紧跟 SVG → 再接文字说明**，不能"只有文字没有图"。
- 静态 SVG 用于"单个状态/概念"，交互式动画只用于"多步骤过程"。
- 整页静态 SVG 数量 ≥ 交互式动画数量。

## 5. 有来源时跟着来源教学脉络走

- PDF/教材：**顺着原文页面顺序**梳理章节，原文先讲什么本页就先写什么，原文用什么例子本页就用什么例子。不要从结论倒推，不要"提炼要点后重新组织"。
- 无来源（纯主题）：按自然教学顺序——这是什么 → 怎么工作（配图）→ 实际例子（配动画）→ 总结（核心要点、易错点）。
