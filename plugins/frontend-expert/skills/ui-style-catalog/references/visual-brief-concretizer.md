# 视觉需求具体化

## 适用场景

- 甲方、产品、市场或团队只给出抽象审美词，需要转成可执行视觉 brief。
- 设计争议来自“好看/大气/高级/有感觉”，而不是明确任务流问题。
- 需要为网页、PPT、海报、品牌物料或产品界面定义视觉方向和避坑清单。
- 如果已经进入 token 落地，转到 [visual-design-foundations](../visual-design-foundations/SKILL.md)；如果是行业视觉方向，联动 `industry-design-presets`；如果是已实现 UI 审查，联动 `frontend-design-review`。

## 核心约束

- 先翻译意图，再谈样式。抽象词不是需求本身，必须落到品牌想传达的形象、受众和使用场景。
- 每个方向必须同时给出正向信号和反模式；只说“多留白、用高级色”不算具体化。
- 用参考证据降低争议：优先找同类品牌、行业页面、历史物料或用户已认可样例，不凭个人审美拍板。
- 不把所有“高端元素”堆到画面里；元素越多，越容易削弱可信度和品牌一致性。
- 如果用户没有授权自由发挥，最多问 3 个关键澄清问题；能从上下文推断的先写成假设。

## 代码模式

先把抽象词归类成意图，再映射到视觉约束：

```json
{
  "abstract_word": "大气",
  "intent": "专业权威可信赖",
  "audience": "金融机构采购负责人",
  "visual_signals": ["真实办公场景", "冷色主调", "稳重无衬线", "充足留白"],
  "avoid": ["抽象宇宙背景", "金色大字堆叠", "无关地标拼贴"]
}
```

常用拆解表：

| 抽象词 | 先确认的真实意图 | 可执行视觉信号 | 常见反模式 |
|--------|------------------|----------------|------------|
| 大气 | 未来感 / 权威感 / 历史感 / 高端感是哪一种 | 大尺度场景、统一色调、强层级、留白 | 地球、高楼、火箭、蓝天白云全堆上 |
| 高级 | 产品细节经得起放大，还是品牌要显得克制 | 少色、细节特写、精确间距、低噪声材质 | 金色大面积铺满、黑金模板化 |
| 专业 | 行业可信、流程严谨、数据可靠 | 真实场景、冷色/中性色、清晰图表、稳定网格 | 摆拍人物抠图、装饰图替代证据 |
| 科技感 | 前沿能力、工程可信、还是 AI 新鲜感 | 结构线、产品界面、数据流 | 紫蓝渐变、霓虹光晕、无意义粒子 |
| 有灵魂 | 情绪记忆点、品牌主张、还是故事线 | 单一主隐喻、真实素材、文案与视觉同向 | 把“灵魂”当装饰，让画面更杂 |

输出 brief 时使用固定结构：

```md
## 视觉 brief

目标词：<用户原词>
我将它解释为：<品牌意图>
适用场景：<页面 / 物料 / 演示 / 受众>

必须出现：<3-5 个视觉信号>

必须避免：<2-4 个反模式>

可验证标准：
- 3 秒内能说出品牌想传达的形象
- 主色不超过 3 类角色
- 画面元素都能解释“为什么属于这个品牌”
```

## 检查清单

- [ ] 已把抽象词翻译成 1 个主品牌意图，而不是保留原词打转。
- [ ] 已确认受众、载体、使用场景和品牌约束。
- [ ] 视觉信号覆盖 imagery、palette、typography、layout 中至少 3 类。
- [ ] 已列出会让画面显得廉价、山寨或跑偏的反模式。
- [ ] 已说明哪些判断来自事实参考，哪些是当前假设。
- [ ] 下一步能交给 `visual-design-foundations`、`ppt-visual`、`concept-to-image` 或前端实现，而不需要二次翻译。

## 反模式

### FAIL: 把抽象词当作设计方案

```
需求：做得大气一点
输出：大气、简洁、高级、有冲击力
→ 每个词都仍然不可执行，设计师和实现者只能继续猜
```

### PASS: 抽象词落到品牌意图和约束

```
需求：做得大气一点
解释：给董事会看的专业权威感
约束：真实业务场景 + 冷色主调 + 留白 + 重点数字优先
反模式：不用宇宙、高楼、金色大字和无关地标拼贴
```
