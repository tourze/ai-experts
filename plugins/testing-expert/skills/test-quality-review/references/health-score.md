# 测试 Health Score 与输出格式

## 输出格式

```markdown
## 测试质量审查

**Test Health Score: XX/100**

共发现 N 个问题：🔴 关键 X 个，🟡 重要 Y 个，🟢 建议 Z 个。

### 测试套件概览

| 指标 | 值 |
|------|------|
| 测试文件数 | N |
| 涉及风险类型 | T1, T3, T4 |
| 最高风险 | TX — 一句话概括 |

### 🔴 关键

**[TX] 问题标题** — `path/to/test:42`
- **Symptom:** 在测试代码中观察到了什么
- **Source:** 为什么这是测试衰退风险
- **Consequence:** 不修复会导致什么后果
- **Remedy:** 怎么修（方向，不给整段代码）

### 🟡 重要

（同上四要素格式）

### 🟢 建议

（同上四要素格式）
```

## Health Score 计算

基础分 100，按 finding 扣分：

| 严重度 | 每条扣分 | 上限 |
|--------|---------|------|
| 🔴 关键 | -15 | 无上限 |
| 🟡 重要 | -7 | 无上限 |
| 🟢 建议 | -3 | 最多扣 15 分 |

最低分 0。

| 分数 | 解读 |
|------|------|
| 90-100 | 测试套件健康，可信赖 |
| 75-89 | 良好，处理重要问题后更可靠 |
| 60-74 | 需改进，存在信任缺口 |
| 40-59 | 堪忧，测试套件不能有效防护回归 |
| < 40 | 测试套件需重建 |

## 打分纪律
- 无 finding 不硬凑——100 分合法
- 不因测试风格偏好扣分——只扣衰退风险
- 同一根因的多个 symptom 合并为一条 finding
