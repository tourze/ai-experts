# 六顶思考帽测试法

## 适用场景

- 新功能测试策略设计卡住，需要强制换视角。
- 线上事故复盘后，希望同时看到事实、风险和改进机会。
- 团队对测试投入、风险判断或优先级存在分歧。
- 需要为 [testing-strategy](../testing-strategy/SKILL.md) 提供输入。

## 核心约束

- 六顶帽子必须分时段使用，不能混着写。
- 红帽只记录直觉，不要求论证；白帽只记录事实，不掺观点。
- 蓝帽负责收敛行动项，不能停留在“大家都说得对”。
- 若开团队会，必须限时，防止讨论无限发散。
- 可直接复用这些模板：
  - [solo-session-template.md](./resources/templates/solo-session-template.md)
  - [team-session-template.md](./resources/templates/team-session-template.md)
  - [api-testing-example.md](./resources/examples/api-testing-example.md)

## 代码模式

### 模式 1：按帽子顺序输出

```markdown
## 🤍 白帽：事实
- 47 个接口
- 自动化覆盖率 32%
- 最近两周线上缺陷 5 个

## ❤️ 红帽：直觉
- 我对权限边界没有信心
- 我担心回归周期太长

## 🖤 黑帽：风险
- 未覆盖鉴权失败路径
- 限流策略没有压测

## 💛 黄帽：优势
- API 文档完整
- CI 已接入冒烟测试

## 💚 绿帽：新思路
- 引入契约测试
- 用生产流量回放做回归

## 🔵 蓝帽：行动
- 本周先补鉴权与限流
- 下周引入契约测试 PoC
```

### 模式 2：团队会议节奏

```markdown
- 白帽 10 分钟：事实盘点
- 红帽 5 分钟：一人一句直觉
- 黑帽 12 分钟：风险清单
- 黄帽 8 分钟：可利用优势
- 绿帽 15 分钟：发散方案
- 蓝帽 10 分钟：收敛 owner 和截止时间
```

## 检查清单

- [ ] 六顶帽子内容没有互相污染
- [ ] 白帽事实可验证
- [ ] 红帽直觉没有被要求“证明”
- [ ] 黑帽风险已连接到具体业务或技术后果
- [ ] 绿帽至少给出 3 个可执行新想法
- [ ] 蓝帽输出了 owner、优先级和下一步

## 反模式

### FAIL: 帽子互相污染

```md
## 白帽：事实
- API 47 个
- 我觉得测试覆盖率不够（← 观点，应该在红帽）
- 限流没做（← 风险，应该在黑帽）
```

### PASS: 严格分离

```md
## 白帽：纯事实
- API 47 个 / 自动化覆盖 32% / 上周 5 个线上 bug

## 红帽：直觉（不要求论证）
- 担心权限边界 / 担心回归周期

## 黑帽：风险（基于事实推断）
- 鉴权失败路径未覆盖 / 限流未压测
```

### FAIL: 只有黑帽

```
讨论 1 小时全在挑问题
→ 团队士气崩 / 没有可行动方案
```

### PASS: 全 6 顶 + 收敛

```
黑帽（风险）+ 黄帽（已有优势）+ 绿帽（新方案）
→ 蓝帽收敛：本周 + 谁 + 截止
不能停在 “大家说得都对”
```
