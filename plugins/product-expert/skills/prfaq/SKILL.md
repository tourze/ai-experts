# PRFAQ（新闻稿 + FAQ）

## 适用场景
- 新产品/功能立项前，用"从终点倒推"的方式验证用户价值主张。
- 需要在团队或管理层之间对齐"我们到底要做什么、为谁做"。
- 与 [create-prd](../create-prd/SKILL.md) 配合：PRFAQ 先定"为什么做"，PRD 再定"怎么做"。

## 核心约束
- 新闻稿必须从**用户视角**写，不是从公司视角；主语是用户，不是"我们"。
- 新闻稿限 1 页（≤500 字）；FAQ 分内部和外部两组，各 3-5 个问题。
- 如果写不出让人想点击的标题，说明价值主张还不清楚，先退回去想清楚。
- FAQ 必须直面最难的问题（"为什么现在做""为什么是我们""失败了怎么办"），不允许只放软球。

## 代码模式

```markdown
# [产品/功能名称] — 新闻稿

**标题**：一句话说清楚用户获得了什么（不超过 15 字）
**副标题**：补充目标用户和核心价值

**日期 | 地点** — 开头段落：用户是谁、他们的痛点、我们提供了什么。

**问题段**：用户今天面临什么困境？（2-3 句）

**方案段**：产品如何解决这个问题？（2-3 句，用户视角）

**用户引言**："[虚构用户名]，[角色]，说：……"（展示用户获得的价值）

**如何开始**：用户从哪里获得/使用这个产品。

**负责人引言**："[负责人名]说：……"（说明为什么现在做这件事）

---

## 外部 FAQ（面向用户）
1. 这和现有方案有什么不同？
2. 我需要付费吗？定价是什么？
3. 我的数据安全吗？

## 内部 FAQ（面向团队/管理层）
1. 为什么现在做这个？机会窗口是什么？
2. 需要多少资源？ROI 预期是什么？
3. 最大的技术风险是什么？
4. 如果失败，退出策略是什么？
5. 竞品为什么没做 / 做了但没成功？
```

## 检查清单
- [ ] 新闻稿从用户视角撰写，主语不是"我们的产品"。
- [ ] 标题让目标用户有点击欲望。
- [ ] 内部 FAQ 包含至少一个"为什么不做"类的硬问题。
- [ ] 团队读完后能回答"做什么、为谁做、为什么现在做"。

## 反模式

### FAIL: 公司视角的广告

```
标题："XX 公司发布业界领先的 AI 平台"
→ 用户读完不知道自己能得到什么
→ 全篇都是"我们的技术""我们的团队"
```

### PASS: 用户视角的价值

```
标题："跨境卖家现在可以 10 分钟完成多币种对账"
→ 用户立刻知道：这是给我的，解决对账慢的问题
```

### FAIL: FAQ 全是软球

```
Q: 产品好用吗？ A: 非常好用。
Q: 安全吗？ A: 非常安全。
→ 没有回答任何真正的疑虑
```

### PASS: FAQ 直面硬问题

```
Q: 为什么竞品 A 做了类似功能但没起来？
A: A 只支持 3 种货币，且需要手动导入；我们接了 15 种货币的自动对接。
   A 失败的核心原因是数据源覆盖不够，这是我们 6 个月的数据集成投入解决的。
```
