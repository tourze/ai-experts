# SWOT 分析

## 适用场景
- 产品/公司战略评估、年度复盘、竞品对比或进入新市场前的结构化判断。
- 需要看行业结构或具体对手时，可配合 [porters-five-forces](../porters-five-forces/SKILL.md) 与 [competitive-teardown](../competitive-intelligence/SKILL.md)。

## 核心约束
- Strength/Weakness 写内部能力，Opportunity/Threat 写外部环境，别混淆。
- SWOT 不是四格词云，每个点都要连接到战略动作。
- 先列事实和证据，再做判断，避免把偏好写成优势。

## 代码模式
```markdown
| 类别 | 事实/证据 | 战略含义 | 建议动作 |
| --- | --- | --- | --- |
```

## 检查清单
- [ ] 四个象限都有证据支撑而非口号。
- [ ] 机会与威胁区分了行业变化和竞争变化。
- [ ] 已把关键组合转成具体战略动作。
- [ ] 分析结论能衔接资源投入与优先级决策。

## 反模式

### FAIL: "团队努力" 当优势

```
S: 团队很努力 / 文化很好 / 老板有远见
→ 这些是任何公司都会写的话，无差异
```

### PASS: 可证据化的能力

```
S：核心算法专利（已授权）+ 5 年历史用户数据（不可复制）
W：销售团队 < 5 人 / 海外市场无经验
O：行业法规变化创造新需求窗口（6 个月）
T：上游供应商集中度提高 + 提价 30%
→ 每条都能列证据
```

### FAIL: 四象限 + 句号

```
[完整 SWOT 矩阵]
→ 提交董事会 → 没人知道下一步做什么
```

### PASS: SO/WO/ST/WT 战略动作

```
SO（用优势抓机会）：算法 × 法规窗口 → 优先攻金融客户
WO（补劣势抓机会）：海外经验弱 + 海外机会 → 招 1 个 BD
ST（用优势防威胁）：数据壁垒 × 上游涨价 → 自建数据管道
WT（避险）：销售小 × 上游集中 → 不进入低毛利赛道
→ 每条对应季度行动
```
