# Greiner 组织成长模型

## 适用场景
- 判断企业当前处于哪个发展阶段，预判即将到来的危机。
- 避免"用上一阶段的成功方法解决当前阶段的问题"。
- 与 [team-composition-analysis](../team-composition-analysis/SKILL.md) 配合：Greiner 判断阶段，团队分析定人员结构。

## 核心约束
- 五阶段，每阶段以特定方式成长、以特定危机结束：
  1. 创业期（创始人主导）-> 领导力危机（需要职业经理人）
  2. 引导期（规范管理）-> 自主危机（一线缺乏授权）
  3. 授权期（分权运营）-> 控制危机（总部失去把控）
  4. 协调期（职能管控）-> 官僚危机（流程繁琐效率低）
  5. 合作期（团队协作）-> 未知危机
- 危机不是坏事——是进入下一阶段的信号。关键是识别正在经历哪种危机。
- 不能跳过阶段，但可以通过预见性管理缩短过渡期。
- **最大陷阱**：用上一阶段的成功方法解决当前危机（如创业期用大公司流程）。

## 不适用场景
- 非成长型组织（政府机构、稳态企业）：Greiner 假设企业在持续成长，稳态组织无阶段跃迁。
- 急需具体操作方案：Greiner 定诊断方向，具体组织设计用 [org-canvas](../org-canvas/SKILL.md)，人才策略用 [talent-management](../talent-management/SKILL.md)。

## 检查清单
- [ ] 准确识别了当前阶段和正在经历的危机。
- [ ] 没有用上一阶段的方法解决当前危机。
- [ ] 应对方案与危机类型匹配。

## 反模式

### FAIL: 创业期用大公司方法

```
10 人团队 -> 引入 OKR + 月度绩效考核 + 部门周报
-> 阶段 1 用阶段 4 的工具 -> 过早官僚化
-> 应该：创始人直接带队，非正式沟通，快速试错
```

### PASS: 阶段匹配管理

```
当前：阶段 2 -> 3（引导期末，自主危机初现）
信号：一线抱怨"什么都要审批""管理层不信任我们"
应对：
- 选 2 个成熟业务线试点分权
- 设定底线（预算/品牌红线），其余授权
- 3 个月后评估：效率提升则扩大，失控则收回
```
