# 设计增长飞轮

## 适用场景
- 产品已有一定留存基础，想把拉新、激活、留存和分享串成闭环。
- 需要结合 [references/guest-insights.md](references/guest-insights.md) 提炼常见增长模式或限制条件。

## 核心约束
- 先确认核心用户价值和自然传播时刻，再设计分享或邀请机制。
- 飞轮必须说明输入、产出、反馈路径与放大条件，不能只是一张漏斗图。
- 如果留存、LTV 或用户价值不足，优先修产品基础，而不是强推裂变。

## 代码模式
```markdown
输入动作 -> 用户获得价值 -> 产生可传播资产 -> 新用户进入 -> 留存放大
```

## 检查清单
- [ ] 已识别主循环、触发点和关键阻尼项。
- [ ] 指标覆盖转化、留存、分享率和回流效率。
- [ ] 已明确需要的产品、运营或激励支撑。
- [ ] 飞轮假设可被实验验证，而不是纯口号。

## 反模式

### FAIL: 一次性活动当飞轮

```
"邀请 3 人送 30 天 VIP，活动结束"
→ 30 天后回到原点，无放大
```

### PASS: 持续闭环

```
邀请 → 双方都得 7 天高级功能 → 高级功能产生分享内容
→ 分享内容引流 → 新用户注册 → 又触发邀请
→ 闭环可以持续运转，越多用户越快
```

### FAIL: 留存差就堆奖励

```
D7 留存 8% → 加签到送积分 → 加首充 50% off
→ 数字短期上升，撤销奖励后回到 8%
→ 真实问题：核心价值未被感知，奖励只在掩盖
```

### PASS: 先修核心价值

```
1. 用户调研：D7 流失原因（不是奖励不够，是没看到价值）
2. 改 onboarding：48h 内必须出现 1 个 aha moment
3. 留存稳定到 25% 后，再叠飞轮
→ 飞轮是放大器，不是修补剂
```
