
# S 形增长曲线

## 适用场景
- 判断产品/业务当前处于导入期、成长期、成熟期还是衰退期。
- 规划"第二曲线"——在当前业务成熟前孵化下一个增长引擎。
- 与 [bcg-matrix](../bcg-matrix/SKILL.md) 配合：S 曲线判断阶段，BCG 做组合决策。

## 核心约束
- 四个阶段的工作重点完全不同，**用成熟期的策略做导入期 = 死**：
  - 导入期：找 PMF、种子用户（关注留存率）。常见错误：过早规模化
  - 成长期：规模化获客、建立流程（关注增长率）。常见错误：忽略单位经济
  - 成熟期：提高壁垒、优化利润（关注毛利率）。常见错误：不启动第二曲线
  - 衰退期：收割现金、迁移客户（关注现金流）。常见错误：继续大量投入
- 破局点和拐点事后才能精确确认，事前只能通过多信号交叉判断。
- 第二曲线必须在第一曲线到达拐点**之前**启动，否则资源已经不够。

## 不适用场景
- 多业务组合决策：S 曲线看单条业务的阶段，多业务比较用 [bcg-matrix](../bcg-matrix/SKILL.md)。
- 需要具体运营策略：S 曲线判断阶段，各阶段的营销打法用 `customer-lifecycle`。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 用多个信号交叉判断当前阶段（增长率 + 客户类型 + 竞争格局）。
- [ ] 工作重点与阶段匹配。
- [ ] 如果在成长中后期，已考虑第二曲线启动时机和资源分配。

## 反模式

### FAIL: 成熟期还在烧钱获客

```
增速从 30% 降到 8%，竞品同质化严重
-> 继续加预算做增长 -> CAC 飙升 -> 利润消失
-> 应该：转向效率优化，启动第二曲线
```

### PASS: 阶段匹配 + 第二曲线

```
第一曲线（核心 SaaS）：成长后期，拐点临近
- 信号：增速从 40% 降到 15%，新客放缓
- 策略：优化单位经济，NRR > 120%，砍低 ROI 渠道

第二曲线（AI 功能模块）：导入期
- 已投入 3 个月，10 个种子客户测试中
- 资源分配：第一曲线 75% / 第二曲线 25%
- 里程碑：6 个月内 PMF 验证（留存 > 60%）
```
