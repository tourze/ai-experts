
# PLG 成熟度评估

## 适用场景
- 决定产品增长模式（PLG vs SLG vs 混合），需要系统化评估适用性。
- 已决定做 PLG，需要设计激活框架、衡量 Time-to-Value 或计算病毒系数。
- 与 [designing-growth-loops](../designing-growth-loops/SKILL.md) 配合设计增长飞轮，与 [pricing-strategy](../pricing-strategy/SKILL.md) 配合设计 freemium/free trial 策略。
- 需要诊断 PLG 漏斗哪个环节漏人最多。

## 核心约束
- PLG 不是万能药；先用适用性检查确认产品类型是否适合，再设计飞轮。
- 激活指标必须是用户行为而非功能使用（"发送第一条消息"而非"点击消息按钮"）。
- 病毒系数 K 和循环时间 ct 必须同时计算，K>1 但 ct=6 个月的增长很慢。
- Freemium 不等于免费 + 收费；免费版必须独立创造价值，不能是残废版。

## 代码模式

### PLG 适用性检查（4 级成熟度）

```markdown
| 维度 | L1 不适合 | L2 部分适合 | L3 适合 | L4 原生 PLG |
|---|---|---|---|---|
| 产品复杂度 | 需要实施团队 | 需要 onboarding 但可自助 | 注册即用 | 邀请即用 |
| 决策链 | 6+ 利益相关方 | 2-3 人 | 个人决策 | 团队自发扩散 |
| 价值感知速度 | 数周到数月 | 数天 | 数小时 | 数分钟 |
| 数据网络效应 | 无 | 弱 | 中 | 强 |
| 市场教育 | 需要大量教育 | 品类已有认知 | 替代品成熟 | 用户主动寻找 |
```

### 激活框架

```markdown
注册 → Setup Moment → Aha Moment → Habit Moment
  │        │              │              │
  │   完成关键配置    首次获得核心价值  重复使用形成习惯
  │   (< 5 min)     (< 24h)         (Week 1-2)
  │
  └─ 衡量：完成率、Time-to-Value、D1/D7/D30 留存
```

### 病毒系数计算

```markdown
K = i × c
- i = 每个用户平均发出的邀请数
- c = 每个邀请的转化率

循环时间 ct = 用户注册到发出第一个邀请的平均天数

有效增长速度 = K / ct

示例：
- i = 3（每用户邀请 3 人）
- c = 15%（邀请转化率）
- K = 0.45（< 1，病毒传播不能独立驱动增长，需要付费获客补充）
- ct = 2 天

K > 1 → 自然病毒增长
0.5 < K < 1 → 病毒传播有辅助作用，降低 CAC
K < 0.5 → 病毒传播贡献可忽略
```

### Freemium 转化漏斗

```markdown
| 阶段 | 指标 | 健康基准 |
|---|---|---|
| 注册 → 激活 | Setup Completion Rate | >60% |
| 激活 → Aha | Time to First Value | <1 天 |
| 免费 → 付费 | Freemium Conversion Rate | 2-5% (B2C), 5-15% (B2B) |
| 付费 → 扩展 | Seat/Usage Expansion Rate | >10%/年 |
```

## 检查清单
- [ ] 已完成 PLG 适用性检查，确认产品类型适合或部分适合。
- [ ] 激活指标是用户行为（而非功能点击），且有明确的时间窗口。
- [ ] 病毒系数 K 和循环时间 ct 同时给出，不只看 K。
- [ ] Freemium 边界有明确的"免费版独立价值"说明。

## 反模式

### FAIL: 所有产品都 PLG

```
"竞品都在做 PLG，我们也做"
→ 产品需要 3 个月实施 + 定制开发
→ 免费版用户注册后什么也做不了
→ 半年后：10 万注册，50 付费，支持团队被免费用户工单淹没
```

### PASS: 先评估再决定

```
PLG 适用性检查 → L2（部分适合）
→ 决策：PLG 做获客入口 + SLG 做大客户转化（混合模式）
→ 免费版：核心功能可用，限 3 人团队
→ 自助升级：移除人数限制
→ 大客户：SDR 跟进活跃免费团队（PQL）
```

### FAIL: 激活 = 注册

```
"我们的激活率 80%"
→ 定义：完成注册 = 激活
→ 实际：注册后 72% 从未使用核心功能
→ D7 留存 5%
→ "激活率"是虚荣指标
```

### PASS: 行为化激活定义

```
激活 = 注册后 48h 内完成以下全部：
1. 创建第一个项目
2. 邀请至少 1 个队友
3. 完成第一个核心工作流

激活率 = 32%（基于行为定义）
激活用户 D30 留存 = 65%
未激活用户 D30 留存 = 4%
→ 激活是留存的强预测因子，优化激活是 P0
```
