# 9 种 Epic 分解模式

## 1. 用户旅程分解
Epic → 按用户完成目标的步骤拆
例：电商下单 → 浏览 | 搜索 | 加购 | 结算 | 支付 | 确认 | 评价

## 2. 角色/Persona 分解
Epic → 按不同用户角色拆
例：报表系统 → 运营看实时数据 | 经理看周报 | 财务看月度汇总

## 3. 业务规则分解
Epic → 按业务规则的复杂度分层
例：定价引擎 → 固定价 | 阶梯价 | 会员价 | 促销叠加 | 跨境税率

## 4. 数据/平台分解
Epic → 按数据来源或平台拆
例：数据同步 → MySQL | PostgreSQL | MongoDB | API webhook

## 5. 性能/质量分解
Epic → 先做能用，再做好用，最后做极致
例：搜索功能 → 精确匹配 | 模糊匹配 | 拼音/同义词 | 个性化排序

## 6. CRUD 分解
Epic → 按创建、读取、更新、删除拆
例：文章管理 → 创建草稿 | 发布 | 编辑 | 归档 | 删除

## 7. 时间/频率分解
Epic → 按触发频率或时间窗口拆
例：通知系统 → 即时通知 | 每日摘要 | 周报汇总 | 里程碑提醒

## 8. 依赖解耦分解
Epic → 先做无外部依赖的部分，再做有依赖的
例：支付集成 → 模拟支付（沙箱）| 单一支付渠道 | 多渠道 | 对账

## 9. Walking Skeleton 分解
Epic → 先跑通端到端最小链路，再逐层丰富
例：审批流 → 单人审批 | 多级审批 | 会签 | 驳回 | 委托 | 催办

## 选择策略
| 场景 | 推荐模式 |
|---|---|
| Epic 是用户流程 | 用户旅程 |
| Epic 涉及多角色 | 角色/Persona |
| Epic 规则复杂 | 业务规则 |
| Epic 跨平台/数据源 | 数据/平台 |
| Epic 有技术不确定性 | 性能/质量 或 Walking Skeleton |
| Epic 是实体管理 | CRUD |
| Epic 涉及定时/周期 | 时间/频率 |
| Epic 依赖外部系统 | 依赖解耦 |
