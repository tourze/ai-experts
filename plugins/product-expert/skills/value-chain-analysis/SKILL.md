# 价值链分析

## 适用场景
- 识别竞争优势具体来自哪个环节（不是笼统的"技术优势"）。
- 决定哪些环节该自建、哪些该外包。
- 与 [porters-five-forces](../porters-five-forces/SKILL.md) 配合：五力看行业，价值链看自己。

## 核心约束
- 主要活动和支持活动都要分析，不能只看"做产品"忽略"做管理"。
- 每个环节必须评估：优势还是劣势？高成本还是高价值？与竞品对比如何？
- 价值链分析的终点是**决策**：加强、外包、改造或放弃——不是画完图就结束。
- 找到核心优势环节后，要论证其**不可复制性**。

## 不适用场景
- 评估行业结构：价值链看自己内部，行业结构用 [porters-five-forces](../porters-five-forces/SKILL.md)。
- 纯互联网/平台业务：传统价值链（入站物流->生产->出站物流）不完全适用，需要改造为数字化价值链（获取->激活->留存->变现->推荐）。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 主要活动和支持活动都已覆盖。
- [ ] 每个环节有明确的优势/劣势判断和证据。
- [ ] 识别出核心竞争优势所在的环节及其不可复制性。
- [ ] 给出了具体决策（加强/外包/改造）。

## 反模式

### FAIL: 把价值链当组织架构图

```
研发部 → 生产部 → 销售部 → 客服部
→ 这是组织架构不是价值链
→ 价值链关注"价值如何被创造和传递"，不是"谁在做"
```

### PASS: 聚焦价值差异

```
技术开发：自研推荐算法，CTR 比行业均值高 40% → 核心优势，持续投入
入站物流：数据采集依赖第三方 API，成本占 25% → 劣势，评估自建数据管道
市场与销售：CAC 是竞品 2 倍 → 劣势，Q2 重构获客漏斗
→ 每个环节都有量化对比和行动
```
