# 商业模式画布

## 适用场景
- 新业务立项、现有业务诊断、融资前商业模式梳理。
- 需要把价值创造、价值交付和价值获取放进同一张画布中讨论。

## 核心约束
- 先定义客户细分和价值主张，再讨论渠道、收入模式和成本结构。
- 画布里每一格都要注明依据，至少区分“已验证”和“待验证”。
- 不要把功能清单、组织架构或愿景口号误写成商业模式。

## 代码模式
```markdown
| 模块 | 当前判断 | 证据 | 主要风险 |
| --- | --- | --- | --- |
| 客户细分 | ... | ... | ... |
| 价值主张 | ... | ... | ... |
```

## 检查清单
- [ ] 九大模块完整，且相互之间逻辑闭环。
- [ ] 收入来源、渠道成本与目标客户规模相匹配。
- [ ] 关键假设、验证方法和下一步动作已标记。

## 反模式

### FAIL: 目标客户”所有人”

```
| 客户细分 | 所有需要数据分析的企业、个人和机构 |
→ 销售不知找谁，产品不知为谁设计
```

### PASS: 显式画像 + 证据

```
| 客户细分 | 50-200 人 SaaS 的 RevOps Lead | 已访谈 12 家 |
| 价值主张 | 月度收入归因从 3 天压到 2 小时 | A/B 客户证实 |
```

### FAIL: 只有收入没成本

```
收入：$99 × 1000 = $99K/月  → 漂亮
忘了：客服 $20K + 服务器 $15K + 销售提成 $25K + 退款 5%
→ 真实毛利是负的
```

### PASS: 收入对应成本

```
收入：$99/用户/月
直接成本：服务器 $15 + 客服 $20 + 退款 $5 = $40
毛利：$59 (60%)
CAC $200，回本 3.4 个月
```

### FAIL: 差异化空泛口号

```
差异：更智能、更易用、更高效
→ 所有竞品都这么写
```

### PASS: 可验证差异点

```
差异：自动 Stripe + Salesforce 双向同步（竞品仅单向）
证据：竞品功能矩阵
风险：竞品 6 个月内补齐 → 差异消失
```
