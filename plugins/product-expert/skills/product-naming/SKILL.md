# 产品命名协作

## 适用场景
- 重要产品/品牌命名，需要从定位、用户心智和竞品环境共同推导名字。
- 需要快速头脑风暴时可先用 [product-name](../product-name/SKILL.md)，需要定位依据时可配合 [obviously-awesome](../obviously-awesome/SKILL.md)。

## 核心约束
- 先理解产品本质和命名边界，再进入候选名发散。
- 命名路线应分主题展开，例如功能型、情绪型、隐喻型，而不是一锅乱炖。
- 收敛前必须看竞品与语义冲突，避免闭门造车。

## 代码模式
```markdown
产品灵魂 -> 命名约束 -> 命名路线 -> 候选名 -> 竞品验证 -> 最终推荐
```

## 检查清单
- [ ] 产品定位、目标用户、语言环境和禁用词已明确。
- [ ] 候选名按路线分组，且每组有命名逻辑。
- [ ] 已检查读音、歧义、竞品雷同和品牌延展性。
- [ ] 最终建议附带淘汰理由与后续验证方式。

## 反模式

### FAIL: 50 个名字盲选

```
[一份 Excel 50 行候选名]
"你最喜欢哪个？"
→ 创始人选了一个，但没人记得为什么
→ 上线后没法回答 "Why this name?"
```

### PASS: 分主题 + 收敛

```
路线 A（功能型）：FlowKit / TaskBoard
路线 B（情绪型）：Calm / Steady
路线 C（隐喻型）：Compass / Anchor
→ 每路线 5 个，先选路线，再决定名字
→ 收敛后 3 个最终候选 + 决策依据
```

### FAIL: 新奇压倒可读

```
名字：Yqfx
→ 域名好买，无人能读
→ 口耳相传失败
→ SEO 全靠付费
```

### PASS: 三秒规则

```
- 听一遍能写出来吗？
- 不解释能猜行业吗？
- 中英文/拼音都通顺吗？
- 域名 .com / .ai 可获取？
→ 通过 → 进竞品/法律检查
```
