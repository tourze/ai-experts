# BLM 业务领先模型

## 适用场景
- 年度战略规划：从差距出发，倒推战略和执行问题。
- 战略执行不力诊断：找出"战略正确但落不了地"的原因。
- 与 [mckinsey-7s](../mckinsey-7s/SKILL.md) 配合：7S 聚焦组织匹配，BLM 覆盖战略+执行全链路。

## 核心约束
- BLM 从**差距**出发（业绩差距 + 机会差距），不是从愿景出发。
  - 业绩差距 = 实际 vs 目标（可量化）
  - 机会差距 = 现有业务 vs 新机会（需要洞察）
- 战略侧（市场洞察/战略意图/创新焦点/业务设计）和执行侧（关键任务/正式组织/人才/氛围文化）必须**对称分析**。华为经验：70% 的战略失败是执行问题。
- 领导力和价值观是贯穿始终的底座，不是摆设——要有具体判断。

## 不适用场景
- 早期验证阶段（Pre-PMF）：BLM 假设已有业绩基线可以对比，没有历史数据就无法计算差距。用 [idea-validator](../idea-validator/SKILL.md)。
- 纯执行问题：如果战略方向明确只需解决落地，直接用 [mckinsey-7s](../mckinsey-7s/SKILL.md) 诊断组织匹配。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 从差距出发而非从愿景出发。
- [ ] 战略侧四要素和执行侧四要素都有分析。
- [ ] 业绩差距有量化数据。
- [ ] 领导力和价值观不是空话，有具体判断。

## 反模式

### FAIL: 只有战略没有执行

```
市场洞察很深刻 → 战略方向很清晰 → 业务设计很完美
→ 然后呢？谁来做？组织支不支撑？人够不够？
```

### PASS: 战略执行对称

```
战略：进入企业级市场（机会差距：企业市场 TAM 是现有 C 端的 5 倍）
执行：
- 关键任务：搭建企业销售团队 + 产品企业版改造
- 组织：需要新设 Enterprise BU
- 人才：缺 ToB 销售 VP（Q1 招到）
- 文化：从工程师文化转向客户成功文化（最难的部分）
```
