# BCG 矩阵（波士顿矩阵）

## 适用场景
- 多产品/多业务线公司的资源分配、投资优先级排序。
- 与 [ge-matrix](../ge-matrix/SKILL.md) 配合：BCG 看高层全景，GE 做更精细多维评估。

## 核心约束
- **相对**市场份额：对标最大竞品，不是绝对数字。分界线按行业调——SaaS 的"高增长"和制造业完全不同，不要默认 10%/1.0。
- 问题象限是唯一需要 Go/No-Go 选择的——给窗口期 + 里程碑，到期不达标就砍。
- 全投 = 没决策。BCG 的核心价值是**强迫取舍**。
- 最终输出必须包含资源流向：现金牛 →→ 明星 + 被选中的问题业务，瘦狗 → 退出释放资源。

## 不适用场景
- 单产品公司：BCG 是多业务组合工具，单产品没有"取舍"可做。用 [s-curve-growth](../s-curve-growth/SKILL.md) 判断生命阶段。
- 需要精细评估：BCG 只有 2x2，太粗时升级到 [ge-matrix](../ge-matrix/SKILL.md)。
- 评估外部竞争：BCG 看内部组合，不看行业结构。用 [porters-five-forces](../porters-five-forces/SKILL.md)。

## 检查清单
- [ ] 相对市场份额是对比最大竞品，不是绝对数字。
- [ ] 分界线设定有行业依据。
- [ ] 问题业务做了选择（不能全投也不能全砍）。
- [ ] 给出了明确的资源流向建议。

## 反模式

### FAIL: 所有业务都是明星

```
产品A: 明星 → 加大投入
产品B: 明星 → 加大投入
产品C: 问题 → 加大投入变明星
→ 全投 = 没决策，BCG 的核心是取舍
```

### PASS: 敢砍敢投

```
产品A（明星）：市场增速 25%，份额第一 → 追加 40% 预算
产品B（现金牛）：增速 3%，份额第一 → 砍 30% 预算，利润输血 A
产品C（瘦狗）：增速 -2%，份额第四 → Q3 前完成出售或关停
产品D（问题）：增速 20%，份额第三 → 给 6 个月窗口期，份额进前二则追投，否则放弃
```
