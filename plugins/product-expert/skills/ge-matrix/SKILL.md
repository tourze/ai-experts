# GE-麦肯锡矩阵（九宫格矩阵）

## 适用场景
- BCG 太粗需要更精细评估时的升级工具。
- 与 [bcg-matrix](../bcg-matrix/SKILL.md) 配合：BCG 做快速分类，GE 做深度评估。

## 核心约束
- X 轴竞争实力（内部）、Y 轴市场吸引力（外部），都是多指标加权得分。
- **权重不能平均分配**——权重必须反映行业关键成功因素：
  - SaaS：增长率 30%、NRR 天花板 25%（增长是估值核心）
  - 制造业：盈利性 30%、进入壁垒 25%（资本密集）
  - 快消品：市场规模 30%、竞争强度 25%（渠道为王）
- 九宫格结论是**战略方向**，不是精确数字——争论 3.2 还是 3.3 没有意义。
- 内外部维度至少各用 5 个指标，打分差异 > 1 分才有区分度。

## 不适用场景
- 快速粗筛：GE 需要大量指标和权重设定，快速决策用 [bcg-matrix](../bcg-matrix/SKILL.md)。
- 缺乏行业数据：权重设定依赖行业理解，数据不足时 GE 的精度优势不存在。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 指标权重有行业依据，不是平均分配。
- [ ] 内外部维度至少各 5 个指标。
- [ ] 九宫格定位转化为了具体战略动作。
- [ ] 与 BCG 分析结果交叉验证（如有）。

## 反模式

### FAIL: 所有指标权重一样

```
市场规模 20%、增长率 20%、盈利性 20%、竞争 20%、技术 20%
→ 等于没做判断，权重应反映行业关键成功因素
```

### PASS: 行业化权重

```
SaaS 行业市场吸引力权重：
- 市场增长率 30%（增长是 SaaS 估值核心）
- NRR 天花板 25%（决定复合增长潜力）
- 竞争集中度 20% / 监管风险 15% / 技术替代 10%
→ 权重体现行业特性，打分结果有区分度
```
