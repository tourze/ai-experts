# 获客渠道经济性分析

## 适用场景
- 季度预算分配，需要决定各获客渠道的投入比例。
- 新渠道评估：是否值得投入、预期 ROI 和达到规模化的时间。
- 渠道效率下降诊断：为什么同样的投入带来的客户变少了。
- 与 [saas-metrics](../saas-metrics/SKILL.md) 配合计算 CAC 和 LTV/CAC，与 [funnel-architect](../funnel-architect/SKILL.md) 配合设计转化漏斗。

## 核心约束
- CAC 必须**按渠道分拆**计算，不能只看混合 CAC；混合 CAC 掩盖低效渠道。
- 渠道评估必须同时看**获客成本和客户质量**（该渠道客户的 LTV、留存和扩展率）。
- 渠道有生命周期：早期红利 → 规模化 → 饱和 → 衰退；判断当前阶段再决策。
- 至少保持 2-3 个健康渠道，不能把 80% 预算押在单一渠道上。

## 代码模式

### 渠道经济性矩阵

```markdown
| 渠道 | 月投入 | 新客数 | 渠道CAC | 客户LTV | LTV/CAC | Payback | 阶段 |
|---|---|---|---|---|---|---|---|
| 内容SEO | | | | | | | 规模化 |
| Google Ads | | | | | | | 饱和 |
| 社交广告 | | | | | | | 早期 |
| 推荐计划 | | | | | | | 规模化 |
| SDR外呼 | | | | | | | 规模化 |
| 合作伙伴 | | | | | | | 早期 |
```

### 渠道生命周期判断

```markdown
早期红利：CAC 持续下降，获客量快速增长 → 加大投入
规模化：CAC 稳定，获客量线性增长 → 维持并优化
饱和：CAC 开始上升，增量放缓 → 控制预算，寻找新渠道
衰退：CAC 显著上升，同比获客下降 → 减少投入，保留品牌底线
```

### 渠道质量评估

```markdown
| 渠道 | 30天留存 | 6月NRR | 平均扩展率 | 支持工单/客 | 质量评级 |
|---|---|---|---|---|---|
```

## 检查清单
- [ ] CAC 按渠道分拆，不是只看混合数字。
- [ ] 每个渠道同时评估了获客成本和客户质量。
- [ ] 判断了每个渠道的生命周期阶段。
- [ ] 预算分配没有超过 50% 押在单一渠道。

## 反模式

### FAIL: 只看混合 CAC

```
"我们的 CAC 是 $200，行业基准 $300，很好"
→ 细拆：SEO $50（占 20%）、广告 $400（占 60%）、SDR $600（占 20%）
→ 广告和 SDR 的 CAC 远超基准
→ 混合 CAC 被低成本 SEO 拉低，掩盖了问题
```

### PASS: 按渠道分拆决策

```
SEO $50, LTV/CAC 10x → 继续投入内容团队
广告 $400, LTV/CAC 1.8x → 低于 3x 线，优化或缩减
SDR $600, LTV/CAC 3.5x → CAC 高但客户质量好，维持
→ 缩减广告 30%，转投内容和 SDR
```

### FAIL: 渠道饱和还在加钱

```
Google Ads 过去 3 个月：
- 月投入：$10K → $15K → $20K
- 新客数：50 → 55 → 52
- CAC：$200 → $273 → $385
→ 典型饱和曲线，追加投入几乎无增量
```

### PASS: 识别饱和并转移

```
Google Ads 进入饱和期 → 预算封顶 $12K/月
释放 $8K/月 → 测试 2 个新渠道（LinkedIn Ads + 合作伙伴推荐）
每渠道 $4K/月，跑 3 个月看 CAC 趋势和客户质量
```
