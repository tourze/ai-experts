# 理想客户画像

## 适用场景
- B2B/B2B2C 早期定位、销售聚焦、市场切入和外呼名单筛选。
- 需要完整方法时可阅读 [references/full-guide.md](references/full-guide.md)。
- 需要结合想法验证或市场空间时，可配合 [idea-validator](../idea-validator/SKILL.md) 与 [market-sizing-analysis](../market-sizing-analysis/SKILL.md)。

## 核心约束
- ICP 必须同时覆盖公司画像、买方角色、使用者角色和购买触发因素。
- 先找高痛点、高付费意愿、高成交概率的客户，而不是“最大的市场”。
- 画像要服务销售和产品决策，不是堆一份漂亮的人设。

## 代码模式
```markdown
| 维度 | 理想特征 | 排除条件 |
| --- | --- | --- |
| 公司规模 | ... | ... |
| 购买人 | ... | ... |
```

## 检查清单
- [ ] 公司画像、买方画像和用户画像已区分。
- [ ] 已说明触发事件、预算来源和典型异议。
- [ ] 有“做什么客户”和“不做什么客户”两套标准。
- [ ] 结果可直接指导销售名单、信息架构和路线图。

## 反模式

### FAIL: ICP = 所有人

```
ICP："任何想提升效率的中小企业"
→ 销售名单 = 所有公司
→ 转化率 1%，CAC 高得离谱
```

### PASS: 窄而深

```
ICP（核心）：
- 公司：中文区电商，年 GMV 500w-5000w，员工 20-100
- 买方：运营总监（负责数据看板）
- 触发：刚完成融资 / 新增渠道
- 排除：传统零售 / 海外品牌 / 员工 < 20
→ 销售名单从 10 万缩到 800
→ 转化率 8%，CAC 降低 60%
```

### FAIL: 只人口统计

```
ICP："25-40 岁，月收入 1w+，一线城市"
→ 不知道为什么会买
→ 营销投放精准但转化低
```

### PASS: 加触发 + 痛点

```
人口（基线）：25-40 岁，月入 1w+
触发事件：
- 刚换工作（3-6 个月内）
- 刚结婚 / 刚有娃
- 刚搬家
痛点：搬家后通勤时长 +40 分钟
→ 投放定向："换工作 + 通勤 > 1h"
```
