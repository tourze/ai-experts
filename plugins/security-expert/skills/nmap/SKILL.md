# 网络侦察与端口扫描

## 适用场景
- 需要建立资产清单、开放端口和服务版本基线。
- 需要为 [wireshark-analysis](../wireshark-analysis/SKILL.md) 和 [protocol-reverse-engineering](../protocol-reverse-engineering/SKILL.md) 提供后续切入点。
- 需要做低噪声扫描或对比多轮结果的变化。

## 核心约束
- 先定义目标：找主机、找端口、识别服务，还是验证特定假设。
- 默认从最小必要扫描开始，不把 `-A -p-` 当起手式。
- 把 TCP、UDP、脚本探测和版本识别拆开执行，方便归因。
- 对生产网络控制速率、并发和重试。

## 代码模式
```bash
nmap -Pn -p 80,443,8080 10.0.0.0/24
nmap -Pn -sV -sC -p 22,80,443 10.0.0.10
nmap -Pn -sU -p 53,161 10.0.0.10
```

## 检查清单
- 确认目标网段、禁扫范围、速率限制和联系人。
- 保留原始 XML/grepable 输出，便于复测对比。
- 把开放端口与真实业务、暴露面和可利用性对应起来。
- 对误报端口做二次验证。

## 反模式
- 一开始就全端口、全脚本、高并发扫描。
- 只看端口开放，不确认实际服务。
- 把脚本输出直接当成漏洞结论。
