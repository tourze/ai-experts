# 弹窗转化优化（popup-cro）

## 适用场景
- 设计或审计邮件捕获弹窗、折扣弹窗、退出挽回弹窗。
- 需要调整触发时机、文案、频控、移动端体验和关闭路径。
- 想判断某个弹窗是否真的带来增量，而不是只增加展示量。

## 核心约束
- 先定义弹窗目标与触发条件，再谈文案和视觉。
- 触发必须服务上下文；不同流量来源、页面类型和设备不应共用同一策略。
- 弹窗要易于关闭、可频控、可记忆用户选择，不能靠遮挡和误触强收集。
- 若任务变成整页转化诊断，切到 [cro-methodology](../cro-methodology/SKILL.md)；若需要代码实现，切到 [redesign-my-landingpage](../redesign-my-landingpage/SKILL.md)。

## 代码模式
- 推荐输出模板：

```md
目标：博客页邮件订阅
触发：滚动 45% + 停留 35 秒
文案：给出“继续读什么”而不是泛泛订阅
频控：7 天仅展示 1 次
验证：订阅率、关闭率、页面跳出率
```

## 检查清单
- 是否明确了触发逻辑、频控和关闭行为。
- 是否给出移动端和桌面端的差异策略。
- 是否说明主指标与副作用指标。
- 是否把弹窗价值与当前页面内容挂钩。

## 反模式

### FAIL: 5 秒弹窗

```
访客刚进站 → 还没看完首屏 → 弹窗”订阅”
→ 70% 直接关闭并跳出
→ 弹窗成为流量杀手
```

### PASS: 上下文触发

```js
// 滚动 60% + 停留 30s + 阅读了 2 篇文章
if (scrollPct > 0.6 && timeOnPage > 30 && readArticles >= 2) {
  showPopup({ relevant: getCurrentArticleTopic() });
}
```

### FAIL: “订阅 newsletter”

```
“订阅我们的 newsletter
获取最新的产品动态”
→ 用户：”凭什么？”
→ 转化率 0.3%
```

### PASS: 上下文价值

```
[正在读”销售自动化”文章时]
弹窗：”看完这 5 个销售自动化 case study？
打包 PDF 邮件给你 → [输入邮箱]”
→ 转化率 5-12%
```

### FAIL: 无频控

```
用户关闭弹窗 → 切换页面 → 弹窗又来
→ 心理拉黑该网站
```

### PASS: 频控 + 记忆

```js
localStorage.setItem('popup_dismissed_at', Date.now());
// 7 天内不再展示
// 已订阅用户 → 永久不展示
```
