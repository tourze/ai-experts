# GEO 引用增强模式

提升内容被 ChatGPT、Claude、Perplexity、Gemini 等 AI 助手引用的概率。

---

## 统计引用块

统计数据使 AI 引用率提升 15-30%。必须附来源。

```markdown
[声明]。根据 [来源/机构]，[带数字和时间范围的具体统计]。[为什么这很重要]。
```

**示例：**
```markdown
移动端优化不再是可选项。根据 Google 2024 年 Core Web Vitals 报告，70% 的 Web 流量来自移动端，未通过移动可用性标准的页面跳出率高 24%。
```

---

## 专家引言块

具名专家归属增加可信度和引用可能性。

```markdown
"[专家直接引语]，"[专家姓名]说，[机构]的[职位]。[1 句上下文或解读]。
```

**示例：**
```markdown
"从关键词驱动搜索到意图驱动发现的转变，是自移动优先索引以来 SEO 最重大的变化，"SparkToro 联合创始人 Rand Fishkin 说。这意味着内容策略必须超越传统关键词优化。
```

---

## 权威声明块

结构化声明以便 AI 轻松提取，带清晰归因。

```markdown
[主题][动词：是/需要/包含][清晰具体的声明]。[来源][确认/报告/发现][支撑证据]。这[意味着/说明][可执行洞察]。
```

---

## 自包含答案块

创建可被 AI 直接引用的独立陈述。

```markdown
**[主题/问题]**：[完整、自包含的答案，无需额外上下文即可理解。2-3 句中包含具体细节、数字或示例。]
```

**示例：**
```markdown
**SEO 博客文章的理想长度**：竞争性话题的最佳长度为 1,500-2,500 词。这个范围既能全面覆盖主题又能保持读者参与度。HubSpot 研究显示长文内容获得的外链比短文多 77%，直接影响搜索排名。
```

---

## 证据三明治块

用证据夹层结构最大化可信度。

```markdown
[开头声明]。

支撑证据：
- [带来源的数据点 1]
- [带来源的数据点 2]
- [带来源的数据点 3]

[将证据连接到可执行洞察的结论]。
```

---

## 分领域 GEO 策略

不同内容领域受益于不同的权威信号。

| 领域 | 关键策略 |
|------|----------|
| 技术 | 版本号、日期、官方文档引用、代码示例 |
| 医疗健康 | 同行评审研究、专家资质（MD/RN）、研究局限性 |
| 金融 | 监管机构引用、带时间范围的具体数字、教育免责声明 |
| 法律 | 具体法条、管辖权说明、专业咨询声明 |
| 商业/营销 | 带可衡量结果的案例、行业报告引用、思想领袖引语 |

## AI 引用审计模板

### Prompt 分类（每类生成 15-20 个）

| 类别 | 示例格式 |
|------|----------|
| 直接推荐 | "最好的 [X] 是什么？"、"推荐一个 [X]" |
| 对比 | "[A] vs [B]"、"[X] 的替代品" |
| 功能导向 | "支持 [功能] 的 [X]"、"有 [能力] 的 [X]" |
| 场景导向 | "[行业] 用的 [X]"、"[具体场景] 的 [X]" |
| 价格导向 | "免费的 [X]"、"最便宜的 [X]" |
| 迁移导向 | "[竞品] 的替代品"、"从 [A] 迁移到 [B]" |

### 审计记录格式

每个 prompt 记录：
1. 品牌是否被提及（Y/N，位置）
2. 哪些来源被引用（URL 列表）
3. 哪些竞品被提及（附位置）
4. 主导来源类型（目录/榜单/品牌站/Reddit）
5. AI 对每个推荐包含了什么信息（功能/价格/优缺点）

### 差距分析输出

- 品牌出现的 prompt 集群
- 品牌缺席的 prompt 集群
- 最常被引用的域名排名
- 站外目标：需要入驻的目录/榜单
- 站内差距：需要创建/更新的内容页
- 格式差距：竞品用了对比表/FAQ/结构化数据而你没有
