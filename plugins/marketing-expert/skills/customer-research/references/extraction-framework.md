# 提取框架

## 逐条提取维度

| 维度 | 说明 |
|---|---|
| Jobs to Be Done | 功能性（完成什么任务）、情感性（想要什么感受）、社会性（想被怎么看） |
| 痛点 | 当前方案哪里让用户痛苦，严重程度 1-5 |
| 触发事件 | 是什么事件让用户开始寻找新方案 |
| 期望结果 | 用户定义的"成功"是什么 |
| 语言 / 词汇 | 用户原话中的关键表达，保留原始措辞 |
| 考虑过的替代方案 | 用户提到的其他工具、方式或竞品 |

## 合成步骤

1. **主题聚类**：按相似洞察分组，命名主题。
2. **频率 + 强度打分**：频率（出现次数）× 强度（用户情绪强烈程度 1-5）。
3. **按画像分段**：不同角色 / 公司规模 / 使用场景是否有显著差异。
4. **提取"金句"**：标记最能代表主题的原始引用，用于后续文案。
5. **标记矛盾点**：不同用户群体之间的对立观点，单独记录。

## 数字水源地渠道

| 渠道 | 适合挖掘的内容 |
|---|---|
| Reddit | 真实吐槽、替代方案讨论、使用场景 |
| G2 / Capterra | 结构化优缺点、评分趋势、竞品对比 |
| Hacker News | 技术用户偏好、深度讨论 |
| LinkedIn | 决策者视角、行业趋势 |
| YouTube 评论 | 教程反馈、功能诉求 |
| App Store / Play Store | 移动端体验、崩溃投诉、功能缺失 |
