# 安索夫矩阵

## 适用场景
- 增长策略选择：现有增长放缓，评估下一步方向。
- 年度战略规划中的增长路径设计。
- 与 [s-curve-growth](../../../product-expert/skills/s-curve-growth/SKILL.md) 配合：S 曲线判断阶段，安索夫选增长方向。

## 核心约束
- 2x2 矩阵：现有/新产品 x 现有/新市场。风险从市场渗透（最安全）到多元化（最危险）递增。
- **大多数企业应该先把市场渗透做到极致**，再考虑其他象限。渗透率 15% 就跳去做多元化 = 浪费。
- 每个象限不是非此即彼，可以同时做多个但必须有明确优先级。
- 多元化分相关（有协同效应）和非相关（纯新领域），非相关多元化的失败率极高。

## 不适用场景
- Pre-PMF 阶段：还没找到 PMF 就谈增长方向是空中楼阁。先用 [idea-validator](../../../product-expert/skills/idea-validator/SKILL.md)。
- 需要具体增长机制：安索夫定方向，具体增长飞轮设计用 [designing-growth-loops](../../../product-expert/skills/designing-growth-loops/SKILL.md)。

## 检查清单
- [ ] 四个象限都做了评估。
- [ ] 市场渗透是否已做到极致（否则不急着跳象限）。
- [ ] 有明确的优先级排序。
- [ ] 高风险象限有风险缓解方案。

## 反模式

### FAIL: 跳过渗透直奔多元化

```
现有市场渗透率 15% -> "我们要做国际化 + 新产品线"
-> 现有市场还有 85% 空间没吃，为什么要去冒险？
```

### PASS: 逐步扩张

```
1. 市场渗透（优先级 1）：渗透率 15% -> 目标 30%
   策略：月付转年付、推出团队版、抢竞品流失客户
2. 产品开发（优先级 2）：基于客户反馈开发 AI 分析模块
   策略：先做 MVP 给现有大客户测试
3. 市场开发（优先级 3）：Q3 试水东南亚
   策略：先做本地化 + 找渠道伙伴
4. 多元化（暂不考虑）：资源有限，专注 1-3
```
