# 产品类型 Onboarding 模板

## B2B SaaS

- 激活定义：完成核心工作流（如创建第一个项目/报告/集成）
- 重点：团队邀请是关键激活杠杆；个人试用转团队使用
- 注意：决策者和使用者可能不同，引导需区分角色

## 双边市场

- 分别定义供需双方的激活事件
- 供给侧：完成第一次上架/发布
- 需求侧：完成第一次浏览/收藏/下单
- 重点：先解决鸡蛋问题，再优化匹配体验

## 移动应用

- 激活窗口更短（通常 < 24 小时）
- Push 通知替代邮件作为召回手段
- 权限请求时机：在用户理解价值之后再请求
- 重点：首次打开体验决定是否被卸载

## 内容平台

- 激活定义：消费 N 篇内容 或 关注 N 个创作者
- 冷启动：兴趣选择 → 个性化推荐 → 快速展示价值
- 重点：内容质量是最好的引导；推荐算法的准确性 = 激活率
