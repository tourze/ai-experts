# 多渠道协同引导矩阵

## 邮件 + 产品内引导时间表

| 时机 | 产品内 | 邮件 |
|------|--------|------|
| 注册即时 | 引导流/Checklist | 欢迎邮件（不重复产品内内容） |
| 注册后 1 天未激活 | 登录后弹出恢复引导 | 价值提醒 + 一键直达操作 |
| 注册后 3 天未激活 | — | 案例/教程邮件 |
| 注册后 7 天未激活 | — | 最后尝试 + 人工联系（高价值） |
| 完成激活 | 庆祝 + 下一步引导 | 进阶功能邮件 |

## 触发式邮件原则

- 基于用户行为触发，不是固定时间表
- 每封邮件只有一个 CTA
- 深度链接到产品内具体操作页面
- 退出条件：用户已激活则停止发送

## 停滞用户分级挽回

| 用户价值 | 策略 | 手段 |
|----------|------|------|
| 低 | 自动化 | 触发式邮件序列 + 产品内提示 |
| 中 | 半自动 | 个性化邮件 + 直播/Webinar 邀请 |
| 高 | 人工 | CSM 主动联系 + 定制化 onboarding session |

## 停滞检测标准

- 注册后 24 小时未完成首次关键操作
- 引导流中途放弃（记录放弃步骤）
- 登录但未执行任何有意义操作
- Checklist 长期停留在同一项

## 产品内恢复引导

- 检测用户回访时的上次离开位置
- 简化恢复路径（"继续上次的操作"按钮）
- 根据停滞原因调整引导内容（A/B 测试不同恢复方案）

## 检查清单

- [ ] 邮件与产品内引导不重复
- [ ] 邮件基于行为触发而非固定时间
- [ ] 每封邮件只有一个 CTA 和深度链接
- [ ] 已激活用户自动退出引导序列
- [ ] 有明确的停滞检测标准
- [ ] 按用户价值分级处理
- [ ] 产品内有恢复引导机制
- [ ] 高价值用户有人工触达通道
