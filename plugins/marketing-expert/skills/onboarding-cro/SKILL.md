# 注册后引导与用户激活优化（onboarding-cro）

## 适用场景

- 激活率低、time-to-value 过长
- 空态页面缺乏引导
- 引导流步骤过多或顺序不合理
- 多渠道引导缺乏协同
- 高价值用户停滞未激活

## 核心约束

1. **Time-to-value 是一切** — 前 5 分钟决定留存；每多一步无价值操作都在流失用户
2. **每次只给一个目标** — 一个 session 只推一件事，不同时展示多个 CTA
3. **让用户做，不是看** — 交互式引导优于被动教程
4. **进度创造动力** — 进度条、打勾、里程碑庆祝降低放弃率

## 定义激活

1. 对比留存用户与流失用户行为差异，找到 aha moment
2. 验证因果关系（不是"留下的人做了 X"，而是"做了 X 导致留下"）
3. 激活事件须与长期留存强相关，窗口 1-7 天
4. 不同角色/场景可有不同激活定义
5. 核心指标：激活率、激活时间、激活步骤数、漏斗各步转化率

> 详见 `references/activation-metrics.md`

## 引导流设计

**注册后第一步** — 三选一：产品优先（自解释产品）、引导设置（需个性化）、价值优先（先展示结果）

**Checklist** — 3-7 项，按价值排序；预填"创建账号"利用蔡格尼克效应；可跳过但记录原因

**空态** — 行动导向文案 + 示例数据/模板 + 社会证明；不写"暂无数据"

**Guided Tour** — 最多 3-5 步，可退出可重来，Tooltip 优于 Modal

> 详见 `references/flow-design.md`

## 多渠道协同

- 邮件与产品内引导内容不重复
- 基于行为触发，非固定时间表
- 每封邮件只有一个 CTA + 深度链接
- 已激活用户自动退出引导序列

> 时间矩阵详见 `references/multichannel-matrix.md`

## 停滞用户处理

**检测**：24h 未完成首次关键操作 / 引导中途放弃 / 登录但无操作

**分级挽回**：低价值→自动化邮件+产品内提示；中→个性化邮件+Webinar；高→CSM 人工联系

**产品内恢复**：检测上次离开位置，提供"继续上次操作"入口

## 度量指标

激活率（B2B SaaS 基准 20-40%）| 激活时间（目标 < 1h）| 引导完成率（> 60%）| Day 1/7/30 留存

漏斗分析：找跌幅最大的步骤、按渠道/设备/角色分群、关注步骤完成时间分布

## 反模式

- 注册后先填长问卷 → 渐进式收集
- 20 步 guided tour → 最多 3-5 步
- 空态只写"暂无数据" → 行动导向 + 模板
- 所有用户同一引导流 → 轻量分群差异化
- 激活定义 = 注册完成 → 找真正 aha moment
- 引导完就不管 → 继续 Day 7/30 留存引导
