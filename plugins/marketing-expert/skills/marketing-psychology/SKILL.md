# 营销心理学（marketing-psychology）

## 适用场景
- 解释用户为何迟迟不转化，或为何对某种表达反应更好。
- 需要把心理模型映射到定价页、产品页、邮件或促销设计。
- 团队想理解“为什么这个策略有效”，而不只复制做法。

## 核心约束
- 先确认业务场景和用户决策阶段，再选模型；不能把所有偏差都套一遍。
- 心理学模型用于解释和设计，不用于伪造紧迫感或操纵用户。
- 输出必须从“心理机制 → 设计动作 → 风险提示”三层展开。
- 若要落地到具体说服手法，配合 [influence-psychology](../influence-psychology/SKILL.md)；若要规划内容主题，配合 [content-strategy](../content-strategy/SKILL.md)。

## 代码模式
- 推荐输出模板：

```md
场景：定价页
心理机制：损失厌恶 + 选择过载
设计动作：默认推荐一个方案；把“不购买的代价”写清楚
风险：不要制造虚假对比和伪稀缺
```

## 检查清单
- 是否说清楚用户当前在做什么决策。
- 是否把抽象模型翻译成页面、文案或流程动作。
- 是否标明哪些做法属于高风险操纵，不建议采用。
- 是否区分“提升理解”与“强迫转化”。

## 反模式

### FAIL: 只讲理论名词

```
“利用锚定、损失厌恶、社会证明、互惠、稀缺、权威、宜家效应...”
→ 客户不知道具体该改什么
```

### PASS: 机制 → 动作 → 风险

```
场景：定价页转化弱
机制：选择过载（5 个套餐让用户瘫痪）
动作：默认推荐”专业版”，标”最热门”
落地：PricingTable 加 isRecommended prop
风险：别把推荐标在最贵套餐
```

### FAIL: 给低质量产品背书

```
留存差 → 加”不升级损失 30%”+ 伪稀缺 + 假 logo
→ 短期转化升，长期口碑崩 + 法律风险
```

### PASS: 先修产品再用心理学

```
留存差 → 留存调研 → 修核心痛点 → 心理学优化转化路径
心理学是放大器，不是修补剂
```

### FAIL: 转化问题全归心理

```
转化低 → “加紧迫感”
→ 实际：LCP 8s，注册流程 12 步
```

### PASS: 先排除技术问题

```
1. 页面性能（LCP/INP）
2. 流程摩擦（步骤数、错误率）
3. 信息架构
4. 心理设计（最后一层）
```
