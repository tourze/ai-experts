# 序列类型、文案准则与产出模板

## 序列类型与节奏

### 欢迎序列（Welcome）
- 邮件数：5-7 封 | 周期：12-14 天
- 节奏：Day 0 即时 → Day 1 → Day 3 → Day 5 → Day 8 → Day 12
- 目标：建立品牌认知、传递核心价值、引导首次关键动作
- 退出条件：完成关键动作或进入付费流程

### 线索培育序列（Lead Nurture）
- 邮件数：6-8 封 | 周期：2-3 周
- 节奏：每 2-3 天一封，逐步提升承诺门槛
- 目标：从认知推进到考虑再到决策
- 退出条件：预约 demo、开始试用或明确标记为不感兴趣

### 唤醒序列（Re-engagement）
- 邮件数：3-4 封 | 周期：2 周
- 节奏：Day 0 → Day 4 → Day 8 → Day 14
- 目标：唤回沉默用户，确认是否仍有兴趣
- 退出条件：重新活跃或确认退订

### 上手引导序列（Onboarding）
- 邮件数：5-7 封 | 周期：14 天
- 节奏：基于用户行为触发，非固定时间表
- 目标：推动用户完成激活里程碑
- 退出条件：达成激活指标或试用到期

## 邮件文案准则

### 结构：Hook → Context → Value → CTA → Sign-off
1. **Hook**：开头一句抓住注意力，与读者当前状态共鸣
2. **Context**：简要交代背景或问题
3. **Value**：给出核心价值——洞察、方法、资源或好处
4. **CTA**：一个明确的行动号召
5. **Sign-off**：收尾签名，保持品牌一致性

### 格式规范
- 段落不超过 3 行，段间留白
- 用项目符号拆解列表信息
- 关键信息加粗，每封邮件加粗不超过 3 处

### 篇幅控制
| 类型 | 字数范围 | 适用场景 |
|------|---------|---------|
| 事务型 | 50-125 词 | 确认、提醒、状态更新 |
| 教育型 | 150-300 词 | 教程、技巧、案例分享 |
| 故事型 | 300-500 词 | 品牌故事、客户案例、深度洞察 |

## 主题行策略

- 清晰优于聪明，读者 2 秒内知道邮件关于什么
- 长度 40-60 字符，移动端完整显示
- preheader 与主题行互补，不要重复

| 模式 | 示例 |
|------|------|
| 提问型 | "你的欢迎邮件打开率低于 30% 吗？" |
| How-to 型 | "如何用 3 封邮件把试用转为付费" |
| 数字型 | "提升邮件转化的 5 个被忽略的细节" |
| 直接型 | "你的试用还剩 3 天" |
| 故事悬念型 | "我们差点取消了这个功能" |

### CTA 规范
- 主 CTA 用按钮，措辞以动词开头（"开始试用""查看指南""预约演示"）
- 辅助 CTA 用文本链接，不与主 CTA 竞争
- 每封邮件只设一个主 CTA，辅助链接最多两个

## 产出模板

### 序列总览

```
序列名称：[名称]
触发条件：[什么事件或条件启动这个序列]
序列目标：[用一句话说明最终要推动的结果]
邮件数量：[N 封]
发送节奏：[时间间隔描述]
退出条件：[哪些行为导致用户离开此序列]
```

### 单封邮件详情

```
邮件 #N — [内部命名]
发送时机：[触发后第 N 天 / 事件触发]
主题行：[主题行文本]
预览文本：[preheader 文本]
邮件目标：[这封邮件要完成的单一任务]
正文：
  Hook: [开头]
  Context: [背景]
  Value: [核心价值]
  CTA: [行动号召，含按钮文本和链接描述]
Sign-off: [签名]
```
