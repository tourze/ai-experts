# 注册流程转化优化（signup-flow-cro）

## 适用场景
- 优化注册、账号创建、试用激活的转化率。
- 诊断注册字段过多、流失严重、完成率低。
- 设计新产品注册体验，权衡信息收集与转化效率。

## 核心约束
- 先展示价值，再要求承诺。
- 每多一个字段，流失率上升 5-10%；只留当前步骤必须的字段。
- 减少感知努力：进度指示、内联校验、即时反馈。
- 消除不确定性：告知注册后会发生什么、数据怎么用、能否随时取消。

## 字段优化指南
- **邮箱**：内联校验 + 域名拼写纠错（`gmial.com → gmail.com`）。
- **密码**：显示/隐藏切换，强度指示器替代复杂规则，去掉确认字段。
- **姓名**：B2C 用单字段；B2B 需正式称呼时再拆分。
- **社交登录**：B2C 用 Google+Apple；B2B 用 Google+Microsoft+SSO。按钮放表单上方。
- **手机号**：除非核心功能需要，否则延后到注册后收集。

## 单步 vs 多步

| 条件 | 方案 |
|------|------|
| <= 3 字段 | 单步，一屏完成 |
| 4-6 字段 | 多步，每步 2-3 字段 + 进度条 |
| > 6 字段 | 重审是否都是注册必需 |

渐进承诺：邮箱(门槛最低) → 密码+姓名(沉没成本) → 定制化(可跳过)。

## 信任与减摩
- CTA 旁标注"无需信用卡"；用具体动作替代"提交"——"免费开始""开始试用"。
- 一句话隐私说明，链接完整政策。
- 内联校验：失焦即反馈；错误要具体（"邮箱缺少 @"而非"格式错误"）。
- 表单旁放社会证明：用户数、评分、客户 logo。

## 移动端优化
- 触摸目标 >= 44px，单列布局，字段间距充足。
- 按字段类型切换键盘：`type="email"` / `type="tel"`。
- 启用 `autocomplete`；社交登录在移动端优先级更高。
- CTA 固定在可视区域，不被键盘遮挡。

## 提交后体验
- 立即展示确认 + 下一步指引，不要空白页。
- 邮箱验证：显示目标地址 + 重发按钮 + 修改邮箱入口。
- 等待期间提供有价值内容（产品视频、快速指南）。

## 度量指标

| 指标 | 健康基准 |
|------|---------|
| 表单开始率 | > 60% |
| 完成率 | > 70%(单步) / > 50%(多步) |
| 单字段流失 | < 5% |
| 填写时长(单步) | < 60s |
| 校验错误率 | < 15% |

## 检查清单
- 只保留注册必需字段，其余延后。
- 要求信息前已展示产品价值。
- 内联校验覆盖所有字段，错误提示具体。
- 移动端适配键盘类型、触摸目标、单列布局。
- CTA 传达用户将获得什么。
- 社交登录放在显眼位置。
- 提交后有确认和下一步指引。
- 部署了字段级流失追踪。

## 反模式

### FAIL: 一次要太多
```
姓、名、邮箱、密码、确认密码、手机、公司、职位、规模、行业
→ 10 字段 → 完成率 < 20%
```

### PASS: 渐进收集
```
第一步：邮箱+密码（或 Google 登录）→ 进入产品后再补全资料
→ 完成率 > 75%
```

### FAIL: 注册后死胡同
```
"注册成功！请查看邮箱。" → 空白页 → 用户关标签页 → 激活率 < 40%
```

### PASS: 注册后即时引导
```
显示目标邮箱 + [重发] + [改邮箱] + 产品介绍视频 → 激活率 > 70%
```
