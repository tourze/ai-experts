# 人性化改写（content-humanizer）

## 适用场景
- 用户明确反馈“这段文案太像 AI 写的”。
- 需要把已有初稿改得更像真人说话，有品牌语气和判断。
- 想先检测 AI 腔问题，再决定重写范围。

## 核心约束
- 先诊断再动刀；不知道问题类型，就不要直接“润色”。
- 人性化不等于口语化堆砌，必须保留事实、结构和品牌边界。
- 避免凭空注入不存在的个人经历、客户案例或主张。
- 若任务是页面级营销文案精修，配合 [copy-editing](../copy-editing/SKILL.md)；若是选题规划，切到 [content-strategy](../content-strategy/SKILL.md)。

## 代码模式
- 先跑诊断分数：

```bash
python3 scripts/humanizer_scorer.py draft.txt --json
```

- 再用 [ai-tells-checklist](references/ai-tells-checklist.md) 和 [voice-techniques](references/voice-techniques.md) 做定向改写。

## 检查清单
- 是否标出了 AI 腔最重的词、句型、段落节奏问题。
- 是否保留原文事实，不靠编故事“变真实”。
- 是否把品牌语气说清楚：直接、专业、克制、锋利，还是亲切。
- 是否说明哪些问题需要重写，哪些只需微调。

## 反模式

### FAIL: 删几个 AI 词就完事

```
原：”Furthermore, it's worth noting that...”
改：”另外，需要指出...”
→ AI 检测仍 95% / 节奏仍像 AI / 用户读完仍觉得空
```

### PASS: 重写句法 + 注入观点

```
原：”这款产品具有以下显著优势...”
改：”我们用了 3 个月才把响应时间做进 200ms。
不是吹，是被三个早期客户骂出来的。”
→ 节奏短促 / 有具体数字 / 有真实场景
```

### FAIL: 注入虚构经历

```
“为了显得真实” → 编”我们 2019 年遇到一个客户...”
→ 客户：”这是哪个客户？”
→ 答不上 → 信任崩
```

### PASS: 用真实事实

```
拿真案例（征得客户同意 / 已脱敏）：
“X 公司 50 人团队，从 Slack 切换我们用了 3 周。
最难的不是技术，是说服 CFO 这不是又一个工具。”
→ 真实但脱敏
```

### FAIL: 情绪词堆砌

```
“超级棒！！！绝对推荐！！！这个工具改变了一切！！！”
→ 像微商不像专业品牌
```

### PASS: 克制 + 具体

```
“用了一个月。报表时间从 4h 降到 30min。
不是魔法，就是每周省 14h。”
→ 数字 + 平实 + 留下空间
```
