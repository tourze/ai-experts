# 销售赋能材料（sales-enablement）

## 适用场景
- 制作 pitch deck、one-pager 或销售提案。
- 编写异议处理文档或 demo 脚本。
- 构建 ROI 计算器或买家画像卡。

## 核心约束
- **销售信任真实数据**——材料基于真实场景，不是市场部想象。
- **场景驱动**——锁定具体买家角色、行业和购买阶段。
- **30 秒可扫读**——核心信息前置，详细内容放附录。
- **锚定业务结果**——每页回答"对客户生意意味着什么"。

## 按买家类型定制
- **技术买家**：架构适配、安全合规、集成成本。
- **经济买家**：ROI、总拥有成本、回收周期。
- **内部倡导者**：对比表、一句话价值、审批模板。
- **终端用户**：易用性、学习曲线、日常体验。

## 检查清单
- [ ] 锁定了具体买家角色和购买阶段？
- [ ] 前 30 秒能抓到核心信息？
- [ ] 每个论点有数据或案例支撑？
- [ ] 用客户语言而非公司术语？
- [ ] CTA 明确且低摩擦？
- [ ] 差异化点是竞品做不到的？
- [ ] 销售知道怎么用、什么时候用？

## 反模式
| 反模式 | 正确做法 |
|--------|----------|
| 功能清单 deck | 每个功能关联一个业务结果 |
| 万能 one-pager | 按行业或角色做版本 |
| 虚构 ROI | 保守假设 + 标注来源 |
| 产品导览式 demo | 从客户目标倒推展示顺序 |
| 提案模板化 | 每份包含客户专属内容 |
| 材料做完就扔 | 附带使用场景说明 |
