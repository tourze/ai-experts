# B2B 冷邮件（cold-email）

## 适用场景
- 给从未联系过的目标客户写首封邮件，目标是获得回复。
- 设计 3-5 封跟进序列。
- 按职级（C-suite / VP / 一线）调整措辞密度。
- 诊断现有冷邮件回复率低的原因。

## 核心约束
- 先确认 ICP、目标职级、痛点假设和期望动作，再写邮件。
- 正文上限 125 词；C-suite 不超过 75 词。
- 一封邮件只放一个低摩擦 CTA。
- 个性化必须连接到业务问题，不做表面寒暄。
- 生命周期邮件切 `email-sequence`；销售材料切 `sales-enablement`。

## 写作原则
1. **像同行而非供应商**——语气是聪明同事分享相关发现。
2. **每句话挣得下一句的阅读权**——删掉不推动阅读的句子。
3. **个性化落到问题上**——证明你理解处境，不是展示调研。
4. **先进入对方的世界**——开头从对方现实出发。
5. **一个请求，低摩擦**——首封不要求 30 分钟通话。

## 语气与风格
- **C-suite**：3-4 句，只讲结果和战略影响。
- **VP / 总监**：多一层细节，讲问题表现和解决路径。
- **一线 / 技术角色**：可精确到指标、技术栈或工作流。
- 禁止感叹号、全大写、emoji、群发感措辞。

## 结构框架

按场景选择：

| 框架 | 步骤 |
|------|------|
| 观察-问题-证据-请求 | 具体观察 → 隐藏问题 → 一句话证据 → 请求 |
| 问题-价值-请求 | 疑问句开头 → 具体价值 → 请求 |
| 触发事件-洞察-请求 | 公开事件 → 带来的挑战/机会 → 请求 |
| 故事-桥梁-请求 | 相似客户问题(2句) → 连接对方处境 → 请求 |

## 主题行准则
- 2-4 词，全小写，不加标点，像内部邮件。
- 好：`quick question`、`[名字], one idea`、`next quarter`。
- 禁止假 `Re:` / `Fwd:` 前缀。

## 跟进序列
- 3-5 封，间隔递增（+3d → +5d → +7d → +14d）。
- 每封新增一个信息点，跟进比首封更短。
- 详见 [followup-patterns](references/followup-patterns.md)。

## 质量检查
- [ ] 词数在限制内？
- [ ] 首句关于对方而非自己？
- [ ] 个性化连接到业务问题？
- [ ] 单一低摩擦 CTA？
- [ ] 读起来像私人邮件？
- [ ] 主题行短、小写、内部感？

## 反模式
详见 [anti-patterns](references/anti-patterns.md)。关键禁区：`I hope this finds you well`、功能列表、HTML排版、假Re:/Fwd:、首封要求30分钟通话、跟进只说 checking in。
