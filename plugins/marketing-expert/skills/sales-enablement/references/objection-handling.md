# 异议处理

## 异议分类
1. **价格** — "太贵了""超出预算"
2. **时机** — "现在不是好时候""下个季度再说"
3. **竞品** — "我们在用 X""X 也能做"
4. **权限** — "我做不了决定""要问老板"
5. **现状** — "现在的方案还行""换的成本太高"
6. **技术** — "能和我们系统集成吗""安全合规怎么办"

## 响应框架

每条异议按五步处理：

| 步骤 | 说明 |
|------|------|
| 异议原文 | 客户说了什么 |
| 真实原因 | 为什么这样说 |
| 回应话术 | 怎么回答 |
| 证据支撑 | 用什么数据或案例证明 |
| 跟进问题 | 回应后问什么推进对话 |

## 原则
- 先确认而非反驳——"理解您的顾虑"不是"但是"。
- 回应中嵌入客户自己说过的目标。
- 跟进问题必须是开放式的，推动对话而非终结对话。
