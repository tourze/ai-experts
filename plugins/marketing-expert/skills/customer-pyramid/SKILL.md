# 顾客金字塔模型

## 适用场景
- 客户分层管理：不同价值客户给不同服务。
- 资源分配优化：把最好的资源给最有价值的客户。
- 与 [churn-prevention](../churn-prevention/SKILL.md) 配合：金字塔定分层，流失预防做留存。

## 核心约束
- 分层依据是**利润贡献（CLV）**，不是收入或消费频次。高收入低利润客户可能是铅层。
- 铅层客户服务成本 > 利润贡献——不是所有客户都值得留。对铅层的处理方案（提价/减服务/放弃）是必须做的决策。
- 目标不是把所有人变成铂层，而是让每层客户在对应策略下最大化价值。
- 金字塔健康度要看：铂+金层利润占比（是否过度依赖头部）、铅层占比（负利润是否过多）、层级流动性（有多少客户在升降级）。

## 不适用场景
- 客户数 < 100：样本太少无法统计分层，直接逐个客户管理。
- 纯 PLG/自助产品（无客户经理）：金字塔假设可以差异化人工服务，纯自助产品用产品内分群替代。

## 检查清单
- [ ] 分层依据是利润贡献（CLV），不是收入。
- [ ] 四层都有差异化策略。
- [ ] 铅层客户有明确处理方案（不是忽略）。
- [ ] 做了金字塔健康度评估。

## 反模式

### FAIL: 所有客户一视同仁

```
所有客户享受同样的服务标准
-> 铂层客户觉得"花这么多钱没特殊待遇"
-> 铅层客户消耗了不成比例的服务资源
```

### PASS: 差异化服务

```
铂层（50 家，贡献 60% 利润）：每家配专属 CSM，季度业务评审
金层（200 家，贡献 25% 利润）：共享 CSM 团队，月度 check-in
铁层（1000 家，贡献 15% 利润）：自助平台 + 知识库 + AI 客服
铅层（300 家，-5% 利润）：下季度提价 30%，不接受则不续约
-> 资源集中在高价值客户
```
