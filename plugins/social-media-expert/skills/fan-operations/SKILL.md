# 粉丝运营

## 适用场景

- 需要回复评论、私信或设计忠粉维护动作。
- 想从“发完就走”变成“持续互动”，提高粉丝粘性和复访率。
- 需要围绕粉丝互动反推选题、内容节奏或社群活动。
- 需要把公域粉丝进一步沉淀到私域时，联动 [private-domain](../private-domain/SKILL.md)。
- 需要统一品牌语气和人设时，联动 [personal-branding-advanced](../personal-branding-advanced/SKILL.md)。

## 核心约束

- 先解决“回复是否及时”和“回复是否有信息量”，不要只追求高频复制粘贴。
- 评论区和私信中的承诺必须真实可交付，不要为了互动率乱开口子。
- 粉丝运营的目标是提高长期信任，不是短期刷互动。
- 对外表达必须与账号定位一致；账号主张模糊时，先回到 [xiaohongshu-commercial-growth](../xiaohongshu-commercial-growth/SKILL.md) 确认定位。
- 涉及站外导流前，先检查平台安全边界，并可先走 [social-platform-safety](../social-platform-safety/SKILL.md) 的过滤流程。

## 代码模式

高质量评论回复模板：

```text
@用户名 先回答你的问题：<直接结论>。
如果你现在的情况是 <场景>，优先做 <动作>。
你更关心 <选项 A> 还是 <选项 B>？我可以继续展开。
```

新粉欢迎语模板：

```text
嗨，我是 <账号名>。
这里主要分享 <核心内容>，你如果正在解决 <问题>，可以先看我主页置顶。
后面你想看哪类内容，也可以直接回我。
```

每周互动复盘模板：

```markdown
## 本周互动数据
- 新增评论：
- 回复率：
- 私信咨询：
- 被反复问到的话题：

## 下周动作
- 继续放大：
- 停止重复：
- 新增互动实验：
```

## 检查清单

- 评论回复先给结论，再给补充信息。
- 回复内容中至少有一个“继续对话”的钩子问题。
- 新粉欢迎语没有夸张承诺或站外导流冲动。
- 已把高频问题沉淀成后续选题或 FAQ。
- 涉及社群承接或名单沉淀时，已联动 [private-domain](../private-domain/SKILL.md)。
- 账号语气、人设和价值主张与 [personal-branding-advanced](../personal-branding-advanced/SKILL.md) 保持一致。

## 反模式

- 只回“谢谢”“收到”“爱你们”，没有任何信息增量。
- 自动群发欢迎语，导致所有粉丝看到同一套机械回复。
- 把评论区当成交场，频繁甩链接、导流或催单。
- 对负面评论只删不答，错过识别真实阻碍的机会。
- 没有复盘高频问题，导致内容和互动反复重复。
