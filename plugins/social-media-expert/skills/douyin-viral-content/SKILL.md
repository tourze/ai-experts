# 抖音爆款文案生成

## 适用场景

- 用户只有一句金句、一本书摘录或一段主题素材，需要产出可发布的抖音文案。
- 用户已经有历史文案样本，希望总结哪些表达更容易带来完播、互动和转发。
- 用户提供视频摘要或逐字稿，需要二次改写成更适合短视频的口播文本。
- 需要先提取视频内容时，配合 [douyin-video-summary](../douyin-video-summary/SKILL.md) 使用。

## 核心约束

- 不要承诺“必定 5 星”或“保证爆款”；评分只能作为内部比较，不是结果保证。
- 只有在用户明确提供历史样本、目录或表现数据时，才做校准；否则使用通用基线并明确说明。
- 不要虚构播放量、完播率、账号体量和历史数据。
- 输出必须保留平台安全边界，不给违规导流、虚假收益或极端承诺。
- 当素材本身信息不足时，优先补问题，不要硬凑长文案。

## 代码模式

如果用户给了历史文案目录，可按以下模式整理样本：

```bash
find 已发布 -name '*.md' | sort | head
```

推荐输出模板：

```markdown
# 标题
<20 字内钩子标题>

## 口播文案
<3-6 段，单段不超过 40 字>

## 推荐标签
#标签1 #标签2 #标签3 #标签4

## 复盘说明
- 核心钩子：
- 情绪方向：
- 互动动作：
- 适配人群：
```

## 检查清单

- 已说明是否存在“历史样本校准”。
- 口播脚本开头 1-2 句具备明确钩子，不绕弯。
- 标签数量控制在 4-6 个，不堆砌。
- 评分理由与文案结构一一对应，而不是只报分不解释。
- 如果使用了历史数据，已标明是基于用户给定样本的归纳。
- 需要进一步输出视频摘要或引用原视频内容时，已链接到 [douyin-video-summary](../douyin-video-summary/SKILL.md)。

## 反模式

### FAIL: 空洞词汇堆砌

```
“绝对的好东西，强烈推荐！
颠覆你的认知，必看！
3 个让人惊艳的方法...”
→ 没有具体画面 / 没有情绪转折 / 完播率 < 5%
```

### PASS: 画面 + 情绪 + 动作

```
（开场，对镜头）
“我朋友在淘宝花 800 买的，我 80 块就做了一模一样的”
（转身指向桌上成品）
“3 步，你也可以”
（细节特写）
→ 有钩子 / 有反差 / 有动作 / 完播率高
```

### FAIL: 假承诺播放量

```
“按这个改完，预计播放 100w+，账号必涨 5000 粉”
→ 用户：”没达到怎么办？”
→ 信任崩
```

### PASS: 显式说明评分仅供比较

```
评分：78 / 100（基于内部 rubric）
- 钩子强度：8/10
- 节奏感：7/10
- 信息密度：6/10
注：评分仅用于版本对比，不预测真实播放量
```
