# data-storytelling

## 适用场景

- 已经有分析结果，但用户真正需要的是“怎么讲清楚”“怎么组织成报告或汇报”。
- 需要把大量指标压缩成 headline、关键洞察、风险、建议、下一步动作。
- 需要给周报、月报、复盘、董事会材料或项目结论页设计叙事顺序。
- 相关 skill：[data-analysis](../data-analysis/SKILL.md)、[data-visualization](../data-visualization/SKILL.md)、[statistical-analysis](../statistical-analysis/SKILL.md)。

## 核心约束

- 先确认“想让谁做什么”，再决定故事结构；没有目标受众就没有叙事。
- 一个故事只保留 1 个主结论和 2 到 4 个支撑点，避免把所有图表原样堆上去。
- 结论必须能回指具体数据证据；需要时引用 [data-analysis](../data-analysis/SKILL.md) 或 [statistical-analysis](../statistical-analysis/SKILL.md) 的结果。
- 建议先写 headline，再补证据，再给行动项，不要反过来。

## 代码模式

```markdown
# 结论先行

## 发生了什么
- 指标 A 环比上升 18%
- 指标 B 同比下降 6%

## 为什么重要
- 这意味着渠道结构正在变化
- 如果不处理，下一周期利润率会继续被压缩

## 建议动作
1. 优先修复转化率最低的入口
2. 继续保留增长最快的高质量渠道
```

```text
主结论 -> 支撑证据 -> 风险边界 -> 决策建议 -> 下一步负责人
```

## 检查清单

- 主结论是否能在 1 句话里说清楚。
- 每个支撑点是否都有数字证据，而不是形容词。
- 是否明确说明了不确定性、样本边界或统计限制。
- 是否知道这份材料最终要驱动哪个动作。
- 如果需要图表，是否把设计要求同步给 [data-visualization](../data-visualization/SKILL.md)。

## 反模式

### FAIL: dashboard 截图当报告

```
"上周数据看这里："
[贴 12 张 Tableau 截图]
→ 业务方："所以呢？"
```

### PASS: 结论先行 + 行动项

```md
# 上周收入下滑 18%，主因：华东大客户集中续费推迟

## 证据
- 华东 Top 5 客户：3 家本月续费推迟到下月
- 影响金额：$220k（占周收入 78%）

## 建议
1. 销售本周 1v1 跟进 3 家延迟客户
2. 财务调整 Q1 forecast
```

### FAIL: 形容词堆砌

```
"用户增长大幅提升，留存明显改善，整体表现非常强劲"
→ 大幅 = 多少？明显 = 多少？非常 = 相对什么基线？
```

### PASS: 量化 + 基线

```
"DAU 周环比 +23%（基线均值 +5%），主要由邮件 reactivation 驱动；
留存 D7 30% → 36%（CI 95% [+4pp, +8pp]）"
```

### FAIL: 因果相关混说

```
"开通会员的用户留存高 40% → 推动所有用户开会员"
→ 自选择偏差，未做 RCT
```

### PASS: 显式标注证据强度

```
"观察：会员组留存高 40%（相关性，未实验）
推断：会员可能因素 OR 高意愿用户自选会员
建议：本月 A/B 测一组"强制免费试用会员"，30 天看留存"
```
