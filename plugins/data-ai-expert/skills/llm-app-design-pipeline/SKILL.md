# LLM 应用设计 Pipeline

本 skill 提供设计 LLM 应用的结构化流程，覆盖从应用形态选择到离线评估的完整链路。适用于新应用设计和既有应用优化两种场景。

## 使用场景

- 新 LLM 应用（聊天机器人 / RAG / Agent / 内容生成）的架构设计
- 既有 LLM 应用的效果优化（准确率、延迟、成本、用户满意度）
- Prompt 工程与检索策略的系统性调优

## Pipeline 五步

### Step 1: 应用形态确认

| 形态 | 特征 | 适用场景 |
|------|------|---------|
| 单 Prompt | 一次调用完成 | 分类/抽取/翻译/摘要 |
| 多步 Chain | 多次调用串行 | 复杂推理/多阶段处理 |
| RAG | 检索 + 生成 | 知识密集型问答 |
| Agent | 工具调用 + 决策循环 | 多步任务/系统操作 |
| 对话 | 多轮上下文 | 客服/助手/教育 |

确认后定义成功指标：

| 指标类型 | 示例 | 测量方法 |
|---------|------|---------|
| 任务通过率 | 准确率 / F1 / BLEU | 自动 eval |
| 引用准确率 | Groundedness / Faithfulness | RAG 专用 eval |
| 人工偏好 | 人类评审胜率 | A/B 测试 |
| 延迟 | TTFT / TPS / 端到端延迟 | 性能监控 |
| 成本 | Token 单价 / 月费用 | 账单分析 |

### Step 2: 逐段优化

按「输入侧 → 检索 → 推理 → 输出」四段逐一检查：

#### 输入侧

- 用户输入预处理（清洗、截断、意图识别）
- 上下文窗口管理（system prompt / few-shot / 历史）
- 多模态输入处理

#### 检索（RAG 专用）

- 切块策略（固定长度 / 语义 / 递归 / 文档结构）
- Embedding 选型（维度、多语言、领域微调）
- 索引选型（HNSW / IVF / PQ / 混合）
- 检索策略（top-k、相似度阈值、metadata filter、reranking）
- 混合检索（关键词 + 语义 + 知识图谱）

#### 推理

- Prompt 结构（角色 / 任务描述 / 约束 / 输出格式）
- Few-shot 示例选择与数量
- Chain-of-Thought / Tree-of-Thought / ReAct
- 温度与采样参数
- 模型选型（能力 vs 成本 vs 延迟）

#### 输出

- 结构化输出（JSON mode / function calling / XML tags）
- 输出验证（schema 校验、格式修复）
- 后处理（去重、排序、引用标注）

### Step 3: 评估设计

| 评估类型 | 用途 | 方法 |
|---------|------|------|
| 离线 eval | 迭代优化 | 测试集 + 自动评分 |
| 在线 eval | 生产监控 | 采样 + 人工评审 |
| A/B 测试 | 方案对比 | 流量分流 + 统计检验 |
| 压力测试 | 边界情况 | 对抗样本 / 极端输入 |

评分 Rubric 设计要求：
- 每个维度 1-5 分，有明确的分级描述
- 包含正面和负面示例
- 覆盖至少 3 个维度（正确性、完整性、安全性）

### Step 4: 迭代优化

优化优先级：

| 优先级 | 优化方向 | 典型 ROI |
|--------|---------|---------|
| 1 | Prompt 调优 | 高（零成本，快速验证） |
| 2 | 检索策略调整 | 中高（RAG 应用核心杠杆） |
| 3 | Few-shot 示例优化 | 中（质量取决于示例选择） |
| 4 | 切块策略调整 | 中（影响召回率） |
| 5 | 模型升级/降级 | 低（成本高，边际改进递减） |
| 6 | 微调 | 低（数据成本高，仅在特定场景有效） |

每轮迭代只改一个变量，同一测试集对比，记录改动、假设和结果。

### Step 5: 上线与监控

- 延迟 SLA 与降级策略（fallback 模型、缓存、队列）
- 成本预算与熔断（token 上限、每日预算）
- 内容安全过滤（输出审查、PII 脱敏）
- 反馈收集（用户评分、红旗标记、人工评审队列）

## 红线

- **不跳过 eval 就上线**：无离线评估即盲目上线；每条改动绑定 eval case 与 baseline 对比方法。
- **不做无对照优化**：每次只改一个变量，否则无法归因。
- **禁止跨层归因**：检索失败不写成 prompt 问题，prompt 偏差不归因到 embedding。
- **不忽视延迟和成本**：准确率、延迟、成本三维权衡。
- **检索调参不报单点指标**：召回/延迟/内存三角缺一不可。
