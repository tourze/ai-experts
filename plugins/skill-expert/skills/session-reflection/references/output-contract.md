# 报告输出契约

固定五段，不换序。每段必须出现，没内容写「无」并解释为什么没有。

## 1. 一句话结论（TL;DR）

本轮哪条经验最值得沉淀；最值得改的一条是什么。

## 2. 高价值证据点

| 来源 | 现象 | 启示 |
|------|------|------|
| 用户原话 / 自己的失误 / 工具调用结果 | 具体描述 | 一句话提炼 |

来源必须可定位，不写「我观察到」。

## 3. 长期资产建议

### 3.1 全局记忆（MEMORY.md）

- `[新增]` 类型: feedback/project/user/reference — 标题 — 一句话理由
- `[更新]` 现有条目 X — 修改点
- `[删除]` 过时条目 Y — 原因

### 3.2 可复用工作流

步骤化的执行流（命令 / 检查点 / 退出条件），可直接拷进 CLAUDE.md 或新 skill。

### 3.3 Skill 优化

- 命中但表现差的 skill：描述应改成 …
- 漏命中的场景：应新增 / 调整哪个 skill 的 description 关键词
- 误命中：触发词需要排除什么

### 3.4 Hooks 使用

- 噪声 hook：哪个 hook 注入了无用上下文 → 缩窄触发条件
- 缺失 hook：哪类失误 hook 应该提前拦住 → 给出 hook 草案（事件 / 匹配条件）
- gate 误判：哪类回复被错误阻断 / 放行 → 给出修正逻辑
- runtime 故障不在本节：dispatch error / SKILL 脚本调用失败 → 引用 `trigger-telemetry-advisor` 的归因结论，再判断是否补 hook 设计建议

## 4. 不沉淀清单（明示放弃）

哪些观察是一次性的、不值得写入长期资产，原因是什么。
避免把短期上下文当成永久规则塞进记忆。

## 5. 可执行 Next Action

一条可立即执行的命令或一条可直接落地的改动（路径 + 行级建议）。
