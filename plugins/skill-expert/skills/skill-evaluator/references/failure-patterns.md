# 常见失败模式

### FAIL：教程模式（模式 1）

```
"什么是 PDF？"/"如何写 for 循环"/基础库用法
→ Claude 已知 → token 浪费
```

### PASS：纯专家知识

```
非显而易见选择的决策树
只有专家才知道的权衡
真实经验中的边缘案例
```

### FAIL：堆砌模式（模式 2）

```
SKILL.md 800+ 行，全内联
→ 上下文浪费 / 关键信息稀释
```

### PASS：渐进披露

```
SKILL.md < 300 行（路由+决策）
references/ 按需加载详情
```

### FAIL：隐形 Skill（模式 3）

```
description: "帮助处理文档任务"
→ Agent 永远不会触发它
```

### PASS：WHAT + WHEN + KEYWORDS

```
description: "创建、编辑和分析 .docx 文件。处理 Word 文档、修订跟踪或专业文档格式时使用。"
```

### FAIL：自由度错配（模式 4）

```
症状：创意任务给僵硬脚本，脆弱操作给模糊指引
根因：未考虑任务脆弱性
修复：高自由度给创意（原则，非步骤）
     低自由度给脆弱（精确脚本，无可变参数）
```
