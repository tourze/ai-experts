# 完成前验证检查清单

在声称任务完成、修复生效或测试通过之前，必须按此清单逐项核实。来源：已降级的旧 verification-before-completion 流程。

## 核心约束

**违反字面规则 = 违反规则精神。不存在"灵活变通"。**

- 没有运行验证命令，不声称完成。
- 没有读取完整输出，不判断结果。
- 上一次运行的结果不算——必须是当前消息中的新鲜证据。

## 验证门控流程

```
在做出任何完成/成功/满意声明之前：

1. 识别：哪个命令能证明这个声明？
2. 执行：运行完整命令（新鲜的、完整的）
3. 阅读：完整输出 + 退出码 + 失败数
4. 核实：输出是否支持声明？
   - 否 → 报告实际状态和证据
   - 是 → 带证据做出声明
5. 此时才可以：做出完成声明

跳过任何一步 = 撒谎，不是验证
```

## 常见声明与对应验证

| 声明 | 必须有的证据 | 不充分的证据 |
|------|------------|------------|
| 测试通过 | 测试命令输出：0 失败 | 上一次运行、"应该能过" |
| Lint 干净 | Lint 输出：0 错误 | 部分检查、推测 |
| 构建成功 | 构建命令：exit 0 | Lint 通过 ≠ 构建通过 |
| Bug 已修 | 原始症状已消失的证据 | 代码改了 ≈ 假设修好了 |
| 回归测试有效 | 红绿循环验证 | 测试通过一次 |
| 需求已满足 | 逐条对照清单 | 测试通过 ≠ 需求满足 |

## Red Flags — 出现以下念头时立即停下

| 念头 | 现实 |
|------|------|
| "应该没问题了" | 运行验证命令 |
| "我很有信心" | 信心 ≠ 证据 |
| "就这一次跳过" | 没有例外 |
| "Lint 过了" | Lint ≠ 编译 ≠ 测试 |
| "子代理说成功了" | 独立验证 |
| "我累了，差不多了" | 疲劳不是借口 |
| "部分检查够了" | 部分证明什么都不证明 |
| "换个说法就不算声明了" | 精神高于字面 |

## Rationalization 对照表

| 借口 | 现实 |
|------|------|
| "刚改了一行，肯定没问题" | 一行代码能搞崩整个系统。运行测试。 |
| "已经手动验证过了" | 手动验证不可追溯。运行自动化测试。 |
| "上次跑过测试了" | 上次的结果不代表现在。重新跑。 |
| "只是格式调整" | 格式工具有时会改语义。验证。 |
| "子代理报告成功了" | 子代理可能过于乐观。用 diff 和测试独立确认。 |

## 检查清单

- [ ] 识别了证明声明的命令
- [ ] 在当前消息中运行了完整命令
- [ ] 阅读了完整输出（不是截断的）
- [ ] 确认退出码和失败数
- [ ] 声明与证据一致
- [ ] 没有使用"应该""可能""看起来"等模糊词
- [ ] 完成 Task Closure 4 问 AAR 扫描（命中 → 已按 2/3 门槛沉淀；全否 → 跳过）
