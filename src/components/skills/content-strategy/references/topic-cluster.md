
# 主题聚类策略（topic-cluster）

## 适用场景
- 围绕一个核心主题设计 pillar page + 8-12 篇支撑文章。
- 规划聚类内链拓扑，避免关键词蚕食。
- 为内容团队输出带优先级和执行顺序的创作路线图。

## 核心约束
- 每篇支撑文章的主关键词必须与其他文章和 pillar page 不同。
- Pillar page 必须双向链接到所有支撑文章。
- 先做关键词分层（pillar / supporting / long-tail），再定文章。
- 输出必须包含蚕食检查和执行路线图。
- 单篇选题用 [content-strategy](../SKILL.md)；搜索优化配合 [seo](../../seo/SKILL.md)。

## 代码模式

### 1. 关键词分层

| 层级 | 搜索量 | 竞争度 | 角色 |
|------|--------|--------|------|
| Pillar | 1000+/月 | 高 | 聚类入口，覆盖广 |
| Supporting | 100-1000/月 | 中 | 展开子话题 |
| Long-tail | <100/月 | 低 | 精准意图，FAQ |

### 2. 支撑文章定义

每篇支撑文章需明确：

| 字段 | 说明 |
|------|------|
| 标题 | 含主关键词的工作标题 |
| 主关键词 | 必须唯一，不与其他文章重复 |
| 搜索意图 | 信息型 / 教程 / 商业比较 |
| 与 Pillar 的关系 | 展开哪个 H2 章节 |
| 优先级评分 | 按 [优先级框架](./cluster-prioritization.md) 计算 |

### 3. 内链拓扑

输出 Hub-and-spoke 链接矩阵：
- Pillar → 所有支撑文章（必须）
- 所有支撑文章 → Pillar（必须）
- 相关支撑文章之间交叉链接（每篇 2-3 个）
- 与站内现有内容的链接点

### 4. 蚕食检查

对每对可能重叠的文章，标注重叠查询和差异化策略。若无法差异化，合并为一篇。

### 5. 执行路线图

分三阶段：
- Phase 1（基础）：Pillar + 最高优先级 3-4 篇
- Phase 2（建权威）：中优先级 3-4 篇
- Phase 3（补全）：剩余 2-4 篇

## 检查清单
- 所有支撑文章的主关键词是否互不重复。
- Pillar page 是否双向链接到每篇支撑文章。
- 是否输出了蚕食检查表。
- 是否给出了分阶段执行路线图。
- 优先级评分是否有依据（量/难度/意图/聚类价值）。

## 反模式

### FAIL: 只列主题不做关键词分层

```
聚类：内容营销
文章：写博客、SEO 技巧、社交分发、内容策略
→ 没有搜索量、没有优先级、可能蚕食
```

### PASS: 关键词驱动 + 蚕食检查

```
Pillar: "内容营销指南" (vol: 2400, KD: 72)
Art 1: "B2B 内容营销策略" (vol: 880) — 展开 Pillar H2 §2
Art 2: "内容营销 ROI 计算" (vol: 320) — 展开 Pillar H2 §5
蚕食检查：Art 1 vs Art 3 "内容策略" 有重叠 → Art 3 限定为"SaaS 内容策略"
```
