# 主题集群优先级框架

## 适用场景

- 一个主题集群已有 10+ 个候选选题，需要排执行顺序。
- 需要平衡搜索量、关键词难度、商业价值和内容依赖关系。
- 需要避免先写低价值长尾，导致 pillar page 没有权重支撑。

## 评分模型

| 维度 | 分值 | 说明 |
| --- | --- | --- |
| 搜索需求 | 0-5 | 搜索量、趋势和问题频率 |
| 商业意图 | 0-5 | 是否靠近线索、试用、咨询或购买 |
| 难度/可赢性 | 0-5 | 竞争强度、现有权威、可差异化空间 |
| 集群价值 | 0-5 | 是否支撑 pillar、补齐关键路径或减少蚕食 |
| 制作成本 | -0 至 -5 | 采访、数据、设计、工程支持成本 |

```text
优先级 = 搜索需求 + 商业意图 + 难度/可赢性 + 集群价值 - 制作成本
```

## 分阶段建议

- Phase 1：pillar page + 高商业意图 + 可赢性高的 3-4 篇。
- Phase 2：补齐中层 supporting article，建立内链网络。
- Phase 3：长尾 FAQ、对比页、工具页和内容刷新。

## 蚕食检查

| 选题 A | 选题 B | 重叠查询 | 决策 |
| --- | --- | --- | --- |
| [标题] | [标题] | [query] | 合并 / 差异化 / canonical |

## 输出要求

- 排序表必须包含分数和一句理由。
- 先写哪些内容、为什么先写，要能解释给销售和编辑团队。
- 对重叠选题必须明确合并或差异化策略。
