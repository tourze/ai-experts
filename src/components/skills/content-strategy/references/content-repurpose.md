
# 内容多平台分发（content-repurpose）

## 适用场景
- 把已发布或已完稿的文章适配到 LinkedIn、Medium、Reddit、Quora 等平台。
- 通过多表面分发提升被 AI 搜索引擎引用的概率。
- 一篇文章 → 4-5 个平台版本，不是复制粘贴而是真正适配。

## 核心约束
- 每个平台版本必须真正适配该平台的语气、格式和用户期待，不是换个标题就发。
- 所有版本必须回链原文。
- Reddit/Quora 版本绝对不能有营销语言；以真人分享经验的口吻写。
- 先分析原文核心论点和最强数据点，再开始适配。
- 平台适配规则详见 [platform-adaptation-rules](./platform-adaptation-rules.md)。

## 代码模式

### 适配流程

1. **分析原文**：提取主关键词、核心论点、3-5 个要点、最强数据点和引言
2. **按平台生成**：每个平台一个独立版本（见 references）
3. **输出分发清单**：带平台、状态、回链的 checklist

### 平台优先级

| 平台 | 格式 | 语气 | 回链方式 |
|------|------|------|----------|
| LinkedIn | 800-1200 词长文 | 专业洞察 | "我写了更详细的分析：[链接]" |
| Medium | 1200-1800 词 | 接近原文 | "Originally published on [博客]" |
| Reddit | 100-250 词评论 ×2-3 | 完全口语，零营销 | 最多 1 个版本含链接 |
| Quora | 300-500 词 | 权威但亲切 | "更详细的拆解在这里：[链接]" |
| YouTube | 脚本大纲 150-300 词 | 口播/演示 | 描述栏放链接 |

## 检查清单
- 每个平台版本是否真正适配（不是复制粘贴）。
- LinkedIn 版本是否强调商业价值和洞察。
- Reddit 版本是否通过"真人会这么写吗"测试。
- Quora 版本是否第一句直接回答问题。
- 所有版本是否回链原文。
- 是否输出了带日期和状态的分发清单。

## 反模式

### FAIL: 换个标题发全平台

```
把博客文章原封不动贴到 LinkedIn 和 Medium
→ 没有平台适配，LinkedIn 读者不看 2000 词技术文
```

### PASS: 按平台重塑

```
原文：2500 词 B2B SaaS 内容策略指南
LinkedIn：800 词，聚焦"内容 ROI 的 3 个反直觉发现"
Reddit：150 词评论，分享一个具体数据点，像同行闲聊
```

### FAIL: Reddit 里做推广

```
"我们公司刚发布了一篇全面的行业领先指南..."
→ 秒删 + 被 downvote
```

### PASS: Reddit 里做人

```
"去年我们团队踩了这个坑，后来发现关键是 X。
有个不错的数据：Y 公司做 Z 后转化率涨了 34%。"
→ 如果有人追问再顺势分享链接
```
