## 适用场景
- 从零规划内容方向、内容支柱和季度选题。
- 判断“该写什么”而不是“这篇怎么改”。
- 需要把品牌叙事、搜索流量和销售线索整合成一套发布计划。

## 核心约束
- 先定义目标受众、业务目标和差异化，再排选题。
- 每个选题都要说明它服务搜索、分享、教育还是转化，不能只看“感觉能写”。
- 内容策略要和站点结构、分发渠道、复用方式配套考虑。
- 若任务转向具体页面优化，配合 [seo](../seo/SKILL.md)。

## 代码模式

### 选题矩阵

```md
内容支柱：
1. 问题教育  2. 方案比较  3. 成功案例

| 主题 | 目标读者 | 搜索意图/传播意图 | 主分发渠道 | CTA |
|---|---|---|---|---|
| CRM 迁移清单 | RevOps | 搜索 | Blog + SEO | 预约演示 |
```

### 内容日历

按 [content-calendar-methodology](references/content-calendar-methodology.md) 做月度发布计划：按周策略排优先级（第 1 周快速胜利、第 2 周混合、第 3 周聚类建权、第 4 周战略补全），输出带日期和执行命令的日历。

### 优先级排序

4 类内容源交叉去重后排序：
- **Quick Win**（排名 11-20，现有内容优化即可进首页）
- **Rewrite**（流量下滑或过时内容）
- **New Article**（关键词空白或聚类缺口）
- **Trending**（搜索量上升、时效性强）

- 若涉及 CMS 组织方式，可参考 [headless-cms](references/headless-cms.md)。
- 若做主题聚类，配合 [topic-cluster](references/topic-cluster.md)。

## 检查清单
- 是否明确业务目标、受众层级和内容支柱。
- 是否区分短期获客内容与长期品牌内容。
- 是否说明更新频率、负责角色和复用方式。
- 是否把内容与 CTA、产品页、站点结构关联起来。

## 域名方向发散

为新产品、栏目或品牌发散域名时，按"主品牌 / 活动页 / 实验性"分组输出候选，给出记忆点与拼写测试，避免与现有品牌近似。方法和模板见 [domain-name-brainstormer](references/domain-name-brainstormer.md)。

## 反模式

### FAIL: 只堆主题词

```
选题：AI 趋势、远程办公、团队协作、数据分析
→ 谁读？读完做什么？没说
```

### PASS: 主题 + 读者 + 意图 + CTA

```
| 主题 | 读者 | 意图 | CTA |
|------|------|------|-----|
| RevOps 季度复盘模板 | RevOps Lead | 搜索 | 下载（leads） |
| Stripe 集成对比 | 工程主管 | 决策 | 预约演示 |
```

### FAIL: 全 SEO 化

```
Q1 12 篇全是 long-tail SEO
→ 排名上来了，但客户不知道品牌是什么
```

### PASS: 三类平衡

```
40% 搜索（获客）：教程、对比页
40% 销售支持（转化）：案例、ROI 计算器
20% 品牌（建权威）：行业洞察、创始人观点
```

### FAIL: 无视发布能力

```
"每周 5 篇 + 月 1 白皮书 + 季 1 研究"
团队：1 个编辑，0 个设计 → 第 2 周就崩
```

### PASS: 按产能定节奏

```
1 人团队 → 每周 1 篇深度 + 每月 1 个轻量
扩团队后再升级到 2 篇/周
```
