# 战略钟模型

## 适用场景
- 定价策略选择：在价格和感知价值之间找到最佳组合。
- 竞争定位分析：把自己和竞品放到同一张图上比较。
- 与 [pricing-strategy](../pricing-strategy/SKILL.md) 配合：战略钟定方向，定价策略定细节。

## 核心约束
- 8 种位置由价格（低到高）和感知价值（低到高）两轴决定。
- **位置 6-8 是死区**（高价低值），除非有垄断保护，否则不可持续。
- 位置 3（混合策略：高品质 + 合理价格）最具竞争力，因为最难被模仿。
- 不要只定位，要明确**移动方向**——从哪到哪，需要什么能力。
- "最低价格 + 最高品质 + 最佳体验"不存在，战略就是取舍。

## 不适用场景
- B2B 企业级销售：感知价值难以简单量化，定价受合同谈判影响大。用 [pricing-strategy](../pricing-strategy/SKILL.md) 直接做定价分析。
- 垄断/寡头市场：战略钟假设竞争市场，垄断者不受价格-价值约束。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 准确定位了自己和主要竞品的位置。
- [ ] 明确了移动方向（从哪到哪）。
- [ ] 避开了位置 6-8 的死区。
- [ ] 移动所需的能力建设已识别。

## 反模式

### FAIL: 想占所有好位置

```
"我们要做到最低价格 + 最高品质 + 最佳体验"
→ 物理上不可能同时做到，战略就是取舍
```

### PASS: 明确取舍和路径

```
当前：位置 2（低价中值，靠补贴获客）
目标：位置 3（混合策略，提升产品价值同时保持价格竞争力）
路径：停止补贴 → 投入产品体验 → 用品质而非价格留客
所需能力：产品团队扩编、NPS 监控体系、品牌升级
```
