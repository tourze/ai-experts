## 代码模式

### 模式 1：按模块列轻量测试计划

```markdown
| 模块 | 目标 | 测试层级 | 关键场景 |
|------|------|----------|----------|
| 登录接口 | 鉴权正确 | 单测 + 集成 | 正常登录、密码错误、锁定、限流 |
| 支付页 | 用户流程可用 | 组件测试 + E2E | 成功支付、取消、重复提交 |
| 定时任务 | 数据一致性 | 单测 + 集成 | 幂等、失败重试、时间边界 |
```

### 模式 2：覆盖目标

```markdown
- 单测：业务规则、边界值、错误处理
- 集成：数据库、外部 API、消息队列、权限边界
- E2E：关键用户路径、跨页面状态、浏览器交互
```

### 模式 3：生产级风险矩阵

```markdown
| 区域 | 业务影响 | 技术风险 | 推荐测试深度 | 备注 |
|------|----------|----------|--------------|------|
| 支付链路 | 高 | 高 | 单测 + 集成 + E2E + 性能 | 必须全链路回归 |
| 后台报表 | 中 | 低 | 单测 + 集成 | 可接受人工抽样 |
| 营销页 | 低 | 低 | 冒烟 + 可访问性 | 视觉回归按需 |
```

### 模式 4：质量门

```markdown
## 质量门

- 开发完成：单测通过，新增关键分支具备失败路径覆盖
- 提测前：关键 API 集成测试全绿，P0 缺陷为 0
- 发布前：冒烟回归通过，性能回归未超过基线 10%
```
