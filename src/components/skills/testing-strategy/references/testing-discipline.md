# 通用测试纪律

从各语言测试 skill 中收敛的通用原则，适用于所有语言和框架。

## 核心原则

1. **一测一行为**：一个测试只验证一个行为；多个断言必须在同一语义下。
2. **Mock 边界**：Mock 只隔离外部依赖（数据库、API、文件系统），不 mock 被测逻辑本身。
3. **测行为合同**：测试公开接口的输入/输出约定，不测试内部实现细节。
4. **AAA 结构**：默认采用 Arrange/Act/Assert 组织测试体。

## 检查要点

- 测试名必须描述行为和预期结果，而非"testFunction"。
- 覆盖成功路径、失败路径和边界条件，不只测 happy path。
- 异步测试必须显式 await/return，禁止用 sleep 等待。
