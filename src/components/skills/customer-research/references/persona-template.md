# Persona 卡片模板

Persona 必须从研究数据构建，禁止凭空发明。

```md
## Persona: [名称]

**画像**：角色、公司规模、行业、职责
**核心 JTBD**：
  - 功能性：...
  - 情感性：...
  - 社会性：...
**触发事件**：什么促使 TA 开始寻找方案
**主要痛点**：（按频率×强度排序）
  1. ...
  2. ...
**期望结果**：TA 定义的成功
**常见反对意见**：
  - ...
**原话词汇**：（直接引用 3-5 条）
  - "..."
  - "..."
**数据来源**：访谈 N 条 / 评论 N 条 / 工单 N 条
```

## 交付物清单

| 交付物 | 内容 |
|---|---|
| 研究综合报告 | 方法、发现、主题聚类、置信度、建议 |
| VOC 语料库 | 按主题分组的原始引用 + 标签 + 来源 |
| Persona 文档 | 按上述卡片结构，每个 persona 一页 |
| JTBD 地图 | 功能 / 情感 / 社会三层，按优先级排序 |
| 竞品情报摘要 | 用户视角的竞品优劣势 + 迁移原因 |
| 研究缺口分析 | 哪些假设缺数据、下一步建议的研究动作 |
