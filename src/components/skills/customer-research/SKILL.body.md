# 客户研究（customer-research）

## 适用场景
- 有访谈记录、问卷、工单或 NPS 回复，需要系统提炼洞察。
- 从 Reddit、G2、Capterra、HN、App Store 等渠道挖掘用户声音。
- 基于真实数据构建 persona 或 JTBD 地图。

## 核心约束
- 结论必须有原始引用支撑，禁止凭空编造 persona。
- 每次明确当前模式：分析已有素材 vs. 在线挖掘。
- 所有洞察标注置信度（高/中/低）和样本量。
- 文案任务改用 `copywriting`；页面优化改用 `page-cro`；竞品分析配合 `competitive-teardown`。

## 模式一：分析已有素材

输入：访谈转写、问卷开放题、客服工单、赢/丢单笔记、NPS 文本。

按 [extraction-framework](references/extraction-framework.md) 逐条提取 JTBD（功能/情感/社会）、痛点、触发事件、期望结果、原话词汇、替代方案，然后主题聚类 → 频率×强度打分 → 按画像分段 → 提取金句 → 标记矛盾。

## 模式二：在线挖掘

目标渠道见 [extraction-framework](references/extraction-framework.md)。流程：
1. 确定关键词，优先最近 12 个月内容。
2. 逐条提取：角色、痛点、JTBD、原话、情绪极性。
3. 交叉验证同一观点是否跨渠道出现。
4. 汇入模式一的提取框架统一合成。

## 研究质量护栏

| 置信度 | 条件 |
|---|---|
| 高 | 5+ 独立来源，多渠道交叉验证 |
| 中 | 2-4 来源，或单渠道样本充足 |
| 低 | 单一来源或样本 < 5，待验证 |

- 时效窗口：默认 12 个月，超期标注为历史参考。
- 样本偏差：检查是否只覆盖单一用户类型。
- 最小样本：每细分群体 5+ 数据点，不足标"初步信号"。

## Persona 生成

按 [persona-template](references/persona-template.md) 构建，结构含：画像、JTBD、触发事件、痛点（频率×强度排序）、期望结果、反对意见、原话词汇、数据来源。交付物清单同见该文件。

## 反模式

### FAIL: 凭空编造 persona
```
"25 岁女性，喜欢瑜伽" → 无数据支撑
```

### PASS: 数据驱动
```
12 条访谈 + 47 条评论 → 痛点按频率×强度排序，附原话
```

### FAIL: 只看好评
```
只收正面反馈 → 忽略流失和差评
```

### PASS: 全谱覆盖
```
promoter / passive / detractor 分组 → 标记偏差和缺口
```
