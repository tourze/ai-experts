# 提案评审清单

在正式发送提案前的最终质量检查。

## 评审维度

### 结构完整性
- 问题陈述是否清晰定义了痛点和影响范围。
- 方案是否直接回应问题，而非泛泛描述能力。
- 执行计划是否有明确的时间线、负责人和交付物。

### 说服力
- 是否有数据或案例支撑核心主张。
- 差异化是否明确，与竞品/替代方案的区别是否清晰。
- 风险和应对措施是否诚实而非回避。

### 可读性
- 摘要能否在 1 分钟内传达核心价值。
- 段落长度是否适合扫读（不超过 5 行）。
- 专业术语首次出现时是否有解释。

## 常见问题

- 方案描述过多"我们能做什么"，过少"为什么这样做"。
- 预算/资源部分过于模糊，缺乏分项依据。
- 时间线过于乐观，没有缓冲和检查点。
