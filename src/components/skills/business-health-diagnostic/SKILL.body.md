## 检查清单
- [ ] 4 个象限都做了扫描，没有只看增长忽略效率。
- [ ] 每个指标有当前值、趋势和基准三要素。
- [ ] 红黄绿状态有明确阈值定义。
- [ ] 诊断结论收敛到 ≤3 个优先行动。

## 反模式

### FAIL: 只看收入

```
"ARR 增长 50%，业务很健康"
→ 忽略：Burn Multiple 4x，Runway 6 个月
→ 忽略：NRR 85%，老客户在流失
→ 增长在掩盖留存和效率问题
```

### PASS: 四象限全扫

```
增长 🟢：MRR +12% MoM
留存 🔴：NRR 88%，Logo Churn 7%/月
效率 🟡：CAC Payback 20 个月，略超基准
现金 🟡：Runway 14 个月，但 burn 在加速

→ 诊断：增长掩盖了留存危机
→ P0：修留存（D7 留存从 18% 提到 30%）
→ P1：降 CAC（砍低效渠道）
```

### FAIL: 列 20 个待改善项

```
"我们需要改善留存、优化 onboarding、降低 CAC、提升 NPS、
 改进客服响应、优化定价、增加功能、改善文档..."
→ 什么都想做 = 什么都做不到
```

### PASS: 聚焦 Top 3

```
根据影响量级和可行性排序：
1. 🔴 留存：激活率从 25% 提到 50%（影响 NRR 和 LTV 最大杠杆）
2. 🟡 效率：砍掉 ROI <1 的 2 个获客渠道（省 $3K/月）
3. 🟡 增长：上线 self-serve 升级（减少 sales 依赖）
```
