# 平衡计分卡（BSC）

## 适用场景
- 年度战略目标分解为可衡量的 KPI 体系。
- 避免只看财务指标而忽略客户、流程和成长维度。
- 与 [business-health-diagnostic](../SKILL.md) 配合：BSC 做系统设计，健康诊断做快速检查。

## 核心约束
- 四个维度有因果链：学习与成长 → 内部流程 → 客户 → 财务。**先投入后产出**，不能倒推。
- BSC 不是绩效考核工具，是**战略翻译工具**——从战略到行动的桥梁。
- 每个维度 3-5 个指标就够，不要做成 KPI 大全（总数控制在 12-20 个）。
- 指标必须有目标值和当前值，只有名称没有数字等于没设。
- **最常见的失败：BSC 变成 KPI 收集器**。堆到 50 个指标谁都不看 = 废纸。少即是多，每个维度 3-5 个聚焦指标。
- **学习与成长维度最容易被敷衍**。"加强培训""提升能力"不是指标。量化方法：人均培训小时数、关键岗位继任者覆盖率、员工 eNPS、内部晋升率。

## 不适用场景
- 早期创业（< 20 人）：BSC 假设组织有多层级和成熟流程，早期团队用 OKR 更轻量。
- 纯财务评估：BSC 的价值在于非财务维度，只看财务直接用财务报表分析。
- 临时项目管理：BSC 是战略翻译工具，不是项目跟踪工具。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 四个维度都有指标，没有偏科。
- [ ] 每个指标有当前值和目标值。
- [ ] 因果链条逻辑通顺（投入 → 产出）。
- [ ] 指标总数 12-20 个，不是 KPI 大全。

## 反模式

### FAIL: 只有财务指标

```
收入增长 20%、毛利率 60%、净利率 15%
→ 知道目标但不知道怎么达成
→ 缺失：客户怎么来？流程怎么改？团队怎么建？
```

### PASS: 四维联动

```
学习与成长：工程师人均培训 40h/年 → 新技术采纳率提升
内部流程：发布周期从 4 周缩短到 2 周 → 客户反馈响应加快
客户：NPS 从 32 提升到 45 → 续约率提升
财务：NRR 从 105% 提升到 115% → ARR 增长 30%
→ 每一层驱动下一层
```
