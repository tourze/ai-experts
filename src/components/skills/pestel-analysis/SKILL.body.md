## 代码模式

### PESTEL 分析模板

```markdown
| 维度 | 关键因素 | 影响方向 | 时间窗 | 量级 | 行动含义 |
|---|---|---|---|---|---|
| Political（政治） | | ↑利好/↓利空/↔不确定 | 短/中/长 | 高/中/低 | |
| Economic（经济） | | | | | |
| Social（社会） | | | | | |
| Technological（技术） | | | | | |
| Environmental（环境） | | | | | |
| Legal（法律） | | | | | |
```

### 各维度常见扫描点

```markdown
Political（政治）
- 政府产业政策、补贴方向
- 国际贸易关系、制裁和关税
- 政治稳定性、政权更迭风险
- 税收政策变化

Economic（经济）
- GDP 增速、通胀率、利率
- 汇率波动（跨境业务）
- 消费者信心指数、购买力
- 融资环境（VC 活跃度、估值倍数）

Social（社会）
- 人口结构变化（老龄化、Z 世代）
- 消费习惯和价值观迁移
- 远程办公/数字化生活趋势
- 隐私意识和数据伦理态度

Technological（技术）
- AI/LLM 能力边界变化
- 基础设施成熟度（5G、云计算）
- 开源生态和技术标准演进
- 技术人才供需

Environmental（环境）
- 碳中和/ESG 监管要求
- 数据中心能耗限制
- 供应链可持续性要求
- 极端天气对运营的影响

Legal（法律）
- 数据保护法规（GDPR、个保法）
- 反垄断和平台监管
- 知识产权保护变化
- 行业特定许可证要求
```

### 影响矩阵

```markdown
           低量级        中量级        高量级
短期    监测即可     制定应对方案    立即行动
中期    定期扫描     纳入路线图     战略调整
长期    保持关注     情景规划       重新评估商业模式
```

## 检查清单
- [ ] 六个维度都做了扫描，没有遗漏。
- [ ] 每个关键因素有影响方向、时间窗和量级判断。
- [ ] 从因素推导出了具体的行动含义或战略建议。
- [ ] 数据和趋势标注了来源和时效。

## 反模式

### FAIL: 罗列新闻

```
Political: "政府出了新的数据安全法"
Economic: "GDP 增速放缓"
Social: "年轻人喜欢短视频"
→ 列了 20 条新闻，没有一条说明对我们业务的具体影响
→ 等于没分析
```

### PASS: 因素 → 影响 → 行动

```
Legal: 个人信息保护法 2024 修订（来源：国务院法制办，2024.03）
- 影响方向：↓利空（短期合规成本）→ ↑利好（长期淘汰不合规竞品）
- 时间窗：短期（6 个月内必须合规）
- 量级：高（不合规可能导致产品下架）
- 行动：Q1 完成隐私合规审计，Q2 上线用户数据管理中心
  → 合规先行者可以把"隐私保护"变成差异化卖点
```

### FAIL: 只看利好不看利空

```
Technological: "AI 让我们开发效率提升 3 倍"
→ 忽略：AI 也让竞争对手效率提升 3 倍
→ 忽略：AI 可能让我们的核心功能被大模型替代
→ 单边分析导致盲目乐观
```

### PASS: 双面分析

```
Technological: LLM 能力边界快速扩展（GPT-5 级别）
- ↑利好：我们的 AI 功能体验可以显著提升，ARPA 有上涨空间
- ↓利空：竞品可以更快复制我们的 AI 特性，壁垒降低
- ↔不确定：大模型厂商可能直接进入我们的垂直领域
- 行动：加深数据壁垒（私有数据+行业 know-how），不依赖模型能力差异
```
