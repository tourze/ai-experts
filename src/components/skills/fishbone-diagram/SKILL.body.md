## 代码模式

本 skill 是思维框架，不要求执行代码。输出时先给结构化框架，再列证据、假设和追问，最后给可验证的下一步。

## 反模式

### FAIL: 只列第一层

```
销售额下降 <-
|- 产品不好
|- 价格太高
|- 竞争激烈
|- 市场不好
-> 第一层谁都能写，没有深入到根因
```

### PASS: 追问到根因

```
销售额下降 <-
|- 产品：功能不满足需求
|  -> 为什么？-> 需求调研只做了 3 家客户
|     -> 为什么？-> 产品团队没有客户访谈机制 <- 根因 1
|- 销售：成单周期从 30 天变 60 天
|  -> 为什么？-> 需要 VP 审批才能给折扣
|     -> 为什么？-> 授权机制未更新（公司规模翻倍后）<- 根因 2
-> 解决根因 1：建立月度客户访谈制度
-> 解决根因 2：下放 20% 以内折扣权限给一线
```
