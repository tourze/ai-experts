## Part 1：三态标注

每段结论显式标注 `[事实]` / `[推断]` / `[假设]`（或英文 `[FACT]` / `[INFER]` / `[ASSUME]`），不允许沉默。

| 态 | 定义 | 要求 |
|----|------|------|
| **事实** | 直接可观测、可被同类工具复现 | 附定位 |
| **推断** | 基于事实链的合理结论，未直接验证 | 给推断依据 |
| **假设** | 未观测、桥接逻辑空缺的命题 | 标"待验证" |

**硬约束：**
- 不允许跨态包装。假设写成"我们知道 X"、推断写成"事实是 X"、事实写成"我相信 X" 均属错误。
- 结论级别匹配最弱证据：1 事实 + 2 推断 + 1 假设链出的结论，整体只能标 `[假设]`。
- 关键假设必须可证伪：给出「会改变结论的反例」或「最小验证实验」。
- 升降级：假设找到直接观测 → 升为事实；支撑链断裂 → 降为假设。

## Part 2：发现-证据绑定

每条发现 = **定位 + 证据片段 + 触发条件**，三件套缺一不可。

**定位形式（按可核验度）：**
`文件:行号` > `log 时间戳 + trace_id` > `commit hash` > `metric 名+数值+时间窗` > `命令+退出码+输出`

**硬约束：**
- 证据片段必须粘贴实际内容，不能写「类似这样」。
- 触发条件可重现：分支、数据规模、角色、并发度、环境。
- 多源证据按可信度排序：`直接观测 > 间接推断 > 同类模式类比 > 直觉`。
- 性能 / 安全 / 并发类断言必须有可重现实验或观测窗口；纯代码扫描只能标 `[推断]`。

## 组合工作流

```
证据采集 → 三态标注 → 定位绑定 → 报告输出
```

1. 截取实际代码/diff/log/metric 片段
2. 每条标 `[事实]` / `[推断]` / `[假设]`
3. 事实附定位，推断给依据链，假设给验证路径
4. 综合结论级别匹配最弱环节

## 代码模式

```text
## 发现 N：<标题>

[事实] <观测> —— 定位：<file:line | metric | log>
[推断] <结论> —— 推断依据：<事实链>
[假设] <命题> —— 待验证：<最小实验>

- 影响：<范围与用户可见性>
- 修复方向：<具体改动>
```

```text
## 综合结论
[推断] <整体判断>，置信度受 <最弱环节> 约束。
```
