# 证据质量反模式

## 三态标注反模式

| 危险措辞 | 错在哪 | 替代写法 |
|---------|------|---------|
| "我们知道 X" | 把假设包装成事实 | `[假设] X —— 待验证：...` |
| "显然 X" / "毫无疑问 X" | 抹平三态 | 标具体一态并给依据 |
| "一定是 X 导致的" | 推断当事实 | `[推断] X 可能导致 —— 推断依据：...` |
| 多链综合后只给单一结论 | 隐藏最弱环节 | 综合结论标级匹配最弱环节 |

## 证据绑定反模式

| 危险措辞 | 现实后果 | 替代写法 |
|---------|---------|---------|
| "看起来 / 大概率 / 应该" | 读者无法判断证据强度 | 标 `[推断]` 并给出推断依据 |
| "整体感觉 X 有问题" | 无定位，无法验证 | 引用具体文件:行号 |
| "测试偶尔失败" | 不是触发条件 | 给频率（如 20 次失败 1 次）+ run_id |
| "性能瓶颈在 X" 但没跑过 profile | 推断当事实 | 给验证路径 + 标 `[假设]` |
