# 线索研究（lead-research-assistant）

## 适用场景
- 从产品定位倒推出最值得优先接触的公司类型、岗位和行业。
- 需要生成第一批外联名单，而不是只做宽泛市场分析。
- 需要把 ICP、痛点、触达理由和下一步动作串成一份可执行清单。
- 已经有候选公司或行业，需要进一步找联系人和 buyer intent。
- 需要整理公司背景、岗位分布、公开项目、招聘信息或技术栈信号。
- 需要把公开线索整成销售可直接使用的调研卡片。

## 核心约束
- 先确认产品、客单价、成交周期和理想客户，再开始筛名单。
- 输出必须包含筛选理由，不能只给一串公司名。
- 联系建议要基于公开信息与角色职责，不编造邮箱、手机号或内部数据。
- 只使用公开信息与合理推断，不伪造私人联系方式或内部状态。
- 联系人发现与公司研究要拆开汇报，避免把猜测写成事实。
- 明确标注事实、公开信号、推测结论、信息日期和置信度。

## 代码模式
### 模式 1：ICP 到首批账号

```md
目标画像：B2B SaaS / 50-500 人 / 已有销售团队
优先细分：客户成功、RevOps、市场运营

| 公司 | 触发信号 | 适配原因 | 建议联系人 | 首次切入角度 |
|---|---|---|---|---|
| Company A | 正在扩招销售运营 | 痛点与产品强相关 | Head of RevOps | 降低手工报表时间 |
```

### 模式 2：已有账号深挖

按 [账号调研卡片](references/account-research-guide.md) 结构产出：

```md
公司：Company A
阶段信号：近期招聘 RevOps / 正在扩张北美销售
关键岗位：VP Sales、Head of RevOps
切入点：报表自动化、线索归因、销售效率
风险：预算周期长、已有内部 BI 团队
```

## 检查清单
- 是否明确了行业、公司规模、区域和购买角色。
- 是否解释了每个目标为何进入名单。
- 是否给出了可执行的外联切入点而非空泛描述。
- 是否把“适合试单”和“适合长期跟进”的目标区分开。
- 是否区分事实、公开信号和推测结论。
- 是否给出可执行的联系人顺序与切入点。
- 是否说明信号的新鲜度和可信度。

## 反模式

### FAIL: 没 ICP 撒大网

```
“给我 100 家潜在客户”
→ 列了 SaaS / 电商 / 制造业 / 教育 / 政府...
→ 销售跟进 100 家 → 平均回复率 < 1%
```

### PASS: 窄 ICP

```md
ICP 锁定：
- 行业：B2B SaaS（中文区）
- 规模：50-300 人
- 阶段：A 轮后，正在扩销售团队
- 痛点：销售流程未标准化

输出 20 家精确匹配的公司
→ 跟进回复率 8-15%
```

### FAIL: 只给公司名

```
1. Company A
2. Company B
...
→ 销售：”我打过去说什么？”
```

### PASS: 公司 + 触发 + 切入

```md
| 公司 | 触发信号 | 痛点 | 切入角度 | 联系角色 |
| A | 近期招 5 个 RevOps（LinkedIn） | 销售流程混乱 | 报表自动化 | Head of RevOps |
| B | 刚完成 B 轮（公开信息） | 海外扩张 | 多币种计费 | CFO |
```

### FAIL: 只挖联系人

```
"VP Sales: alice@x.com / bob@x.com / charlie@x.com"
→ 销售："我该联系谁？为什么是 VP Sales？"
```

### PASS: 角色 + 决策 map

```md
## Company A
- 决策者：CEO（最终签合同）
- 影响者：VP Sales（提需求）
- 用户：Sales Ops Lead（日常使用）
- 守门人：CFO（预算审批）

推荐顺序：
1. 先连 Sales Ops Lead（用户层，最易约见）
2. 通过 ta 引荐 VP Sales（影响者）
3. VP Sales → CEO（决策）
```
