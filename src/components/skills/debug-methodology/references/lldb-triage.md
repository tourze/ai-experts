# 线程栈快速判型

先用这张速查表给卡死现象分类，再决定是否继续深挖。

## 死锁 / 锁反转
- 主线程或 UI 线程在等 mutex / lock / condition variable。
- 后台线程握着这把锁，同时又反过来等待主线程（例如同步 IPC、事件回调或 UI eval）。
- 现象：多个线程同时停住；重复抓栈时帧基本不变。

## 忙循环 / 高 CPU
- 同一线程反复出现紧循环热点帧，没有明显阻塞系统调用。
- 多次采样会快速变化，或者总落在同一组热点函数里。

## 阻塞 I/O / IPC
- 线程停在 `read` / `recv` / `poll` / `select` / `epoll` / `kqueue` 一类等待上。
- 另一个线程可能正在等一个永远不会回来的响应或 callback。

## 主线程重入
- UI 线程栈里出现嵌套事件循环或 re-entrant dispatch。
- 常见原因是后台线程同步调用 UI，导致消息循环在错误层级重入。

- 对比主线程和 worker 线程，确认谁在持锁、谁在等待。
- 把线程栈和卡死时间窗口内的应用日志对齐，锁定第一个异常副作用。
