# 页面类型指南

## Homepage
- 面向多个 persona，headline 取最大公约数价值。
- 用 section 分流不同受众（按角色 / 用例 / 行业）。
- 导航清晰，不要在首页塞完所有信息。

## Landing Page
- 单一 CTA，删除导航栏和页脚链接。
- Headline 与广告/邮件中的承诺一致（message match）。
- 每个 section 都在推动同一个转化动作。

## Pricing Page
- 推荐档位视觉突出（highlight + "Most popular" badge）。
- 功能对比表用 ✓/✗，不要用长文字。
- 在价格旁标注价值锚点（"相当于每天 $2"）。
- FAQ 回答退款、升级、合同期等常见疑虑。

## Feature Page
- 每个功能用 "Benefit headline + supporting detail" 结构。
- 配截图或动图，避免纯文字堆砌。
- 底部 CTA 呼应该功能解决的核心痛点。

## About Page
- 讲创始故事时聚焦"为什么做这件事"，而非公司年表。
- 放团队照片和真实数据（客户数、团队规模、融资里程碑）。
- 用价值观段落替代使命宣言模板。

## 产出模板

```md
## 页面类型
[Homepage / Landing Page / Pricing / Feature / About / Product]

## 目标受众
[Persona 描述 + 当前痛点]

## 核心价值主张
[一句话]

## 文案结构

### Hero Section
- Headline:
- Subheadline:
- CTA:
- Qualifier:

### Section 1: [名称]
- Section headline:
- Body copy:
- Supporting evidence:

### Section 2: [名称]
...

### Final CTA
- Headline:
- CTA:
- Qualifier:

## Voice & Tone
- Formality: [casual / neutral / formal]
- Personality: [e.g. friendly-expert, bold-challenger, calm-authority]
```
