# 粘性表达（made-to-stick）

## 适用场景
- 价值主张、标语、首屏标题或核心信息不够好记。
- 团队讲得很完整，但用户听完记不住重点。
- 需要把复杂产品信息压缩成一个更能传播的叙事骨架。

## 核心约束
- 先确定用户最终只需要记住的一句话，再考虑细节扩展。
- SUCCESs 六项不是逐条打卡，而是围绕核心信息做取舍。
- 把“可信”“具体”“情绪”建立在真实证据上，不能靠夸张修辞硬堆。
- 若任务进入逐句编辑或要去除机器腔，配合 [copy-editing](copy-editing.md) reference。

## 代码模式
- 结构化改写模板：

```md
核心句：我们帮销售团队少做报表，多做成交
Simple：一句话说清价值
Unexpected：点出行业里反直觉的事实
Concrete：给出“每周省 6 小时”之类的量化证据
Credible：案例或数据来源
Emotional / Stories：用真实场景收尾
```

- 参考资料： [simple](references/simple.md)、[unexpected](references/unexpected.md)、[concrete](references/concrete.md)、[credible](references/credible.md)、[emotional](references/emotional.md)、[stories](references/stories.md)。

## 检查清单
- 是否明确了一句核心信息。
- 是否至少补足了“具体性”和“可信度”两项。
- 是否去掉了多余枝节，让读者能在 5 秒内复述重点。
- 是否给出记忆点而不牺牲准确性。

## 反模式

### FAIL: 抽象形容词堆砌

```
“我们是领先的、创新的、智能的企业解决方案，
致力于为客户提供卓越的价值和无与伦比的体验”
→ 读完记不住任何一条
```

### PASS: 具体 + 反直觉 + 可信

```
“销售每周省 6 小时报表时间（不是少写，是不写）
→ 全部自动从 CRM 拉，上周 127 家客户对比图”
具体：6 小时、127 家
反直觉：不是少写，是不写
可信：真实数据
```

### FAIL: 信息点太多

```
一段 slogan 塞 8 个特性：
“AI 驱动、云原生、跨平台、实时协作、企业安全、全球部署、24/7、免费试用”
→ 用户一个也记不住
```

### PASS: 压到一句核心信息

```
核心句：”让销售少做报表，多做成交”
其他卖点：放功能页而不是首屏
→ 5 秒内用户能复述
```
