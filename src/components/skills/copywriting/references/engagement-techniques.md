# Engagement Techniques

## Opening Hooks

| Hook | Use When |
| --- | --- |
| Problem-first | Reader already feels the pain. |
| Outcome-first | Reader wants a clear payoff. |
| Contrarian | Market belief is wrong or incomplete. |
| Specific proof | You have a strong metric or result. |
| Mini-story | Trust depends on context and sequence. |

## APP Intro

```text
Agree: name the reader's current reality.
Promise: state the concrete improvement.
Preview: show what the page will prove or explain.
```

## CTA Distribution

- First CTA after the value promise is understood.
- Secondary CTA only when it reduces risk, such as "see demo" or "read case study".
- Repeat CTA after proof, objection handling, and final summary.
- Button text should say what happens next.
