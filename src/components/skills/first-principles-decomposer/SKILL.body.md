## 反模式

### FAIL: 行业标准当事实

```
问题：如何降低电动车成本？
“事实”：电池占成本 40%（行业平均）
→ 没拆到物理层，结论和现状一样
```

### PASS: 拆到不可再压缩的物理层

```
问题：如何降低电动车成本？
挑战：电芯成本 = 报价 → 不是事实，是别人的定价
真实事实：
- 锂/镍/钴原材料市场价 × 用量
- 制造能耗 + 良率
- 电芯 BOM 构成
→ 发现：电芯成本 = 原材料 80% + 加工 20%
→ 结论：自建矿能降 50%（特斯拉的做法）
```

### FAIL: 替代症状不解决根需求

```
用户：”我要更大的锤子”
方案：造更大的锤子
→ 真实需求可能是”把画挂上墙”，该换钻头或膨胀螺丝
```

### PASS: 追问到底层需求

```
“要更大的锤子” → 为什么？
“钉不进去” → 钉什么？
“硬木板” → 为什么要钉？
“要挂画” → 真实需求是挂画
方案：膨胀螺丝 / 无痕挂钩 / 钻孔
```
