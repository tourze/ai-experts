# 竞品广告拆解（competitive-ads-extractor）

## 适用场景
- 收集 Facebook Ad Library、LinkedIn 等平台上的竞品广告。
- 归纳竞品在用的痛点、承诺、CTA、视觉套路和受众假设。
- 为自己的广告测试列表提供灵感和反向约束。

## 核心约束
- 先定义竞品范围、平台范围和时间范围，不要把所有广告混成一锅。
- 重点提炼“他们在卖什么判断”，而不是机械罗列文案截图。
- 竞品研究结论要能回流到 [ad-creative](./ad-creative.md) 或 competitor-alternatives 的输出中。
- 不杜撰投放效果；没有公开数据时只说“采用了什么表达/结构”，不说“因此一定高转化”。

## 代码模式
- 推荐输出模板：

```md
竞品：Company A
平台：Facebook Ad Library

| 主题 | 证据 | 可能目标受众 | 可借鉴点 | 需要避免 |
|---|---|---|---|---|
| ROI 提升 | 标题反复强调 save time | 绩效导向团队 | 量化收益表达 | 空泛的“更高效” |
```

## 检查清单
- 是否按平台、受众、主题对广告做了分组。
- 是否区分“常见套路”和“少数但值得测试的差异化打法”。
- 是否把结论落到可执行的测试假设。
- 是否标注了无法确认的数据和推断边界。

## 反模式

### FAIL: 截图无结论

```
[竞品 A 50 张广告截图]
[竞品 B 40 张广告截图]
"以上是竞品广告库"
→ 创意团队："所以我该做什么？"
```

### PASS: 提取主题 + 假设

```md
## 竞品 A 广告主题分布
- 70% 强调 "save time"（绩效经理痛点）
- 20% 强调 "team collaboration"
- 10% 客户案例

## 我们的差异化假设
- A 没碰：合规角度（GDPR/SOC2）
- 测试创意：合规作为购买决策因素
```

### FAIL: 跟风同质化

```
"5 家竞品都在用同一个 hook → 我们也用"
→ 用户广告流里看到 6 家说一样的话 → 全部刷过
```

### PASS: 反向定位

```
观察：100% 竞品都强调"AI 驱动"
机会：跟"非 AI / 透明算法"打反差
→ 做 1 组反向测试，CTR 可能 +30%
```
