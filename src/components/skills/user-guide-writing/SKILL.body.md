## 代码模式

```json
{
  "目标读者": "首次使用的运营同学",
  "文档类型": "新手上手指南",
  "关键任务": ["登录", "创建项目", "导出报表"],
  "配图策略": "仅对关键分叉步骤截图"
}
```

建议的章节顺序：

```markdown
## 开始前准备
## 第一步
## 第二步
## 常见问题
## 失败时怎么办
```

## 检查清单

- 是否明确了用户画像、前置条件和最终目标。
- 是否每个步骤都可执行，且有成功标准或结果说明。
- 是否把异常情况、权限限制和常见错误单独列出。
- 是否避免使用内部术语、缩写和实现细节。

## 反模式

### FAIL: 内部术语 + 无操作

```markdown
# 导出功能
本功能基于异步 ETL pipeline 实现，通过 Kafka 将数据从
ClickHouse 抽取到 S3 bucket...
→ 终端用户不知道 Kafka、ETL，也不知道点哪里
```

### PASS: 任务导向 + 可执行

```markdown
# 如何导出数据

## 开始前准备
- 管理员权限（如没有，联系 admin@company.com）
- 导出记录 ≤ 10000 条

## 步骤
1. “设置 → 数据管理”
2. 点击右上角 “导出”
3. 选时间范围 + 字段
4. 完成后收到邮件（15 分钟有效）

## 常见问题
没收到邮件？先查垃圾箱 → 看 “导出历史”
```

### FAIL: 省略前置条件

```markdown
点击 “导出” → 选择格式 → 下载
→ 用户点完报错”没有权限”，指南没说过
```

### PASS: 前置条件前置

```markdown
前置条件：
✓ 已实名认证
✓ 角色为管理员或分析师
✓ 当日导出 < 5 次

不满足 → 去 “设置 → 权限” 申请
```
