## 代码模式

### 10 段式 PRD 结构

```markdown
1. 背景与问题        ← 为什么要做（现状痛点、数据证据）
2. 目标与成功指标    ← 做到什么程度算成功（可量化的 KR）
3. 用户与场景        ← 给谁用、什么场景下用（Persona + Job Story）
4. 方案概述          ← 怎么解决（核心流程、关键交互，不写实现细节）
5. 详细需求          ← 功能点列表 + 每条的验收标准（Given/When/Then）
6. 非目标与排除项    ← 明确不做什么，防止 scope creep
7. 风险与依赖        ← 技术风险、外部依赖、合规约束
8. 发布与上线计划    ← 灰度策略、AB 测试、回滚预案
9. 数据与监控        ← 埋点需求、核心看板、告警阈值
10. 附录             ← 竞品参考、用户调研摘要、技术方案链接
```

最小完整集（轻量 PRD）：1-5 段。中型项目加 6-8 段。完整 PRD 用全 10 段。
