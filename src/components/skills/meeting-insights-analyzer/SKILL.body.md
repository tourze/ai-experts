```bash
rg --files . -g '*.md' -g '*.txt' -g '*.srt' -g '*.vtt' -g '*.docx'
```

```markdown
## 发现 1：回避直接反馈
- 观察：2026-04-10 产品周会 00:14:32，你说“也许我们可以再看看时间线？”
- 影响：问题被弱化，对方可能听不到紧迫性。
- 建议：改成“当前时间线已经延误两周，今天需要重排计划。”

## 指标
- 发言占比：42%（基于带说话人标签的 5 场会议）
- 打断：3 次主动打断 / 1 次被打断
- 口头禅：`其实` 8 次，`我觉得` 11 次
```

## 反模式

### FAIL: 无映射伪精确

```
“你发言占 63%，打断了 4 次”
→ 实际：转写没有说话人标签，你是猜的
→ 用户看到具体数字以为真实 → 误导
```

### PASS: 先声明边界

```
“本次分析：5 场会议转写，其中 3 场有说话人标签
可计算：3 场的发言占比 / 打断次数
不可计算：2 场语气感知（音频缺失）
样本小，结论仅作方向参考”
```

### FAIL: 空泛鸡汤建议

```
“你需要更直接”
“你应该多倾听”
→ 用户：”具体哪句话不够直接？我该改成什么？”
```

### PASS: 证据 + 替代表达

```
原句（2026-04-10 00:14:32）：”也许我们可以再看看时间线？”
问题：软化语气让紧迫性丢失
替代：”时间线已延误 2 周，今天需要重排”
```
