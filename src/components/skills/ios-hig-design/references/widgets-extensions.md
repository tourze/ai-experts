# iOS 小组件与应用扩展

小组件、App Clips 和系统扩展的设计指南。

## 小组件设计

### 小组件理念

小组件在主屏幕、锁定屏幕和 StandBy 模式上提供**一目了然的信息**。它们不是迷你应用——它们是进入应用最有用的内容的窗口。

**关键原则：**
- 显示立即可用的信息
- 有思考地更新内容（不是持续更新）
- 尊重用户的主屏幕美学
- 引导用户进入应用以进行更深度的交互

### 小组件尺寸

**主屏幕小组件：**

| 尺寸 | 名称 | 网格单位 | 使用场景 |
|------|------|---------|---------|
| 小型 | `systemSmall` | 2×2 | 单一信息 |
| 中型 | `systemMedium` | 4×2 | 关键内容 + 一次交互 |
| 大型 | `systemLarge` | 4×4 | 丰富内容、多个项目 |
| 超大 | `systemExtraLarge` | 8×4 | 仅 iPad，仪表板视图 |

**锁定屏幕小组件（iOS 16+）：**

| 尺寸 | 名称 | 特点 |
|------|------|------|
| 圆形 | `accessoryCircular` | 小图标或量表 |
| 矩形 | `accessoryRectangular` | 文本 + 小型视觉 |
| 内联 | `accessoryInline` | 纯文本，时间上方 |

### 小组件内容指南

**应做：**
- 展示最重要的信息
- 在有意义的间隔更新内容
- 使用应用的视觉风格
- 支持多种尺寸（让用户选择）
- 如果有不同的使用场景，提供多个小组件类型

**不应做：**
- 塞入太多信息
- 显示过时数据
- 使用小组件做广告
- 需要交互才能看到内容
- 更新过于频繁（消耗电量）

### 小型小组件设计

```
┌─────────────────────┐
│                     │
│    [图标/图片]      │
│                     │
│    主要信息         │
│    次要信息         │
│                     │
└─────────────────────┘
```

**指南：**
- 一个点击目标（整个小组件）
- 仅包含必要信息
- 清晰的视觉层次
- 无按钮或复杂交互

### 中型小组件设计

```
┌─────────────────────────────────────────┐
│  [图标]                                 │
│  标题               ┌─────────────┐    │
│  副标题             │   操作      │    │
│                      └─────────────┘    │
│  额外上下文                             │
└─────────────────────────────────────────┘
```

**指南：**
- 可以有多个点击目标
- 显示 2-4 条信息
- 操作应快捷（打开到特定视图）

### 大型小组件设计

```
┌─────────────────────────────────────────┐
│  标题                            编辑    │
├─────────────────────────────────────────┤
│  ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ 项目 1  │ │ 项目 2  │ │ 项目 3  │   │
│  └─────────┘ └─────────┘ └─────────┘   │
│                                         │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ 项目 4  │ │ 项目 5  │ │ 项目 6  │   │
│  └─────────┘ └─────────┘ └─────────┘   │
└─────────────────────────────────────────┘
```

**指南：**
- 允许多个点击目标
- 显示一个集合或仪表盘
- 包含清晰的视觉分组
- 可选：编辑配置

### 锁定屏幕小组件设计

锁定屏幕小组件空间有限且没有颜色。

**圆形：**
```
┌─────┐
│ 73° │  温度
│ ☀️  │  天气图标
└─────┘
```

**矩形：**
```
┌─────────────────────┐
│ 下一个事件           │
│ 团队会议 @ 下午 2 点 │
└─────────────────────┘
```

**最佳实践：**
- 为小尺寸设计
- 使用 SF Symbols（渲染效果好）
- 在浅色和深色模式下测试
- 考虑 StandBy 模式（更大的显示）

## 小组件配置

### 用户可配置的小组件

允许用户自定义小组件显示的内容：

```swift
struct ConfigurationIntent: WidgetConfigurationIntent {
    static var title: LocalizedStringResource = "Configuration"

    @Parameter(title: "City")
    var city: City?

    @Parameter(title: "Units")
    var units: TemperatureUnit
}
```

**配置 UI：**
- 保持选项简单（少参数）
- 提供合理的默认值
- 在确认前预览更改

### 小组件家族

支持多种尺寸：

```swift
struct MyWidget: Widget {
    var body: some WidgetConfiguration {
        StaticConfiguration(kind: "MyWidget", provider: Provider()) { entry in
            MyWidgetView(entry: entry)
        }
        .supportedFamilies([.systemSmall, .systemMedium, .systemLarge])
    }
}
```

## App Clips

### App Clips 是什么

App Clips 是应用的轻量版本（<10MB），用于快速、聚焦的任务，无需完整安装。

**启动点：**
- NFC 标签
- 二维码
- App Clip 码
- Safari 智能应用横幅
- 地图
- 信息

### App Clip 设计原则

**1. 专注于一个任务**
- 租用自行车
- 订餐
- 支付停车费

**2. 最小化所需信息**
- 只询问必要信息
- 使用 Sign in with Apple
- 使用 Apple Pay

**3. 快速体验**
- 用户期望在一分钟内完成
- 没有冗长的入门引导
- 最小化 UI，最大化功能

**4. 鼓励下载完整应用**
- 展示完整应用的价值
- 使下载变简单（横幅）
- 不要阻止功能以强制下载

### App Clip UI 指南

```
┌─────────────────────────────────────────┐
│  [头部：你可以做什么]                    │
├─────────────────────────────────────────┤
│                                         │
│  [主要操作 UI]                          │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │        [Apple Pay]              │    │
│  └─────────────────────────────────┘    │
│                                         │
├─────────────────────────────────────────┤
│  获取完整应用以获得更多功能              │
│  ┌─────────────────────────────────┐    │
│  │        下载应用                 │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

### App Clip 码设计

App Clip 码是可扫描的码，用于启动 App Clips：

```
       ┌─────────────────┐
      ╱                   ╲
     │   [App Clip 码]     │
     │   圆形图案          │
     │   带 NFC 芯片       │
      ╲                   ╱
       └─────────────────┘
         扫描或点击以
         租用滑板车
```

**放置指南：**
- 码下方放置清晰的操作指引
- 解释将会发生什么
- 可触及的高度（3.5-5 英尺）
- 光线充足、清洁的表面

## Live Activities

### Live Activities 是什么

在锁定屏幕和灵动岛上实时更新进行中的事件：
- 体育比分
- 配送追踪
- 计时器
- 共享出行

### Live Activity 设计

**锁定屏幕（展开）：**
```
┌─────────────────────────────────────────┐
│  [左侧]     [中间]         [右侧]       │
│  主队          vs          客队         │
│  24          Q3             21          │
└─────────────────────────────────────────┘
```

**灵动岛（紧凑）：**
```
┌──────────────────────────────────────┐
│  🏀  24 - 21  Q3                     │
└──────────────────────────────────────┘
```

**灵动岛（展开）：**
```
┌────────────────────────────────────────┐
│  湖人队          vs          凯尔特人  │
│    24                          21      │
│  ────────────────────────────────────  │
│      第三节  •  剩余 4:32              │
└────────────────────────────────────────┘
```

### Live Activity 指南

**应做：**
- 仅在发生有意义的变化时更新
- 为所有灵动岛状态进行设计
- 提供明确的结束状态
- 遵守 8 小时最长持续时间

**不应做：**
- 每秒更新（除非是计时器）
- 显示静态内容
- 用于通知
- 需要交互才能查看状态

## 小组件开发技巧

### 时间线更新

```swift
func getTimeline(in context: Context, completion: @escaping (Timeline<Entry>) -> ()) {
    let entries = [
        SimpleEntry(date: Date(), data: currentData),
        SimpleEntry(date: Date().addingTimeInterval(60*15), data: futureData)
    ]

    let timeline = Timeline(entries: entries, policy: .atEnd)
    completion(timeline)
}
```

**更新策略：**
- `.atEnd` - 所有条目显示后更新
- `.after(date)` - 在特定时间更新
- `.never` - 仅在用户操作时更新

### 深度链接

小组件应链接到特定内容：

```swift
Link(destination: URL(string: "myapp://item/\(item.id)")!) {
    ItemView(item: item)
}
```

### 占位符设计

在加载时显示有意义的占位符：

```swift
struct PlaceholderView: View {
    var body: some View {
        VStack {
            RoundedRectangle(cornerRadius: 8)
                .fill(Color.gray.opacity(0.3))
            RoundedRectangle(cornerRadius: 4)
                .fill(Color.gray.opacity(0.2))
        }
    }
}
```
