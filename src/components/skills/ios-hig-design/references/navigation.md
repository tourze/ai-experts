# iOS 导航模式

## 标签栏（主导航）

标签栏提供对主要应用目标的访问。

- **位置**：屏幕底部，始终可见（模态框/键盘除外）
- **项目**：最多 2-5 个标签页
- **溢出**：如需超过 5 个目标，使用"更多"标签页
- **选中状态**：填充颜色表示活动标签页
- **标签**：10-11pt SF 文本
- **背景**：略微半透明带背景模糊（"磨砂玻璃"）

**行为**：
- 每个标签页记住其导航状态
- 点击活动标签页返回该标签页的根界面
- 在模态框和键盘显示时隐藏标签栏

## 导航栏（上下文导航）

- **返回按钮**：左上角，允许返回到上一个界面
- **操作**：右上角，上下文相关操作
- **标题**：居中（滚动状态）或左对齐大标题（未滚动）

**滚动行为**：
- 大标题在滚动时折叠为紧凑的居中标题
- 搜索栏可以在滚动时移动或隐藏
- 状态之间的平滑动画过渡

## 导航返回

| 方式 | 上下文 |
|------|--------|
| "返回"按钮（左上角） | 标准导航 |
| 从左边缘右滑 | 标准导航 |
| "取消"/"完成"按钮 | 模态视图 |
| 在内容上向下滑动 | 模态框、全屏媒体 |

## 模态 Sheet

使用模态框处理不应该完全打断上下文的聚焦任务。

- 从底部滑入
- 上一个界面在背景中可见（凹入）
- 可通过关闭按钮、向下滑动或完成任务关闭

## 搜索 UI 模式

### 搜索栏位置

| 上下文 | 位置 |
|--------|------|
| 主要搜索（核心功能） | 导航栏，始终显示 |
| 次要搜索 | 导航栏下方，滚动时隐藏 |
| 列表筛选 | 列表上方，内联 |

### 搜索行为

**状态：**
1. **非活动：** 占位符文本，放大镜图标
2. **活动/聚焦：** 键盘出现，取消按钮显示
3. **输入中：** 结果更新（即时或防抖）
4. **结果：** 在下方列表中显示

**SwiftUI 实现：**
```swift
.searchable(
    text: $searchText,
    placement: .navigationBarDrawer(displayMode: .always),
    prompt: "Search items"
)
```

### 搜索建议

- 最近搜索
- 热门/趋势搜索
- 自动完成建议
- 范围建议（按类别筛选）

```swift
.searchable(text: $searchText) {
    ForEach(suggestions) { suggestion in
        Text(suggestion.name)
            .searchCompletion(suggestion.name)
    }
}
```

## 分割视图导航（iPad）

### 两列布局

```
┌──────────────────┬────────────────────────────────┐
│                  │                                │
│  主列表          │     详情视图                   │
│  （侧边栏）      │                                │
│                  │                                │
│  Item 1          │     选中项目详情               │
│  Item 2 ←        │                                │
│  Item 3          │                                │
│                  │                                │
└──────────────────┴────────────────────────────────┘
```

**行为：**
- 主列：默认宽度 320pt
- 详情列：填充剩余空间
- 在紧凑宽度下折叠为单列

### 三列布局

```
┌────────────┬────────────┬──────────────────────────┐
│  侧边栏    │ 内容       │    详情                 │
│            │            │                          │
│  Section 1 │ Item A     │     此处显示项目详情     │
│  Section 2 │ Item B ←   │                          │
│  Section 3 │ Item C     │                          │
└────────────┴────────────┴──────────────────────────┘
```

**SwiftUI：**
```swift
NavigationSplitView {
    Sidebar()
} content: {
    ContentList()
} detail: {
    DetailView()
}
```

### 响应式行为

| 宽度 | 行为 |
|------|------|
| 紧凑（iPhone） | 堆栈导航 |
| 常规（iPad 竖屏） | 两列或覆盖 |
| 常规（iPad 横屏） | 三列可用 |

**最佳实践：**
- 未选择任何内容时在详情中显示占位符
- 在旋转后记住选择
- 支持列大小调整（在适当时）
