## 代码模式

```text
1. 当前问题
- 你真正想解释/发现什么？
- 已有证据和约束是什么？

2. 发散方向
- 跨学科类比：
- 反转假设：
- 尺度切换：
- 新技术切入：

3. 连接与收敛
- 哪几个方向共享同一底层机制？
- 哪个方向最值得先做小实验？

4. 下一步
- 文献补洞：
- 最小实验：
- 需要补的资源/合作：
```

## 反模式

### FAIL: 单向授课

```
用户："想做老年人步态研究"
AI：开始讲 30 分钟生物力学综述...
→ 用户全程接收信息，没参与生成
```

### PASS: 协作发散

```
AI："步态的哪个面更激发你？力学、神经控制、还是社会行为？"
用户："神经控制"
AI："那我们先列 3 个反转假设：
  - 如果跌倒不是肌力问题而是预测误差呢？
  - 如果同侧步态不对称是适应性而非衰退呢？
  - ..."
→ 用户每轮做选择，方向他自己定
```

### FAIL: 脑洞当结论

```
"VR 步态训练能延缓老年痴呆"
→ 用户写进基金申请
→ 评审："证据呢？" → 撤回
```

### PASS: 创意 vs 可验证分开

```
✨ 创意（未验证）：
  - VR 步态训练 → 海马激活假说
🔬 可验证下一步：
  - 文献：是否有 VR + fNIRS 现有数据
  - 最小实验：N=10 受试者，pre/post 海马 fMRI
  - 资源缺口：需要 fMRI 时间 + 神经科合作
```
