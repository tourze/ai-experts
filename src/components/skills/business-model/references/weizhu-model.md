
# 魏朱六要素商业模式

## 适用场景
- 中国市场商业模式设计和诊断（比 BMC 更强调因果逻辑）。
- 与 [business-model](../SKILL.md) (BMC) 配合：BMC 做清单式扫描，魏朱做逻辑链推演。

## 核心约束
- 六要素有因果顺序：定位 → 业务系统 → 关键资源能力 → 盈利模式 → 现金流结构 → 企业价值。前后必须逻辑通顺。
- 与 BMC 的关键区别：BMC 像静态清单，魏朱像动态系统图——强调"为什么这样赚钱"的因果链。
- 定位不是口号，要回答"给谁 x 解决什么 x 怎么解决"。
- 盈利模式必须说清"谁付钱、怎么付、为什么愿意付"。
- 现金流结构要区分账面利润和实际现金——先收钱后服务 vs 先垫资后回款是完全不同的生意。
- 企业价值 = 规模 x 效率 x 持续性，三者缺一不可。

## 不适用场景
- 非中国市场语境：魏朱模型源自中国商业研究，国际市场更熟悉 BMC，沟通时用 [business-model](../SKILL.md)。
- 快速头脑风暴：魏朱强调因果链推演，需要深度思考；快速画布用 BMC 更高效。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 六个要素都有实质内容。
- [ ] 因果链条逻辑通顺（每个箭头都站得住脚）。
- [ ] 盈利模式说清了"谁付钱、怎么付、为什么愿意付"。
- [ ] 现金流结构区分了账面利润和实际现金。

## 反模式

### FAIL: 定位是口号

```
定位："做最好的企业服务平台"
→ 没说给谁、解决什么、怎么做，后续五个要素无法推导
```

### PASS: 因果链完整

```
定位：为年营收 1-5 亿制造业企业，通过 SaaS+顾问解决供应链可视化
业务系统：SaaS 自研 + 顾问驻场 + 供应商数据对接外包
关键资源：100+ 制造业供应链数据模型（3 年积累，不可复制）
盈利模式：SaaS 年费 30 万 + 顾问按天收费
现金流：先收年费后服务（正现金流循环）
→ 每一步都由上一步推导
```
