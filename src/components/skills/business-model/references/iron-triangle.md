
# 业务铁三角

## 适用场景
- 快速检验业务定位是否自洽：产品、职能和市场三者是否相互支撑。
- 新业务立项前的定位校验。
- 比较不同商业模式："同一个产品 + 不同职能/市场 = 完全不同的生意"。

## 核心约束
- 三个顶点：产品（解决什么问题）、核心业务职能（研发型/制造型/渠道型/服务型/平台型）、市场（目标客群和地理范围）。
- 核心是检查三者的**一致性**——不一致就是定位需要调整的地方。
- 三组交叉检查都必须做：产品 x 职能、产品 x 市场、职能 x 市场。

## 不适用场景
- 深度商业模式分析：铁三角是快速一致性检查，完整商业模式分析用 [business-model](../SKILL.md) 或 weizhus-six-elements。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 三个顶点都有明确描述。
- [ ] 做了三组一致性交叉检查。
- [ ] 不一致处给出了调整方向。

## 反模式

### FAIL: 三个顶点互相矛盾

```
产品：高端定制化 ERP
职能：渠道型（不做研发，只做转售）
市场：中小企业
→ 高端定制 x 渠道转售 x 中小企业 = 三者全部矛盾
```

### PASS: 三角自洽

```
产品：轻量级项目管理工具（解决协作效率问题）
职能：研发型（自研产品，PLG 分发）
市场：50-200 人互联网团队
→ 轻量工具 x 自研 PLG x 中小互联网团队 = 三角一致
```
