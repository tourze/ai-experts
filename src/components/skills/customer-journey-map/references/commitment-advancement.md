# Commitment 与 Advancement 信号

## 适用场景

- 需要判断用户是否真的有需求，而不是只表达兴趣。
- B2B 访谈后需要决定是否继续跟进。
- 创业想法验证需要区分弱信号和强信号。

## 信号分级

| 等级 | 信号 | 解读 |
| --- | --- | --- |
| 弱 | 口头表扬、说愿意试试、留邮箱 | 只能说明不反感 |
| 中 | 加 waitlist、愿意再聊、介绍同事 | 有兴趣，但未付出明显成本 |
| 强 | 付定金、开放数据、签 LOI、排期试点 | 有真实推进意愿 |

## Advancement 问法

```text
下一步谁需要参与？
你愿意把当前流程或数据样本发给我吗？
如果我们下周给出原型，你愿意安排 30 分钟让团队试用吗？
这件事如果不解决，本季度会有什么影响？
```

## 判断规则

- 用户愿意付出的成本越高，需求信号越强。
- 承诺必须有时间、对象和动作，不是“以后可以聊”。
- B2B 场景下，引入决策链成员比个人表扬更有价值。
- 如果连续访谈都没有 advancement，问题可能不够痛或目标用户不对。

## 输出建议

- 在旅程图机会点后标注证据等级。
- 对强信号直接转成实验或销售下一步。
- 对弱信号继续访谈，不直接进入功能开发。
