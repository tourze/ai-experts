
# 设计问卷

## 适用场景
- 客户满意度、PMF、功能优先级、入门引导或流失原因调研。
- 需要参考 [references/guest-insights.md](./guest-insights.md) 中的经验时。
- 想把问卷结果和访谈结论结合时，可配合 [mom-test](./mom-test.md)。

## 核心约束
- 先定义研究目标和决策问题，再写题目；每道题只测一件事。
- 样本、触发时机和激励方式必须解释清楚，否则结果容易偏。
- 问卷适合测分布和优先级，不适合替代深度访谈。

## 代码模式
```markdown
| 题目 | 类型 | 目的 | 结果如何使用 |
| --- | --- | --- | --- |
```

## 检查清单
- [ ] 已定义目标人群、发送时机和样本量。
- [ ] 题目没有双重问题、诱导式措辞和模糊量词。
- [ ] 已写明结果如何转成决策或后续访谈。
- [ ] 关键开放题不会让用户写超长负担。

## 反模式

### FAIL: 一卷打天下

```
30 题：满意度 + PMF + 功能优先级 + 流失原因 + 推荐
→ 完成率 < 5%
→ 收回的样本严重偏向"特别有空"的用户
```

### PASS: 单目的短问卷

```
PMF 卷（3-5 题）：
1. 如果不能再用，你会有多失望？(很/有点/不会)
2. 谁会从中受益最大？
3. 你最常用的功能？

→ 完成率 > 30%，决策清晰
```

### FAIL: 只看 NPS 分数

```
"NPS = 32，比上季 28 高"
→ 庆祝
→ 不知道为什么涨：渠道变了？产品变了？老用户流失？
```

### PASS: NPS + 原因 + 分群

```
NPS = 32（上季 28）
- 推荐者 +5pp（主因：新版速度提升）
- 贬损者 -3pp（主因：定价提高 + 取消旧套餐）
- 按 ARR 分群：> $1k 用户 NPS = 65，< $100 用户 NPS = -10
→ 决策：保住高价值，重审低端定价
```
