# 问卷与访谈实践洞察

## 适用场景

- 问卷回收了数据，但不知道如何解释偏差。
- 需要把 NPS、PMF、满意度或流失原因问卷转成后续访谈线索。
- 需要决定问卷触发时机、样本分层和激励方式。

## 关键原则

- 问卷测分布，访谈找原因；不要用问卷替代深度理解。
- 样本来源必须写清：活跃用户、流失用户、试用用户和付费用户不能混着解释。
- 问卷越长，越偏向“特别有空或特别有情绪”的用户。
- 激励会改变样本结构，尤其会吸引低意图填答。

## 设计建议

| 目标 | 推荐问法 | 后续动作 |
| --- | --- | --- |
| PMF | “如果不能再使用，你会有多失望？” | 访谈“非常失望”和“完全不失望”两端用户 |
| 流失 | “你停止使用前最后一次遇到的问题是什么？” | 按问题类型归类并找可修复阻塞 |
| 优先级 | MaxDiff 或强制排序 | 只把 top/bottom 作为决策线索 |
| 满意度 | 分数 + 开放原因 | 分群看趋势，不只看均值 |

## 反模式

- 只看平均分，不按用户类型分群。
- 用“你希望我们做什么功能”替代真实行为调查。
- 样本量很小却做精确百分比结论。
- 问卷结果直接进路线图，没有访谈或行为数据校验。
