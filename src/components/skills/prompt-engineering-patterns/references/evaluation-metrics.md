# 评估指标

用于衡量提示质量、评分标准设计和评分方法的指标。

---

## 核心指标

### 正确性

输出是否包含正确的答案/信息？

| 分数 | 含义 |
| ----- | ------------------------------- |
| 0 | 完全错误或不相关 |
| 1 | 部分正确，有重大错误 |
| 2 | 基本正确，有轻微错误 |
| 3 | 完全正确 |

### 格式合规性

输出是否遵循指定的格式？

| 分数 | 含义 |
| ----- | ------------------------------------ |
| 0 | 格式完全错误 |
| 1 | 格式正确但有显著偏差 |
| 2 | 格式正确有轻微偏差 |
| 3 | 完美符合格式 |

### 完整性

输出是否包含所有必需的元素？

| 分数 | 含义 |
| ----- | ------------------------------- |
| 0 | 缺少大部分必需元素 |
| 1 | 包含部分必需元素 |
| 2 | 包含大部分必需元素 |
| 3 | 所有必需元素均存在 |

### 简洁性

输出是否无多余内容？

| 分数 | 含义 |
| ----- | ---------------------------------- |
| 0 | 极度冗长，大部分为填充内容 |
| 1 | 有显著的不必要内容 |
| 2 | 略微冗长 |
| 3 | 简洁，每句话都有价值 |

### 基于事实程度

所有声明是否都有提供的上下文支持？（适用于 RAG/有依据的任务）

| 分数 | 含义 |
| ----- | ----------------------------------------------- |
| 0 | 大部分为幻觉 |
| 1 | 混合了有依据和幻觉的声明 |
| 2 | 几乎完全有依据，少量无支持的声明 |
| 3 | 每个声明都能追溯到上下文 |

## 评分标准设计

### 按任务类型加权

| 任务类型 | 正确性 | 格式 | 完整性 | 简洁性 |
| -------------- | ----------- | ------ | ------------ | ----------- |
| 分类 | 50% | 20% | 10% | 20% |
| 提取 | 40% | 25% | 25% | 10% |
| 分析 | 35% | 15% | 30% | 20% |
| 摘要 | 30% | 15% | 25% | 30% |
| 生成 | 30% | 20% | 25% | 25% |

### 自定义标准

当核心指标不足时，添加任务特定标准：

| 自定义标准 | 使用时机 |
| ---------------- | ------------------------------------------ |
| 语气/声音 | 品牌特定或受众特定的输出 |
| 具体性 | 答案应具体而非泛泛 |
| 可操作性 | 建议应具有可操作性 |
| 安全性 | 输出不得包含有害内容 |
| 创造力 | 输出应新颖或引人入胜 |

## 评分方法

### 逐查询评分

对于每个测试查询 × 变体：

1. 对每个标准评分（0-3）
2. 应用权重
3. 计算加权和
4. 归一化为 0-100%

```text
分数 = Σ（标准分数 × 权重）/（3 × Σ 权重）× 100
```

### 聚合评分

对于所有测试查询的每个变体：

- **平均分** —— 总体质量
- **最低分** —— 最差性能（对可靠性重要）
- **标准差** —— 一致性
- **通过率** —— 高于阈值的查询百分比

### 比较

| 指标 | 变体 A | 变体 B | 胜者 |
| --------------------- | --------- | --------- | ------ |
| 平均分 | 85% | 78% | A |
| 最低分 | 60% | 45% | A |
| 一致性（标准差） | 8% | 15% | A |
| 通过率（>70%） | 95% | 80% | A |

## 自动评估

### LLM 作为评判者

使用独立的 LLM 调用来评估输出：

```text
You are an evaluation judge. Score the following output on a 0-3 scale for each criterion.

Criteria:
- Correctness: Does the output contain the right answer?
- Format: Does it follow the specified format?
- Completeness: Are all required elements present?

Expected output: {expected}
Actual output: {actual}

Provide scores as JSON:
{"correctness": N, "format": N, "completeness": N, "reasoning": "..."}
```

**局限性：** LLM 评判者有自身的偏见。将其作为信号而非事实依据。
对于主观标准，人类评估更可靠。

### 精确匹配

对于答案明确无歧义的任务：

```python
score = 1.0 if output.strip() == expected.strip() else 0.0
```

### 部分匹配（召回率）

对于提取任务：

```python
expected_items = set(expected)
output_items = set(output)
recall = len(expected_items & output_items) / len(expected_items)
```
