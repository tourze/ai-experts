# Beachhead 市场选择

## 适用场景

- 早期产品有多个可进入细分市场，需要先选一个突破口。
- 增长资源有限，不能同时打多个垂类。
- 需要形成参考客户、口碑和可复制销售打法。

## 选择标准

| 维度 | 好信号 | 坏信号 |
| --- | --- | --- |
| 痛点强度 | 问题高频、成本明确、已有付费替代 | 只是“不错”的增量功能 |
| 可达性 | 有集中渠道、社群、会议或明确买方 | 分散且无法触达 |
| 同质性 | 客户流程相似，案例可复用 | 每个客户都要定制 |
| 整包解决 | 产品、服务、集成能组成完整方案 | 只解决很小一段问题 |
| 扩张潜力 | 邻近细分可顺势进入 | 成功后没有相邻市场 |

## 输出模板

```md
候选市场：[名称]
买方：[角色]
痛点：[具体场景 + 成本]
可达渠道：[渠道]
参考客户潜力：[能否互相影响]
是否 beachhead：[是/否 + 理由]
```

## 反模式

- 为了市场规模大，选择痛点分散的大行业。
- 同时打多个垂类，导致定位、功能和销售材料都不聚焦。
- 没有参考客户策略，只靠广告获客。
