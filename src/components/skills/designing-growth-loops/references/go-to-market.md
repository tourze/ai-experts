# Beachhead GTM 设计

## 适用场景

- 已选择 beachhead，需要设计渠道、销售、内容、案例和扩张顺序。
- 需要把定位、whole product 和增长飞轮串成可执行路径。
- 需要避免把功能 roadmap 当成 GTM。

## GTM 要素

| 要素 | 输出 |
| --- | --- |
| 目标买方 | 决策者、使用者、影响者 |
| 触达渠道 | 社群、会议、合作伙伴、内容、直销 |
| 价值主张 | 针对 beachhead 的一句话定位 |
| 销售动作 | demo、试点、POC、采购路径 |
| 参考客户 | 首批标杆、案例素材、口碑扩散方式 |
| 扩张顺序 | 相邻细分市场和进入条件 |

## 设计顺序

1. 定义买方和触发事件。
2. 选择最高信任的触达渠道。
3. 设计首批 10-30 个参考客户获取路径。
4. 把成功案例固化成内容、销售话术和产品模板。
5. 达到复用阈值后扩到相邻市场。

## 反模式

- “多投几个渠道”但没有清晰买方和触发时机。
- 产品、销售、内容分别讲不同定位。
- 没有标杆客户就急着扩张市场。
