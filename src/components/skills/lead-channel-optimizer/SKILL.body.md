## 代码模式
- 推荐输出模板：

```md
| 渠道 | 成本 | 线索质量 | 规模潜力 | 建议动作 |
|---|---|---|---|---|
| Paid Search | 高 | 高 | 中 | 继续投入 |
| Webinar | 中 | 高 | 低 | 保持 |
| Cold Email | 低 | 低 | 中 | 缩减 |
```

## 反模式

### FAIL: 只看 CPL

```
| 渠道 | CPL | 排名 |
| Cold Email | $5 | 第一 |
| Paid Search | $80 | 最后 |
→ “all-in cold email”
→ 6 个月：cold email 转化率 0.5%，成单 ARR < $10k
→ Paid Search 转化率 12%，砍掉损失 ARR $200k+
```

### PASS: CPL + 质量 + 闭环

```
| 渠道 | CPL | Lead-to-Customer | CAC | LTV/CAC |
| Cold Email | $5 | 0.5% | $1000 | 1.2 |
| Paid Search | $80 | 12% | $670 | 6.5 |
| Webinar | $40 | 8% | $500 | 9.2 |

→ Webinar 才是最优，加大；Paid Search 维持；Cold Email 砍
```

### FAIL: “全都做”

```
“建议：维持 8 个渠道，每个都加 10% 预算”
→ 没有优先级 → 资源摊薄 → 没有渠道做到位
```

### PASS: 4 类动作

```
继续投入（Top 1-2）：每月 +20% 预算
保持（中间）：维持现状，季度复盘
缩减（次差）：3 个月内降 50% 预算或证明
暂停（最差）：本月停，6 个月不再考虑
```
