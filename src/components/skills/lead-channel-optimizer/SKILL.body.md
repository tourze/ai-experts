## 核心约束
- 渠道优先级必须同时看产出、稳定性和执行复杂度，不能只看单次线索成本。
- 先统一口径：时间窗、归因方式、目标客户定义、是否算销售成本。
- 输出必须包含“继续投入 / 保持 / 缩减 / 暂停”四类动作建议。
- 若需要重做广告结构，配合 [paid-ads](../paid-ads/SKILL.md)。

## 代码模式
- 推荐输出模板：

```md
| 渠道 | 成本 | 线索质量 | 规模潜力 | 建议动作 |
|---|---|---|---|---|
| Paid Search | 高 | 高 | 中 | 继续投入 |
| Webinar | 中 | 高 | 低 | 保持 |
| Cold Email | 低 | 低 | 中 | 缩减 |
```

## 检查清单
- 是否说明了渠道的口径、周期和数据来源。
- 是否区分了“当前回报高”与“长期可放大”。
- 是否写出资源重分配后的下一步动作。
- 是否把品牌建设型渠道和直接转化型渠道分开比较。

## 反模式

### FAIL: 只看 CPL

```
| 渠道 | CPL | 排名 |
| Cold Email | $5 | 第一 |
| Paid Search | $80 | 最后 |
→ “all-in cold email”
→ 6 个月：cold email 转化率 0.5%，成单 ARR < $10k
→ Paid Search 转化率 12%，砍掉损失 ARR $200k+
```

### PASS: CPL + 质量 + 闭环

```
| 渠道 | CPL | Lead-to-Customer | CAC | LTV/CAC |
| Cold Email | $5 | 0.5% | $1000 | 1.2 |
| Paid Search | $80 | 12% | $670 | 6.5 |
| Webinar | $40 | 8% | $500 | 9.2 |

→ Webinar 才是最优，加大；Paid Search 维持；Cold Email 砍
```

### FAIL: “全都做”

```
“建议：维持 8 个渠道，每个都加 10% 预算”
→ 没有优先级 → 资源摊薄 → 没有渠道做到位
```

### PASS: 4 类动作

```
继续投入（Top 1-2）：每月 +20% 预算
保持（中间）：维持现状，季度复盘
缩减（次差）：3 个月内降 50% 预算或证明
暂停（最差）：本月停，6 个月不再考虑
```
