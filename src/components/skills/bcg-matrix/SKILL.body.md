## 适用场景
- 多产品/多业务线公司的资源分配、投资优先级排序。
- 默认 BCG 2x2 快速分类；需要更精细的多维评估时切到 GE-McKinsey 九宫格模式（见下文）。

## 核心约束（BCG 2x2 模式）
- **相对**市场份额：对标最大竞品，不是绝对数字。分界线按行业调——SaaS 的"高增长"和制造业完全不同，不要默认 10%/1.0。
- 问题象限是唯一需要 Go/No-Go 选择的——给窗口期 + 里程碑，到期不达标就砍。
- 全投 = 没决策。BCG 的核心价值是**强迫取舍**。
- 最终输出必须包含资源流向：现金牛 →→ 明星 + 被选中的问题业务，瘦狗 → 退出释放资源。

## GE-McKinsey 九宫格模式（多指标加权）
当 2x2 太粗、需要量化打分时切到此模式：

- X 轴竞争实力（内部）、Y 轴市场吸引力（外部），都是多指标加权得分。
- **权重不能平均分配**——必须反映行业关键成功因素：
  - SaaS：市场增长率 30%、NRR 天花板 25%（增长是 SaaS 估值核心）
  - 制造业：盈利性 30%、进入壁垒 25%（资本密集）
  - 快消品：市场规模 30%、竞争强度 25%（渠道为王）
- 九宫格结论是**战略方向**，不是精确数字——争论 3.2 还是 3.3 没有意义。
- 内外部维度至少各用 5 个指标，打分差异 > 1 分才有区分度。
- 与 BCG 2x2 结果交叉验证：分类一致提升信心，分歧暴露被忽略的指标。

## 不适用场景
- 单产品公司：BCG/GE 都是多业务组合工具。用 [s-curve-growth](../designing-growth-loops/SKILL.md) 判断生命阶段。
- 缺乏行业数据：GE 模式权重设定依赖行业理解，数据不足时退回 BCG 2x2 快速分类。
- 评估外部竞争：BCG/GE 看内部组合，不看行业结构。用 [porters-five-forces](../porters-five-forces/SKILL.md)。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 相对市场份额是对比最大竞品，不是绝对数字。
- [ ] 分界线设定有行业依据。
- [ ] 问题业务做了选择（不能全投也不能全砍）。
- [ ] 给出了明确的资源流向建议。
- [ ] 用 GE 模式时：指标权重有行业依据（不是平均分配），内外部维度至少各 5 个指标，九宫格定位转化为具体战略动作。

## 反模式

### FAIL: 所有业务都是明星

```
产品A: 明星 → 加大投入
产品B: 明星 → 加大投入
产品C: 问题 → 加大投入变明星
→ 全投 = 没决策，BCG 的核心是取舍
```

### PASS: 敢砍敢投

```
产品A（明星）：市场增速 25%，份额第一 → 追加 40% 预算
产品B（现金牛）：增速 3%，份额第一 → 砍 30% 预算，利润输血 A
产品C（瘦狗）：增速 -2%，份额第四 → Q3 前完成出售或关停
产品D（问题）：增速 20%，份额第三 → 给 6 个月窗口期，份额进前二则追投，否则放弃
```

### FAIL: GE 模式所有指标权重一样

```
市场规模 20%、增长率 20%、盈利性 20%、竞争 20%、技术 20%
→ 等于没做判断，权重应反映行业关键成功因素
```

### PASS: GE 模式行业化权重

```
SaaS 行业市场吸引力权重：
- 市场增长率 30%（增长是 SaaS 估值核心）
- NRR 天花板 25%（决定复合增长潜力）
- 竞争集中度 20% / 监管风险 15% / 技术替代 10%
→ 权重体现行业特性，打分结果有区分度
```
