## 代码模式

```markdown
# [产品/功能名称] — 新闻稿

**标题**：一句话说清楚用户获得了什么（不超过 15 字）
**副标题**：补充目标用户和核心价值

**日期 | 地点** — 开头段落：用户是谁、他们的痛点、我们提供了什么。

**问题段**：用户今天面临什么困境？（2-3 句）

**方案段**：产品如何解决这个问题？（2-3 句，用户视角）

**用户引言**："[虚构用户名]，[角色]，说：……"（展示用户获得的价值）

**如何开始**：用户从哪里获得/使用这个产品。

**负责人引言**："[负责人名]说：……"（说明为什么现在做这件事）

---

## 外部 FAQ（面向用户）
1. 这和现有方案有什么不同？
2. 我需要付费吗？定价是什么？
3. 我的数据安全吗？

## 内部 FAQ（面向团队/管理层）
1. 为什么现在做这个？机会窗口是什么？
2. 需要多少资源？ROI 预期是什么？
3. 最大的技术风险是什么？
4. 如果失败，退出策略是什么？
5. 竞品为什么没做 / 做了但没成功？
```

## 反模式

### FAIL: 公司视角的广告

```
标题："XX 公司发布业界领先的 AI 平台"
→ 用户读完不知道自己能得到什么
→ 全篇都是"我们的技术""我们的团队"
```

### PASS: 用户视角的价值

```
标题："跨境卖家现在可以 10 分钟完成多币种对账"
→ 用户立刻知道：这是给我的，解决对账慢的问题
```

### FAIL: FAQ 全是软球

```
Q: 产品好用吗？ A: 非常好用。
Q: 安全吗？ A: 非常安全。
→ 没有回答任何真正的疑虑
```

### PASS: FAQ 直面硬问题

```
Q: 为什么竞品 A 做了类似功能但没起来？
A: A 只支持 3 种货币，且需要手动导入；我们接了 15 种货币的自动对接。
   A 失败的核心原因是数据源覆盖不够，这是我们 6 个月的数据集成投入解决的。
```
