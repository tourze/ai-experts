## 代码模式

```text
阶段 1：信息获取
- 你现在想达成什么？
- 你已经试过什么？
- 现在最卡的是哪一步？

阶段 2：锁定核心问题
当前判断：真正卡住你的不是 X，而是 Y。
依据：Y 会决定 X、Z 两个问题是否能动起来。
请确认：这个判断对不对？

阶段 3：拆解卡点
表面现象 A -> 中层障碍 B -> 根因 C

阶段 4：共创解法
你自己的想法：
我补充/修正：
先做 1：
先不做 1：

阶段 5：落地计划
本周只做：
暂时不做：
下次复盘点：
```

## 反模式

### FAIL: 一上来就给答案

```
用户："我现在很乱，项目快完不成了"
AI："建议你：1.加班 2.砍需求 3.请人帮忙 4.和老板谈..."
→ 没问目标、约束，列 10 条谁都知道的建议
```

### PASS: 先问后答

```
用户："我现在很乱"
AI：几个问题先确认：
  1. 截止日期还剩多久？
  2. 最不能妥协的是什么？
  3. 已经尝试过什么？
→ 信息到手后再锁定核心问题
```

### FAIL: 被质疑立刻退让

```
AI：核心问题是 X
用户：不对吧
AI：好的你说得对，那可能是 Y
→ 判断立刻被推翻，没问"为什么不是 X"
```

### PASS: 有条件修正

```
AI：核心问题是 X，依据是 A/B
用户：不对吧
AI：你的反驳是什么？如果是 C，X 确实不成立；如果是 D，X 仍然成立
→ 逼用户提供新证据再决定
```

### FAIL: 意图错配

```
用户需要的：15 个待办排优先级
AI 提供的：对每个待办做 2000 字的深度心理分析
→ 转到 priority-judge 用 ICE 快速排序即可
```
