## 不适用场景
- 极度扁平组织（< 10 人、全员全栈）：角色边界模糊是特性不是 bug，RACI 会引入不必要的官僚。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 反模式

### FAIL: 所有人都是 R

```
| 任务 | 产品 | 研发 | 设计 | 运营 |
|------|------|------|------|------|
| 上线 | R    | R    | R    | R    |
-> 人人负责 = 无人负责
```

### PASS: 职责明确

```
| 任务     | 产品 | 研发 | 设计 | 运营 |
|----------|------|------|------|------|
| 需求定义 | A,R  | C    | C    | I    |
| 技术方案 | C    | A,R  | I    | I    |
| 视觉设计 | C    | I    | A,R  | I    |
| 上线发布 | I    | A,R  | I    | C    |
-> 每个任务一个 A，职责清晰
```
