# 落地页结构与检查清单

## 定义

- 首屏（Above-the-fold）：滚动前可见的网页部分。
- 折页以下（Below-the-fold）：用户需滚动才能看到的网页部分。

使用折叠概念来改进参与度、留存、转化、UX 和 SEO。

## 经过验证的落地页结构

默认区块顺序：

1. 导航栏
2. 主视觉区
3. 社会证明
4. CTA 区块
5. 功能展示
6. CTA 重复 1 到 2 次
7. 页脚

备注：
- 保持导航最小化。避免将注意力引向竞争性目的地。
- 考虑固定或粘性 CTA（尤其在移动端上），当它能减少摩擦时。

## 首屏：5 步检查清单

1. 定义你提供的价值（标题）
2. 详细说明你的方法（支持的副标题）
3. 帮助用户想象它（视觉元素）
4. 建立可信度（社会证明）
5. 简化下一步（行动号召）

快速验证问题：
- 首次访问者能否在 5 秒内解释清楚提供的价值？
- CTA 的成果是否显而易见（点击后会发生什么）？
- 视觉元素是否与承诺匹配（不纯为装饰）？

## 折页以下：说服序列

1. 展示清晰价值（功能）
2. 激发行动（社会证明）
3. 澄清问题（FAQ）
4. 重复你的行动号召（CTA）

仅在需要时添加可选区块：
- 定价与方案
- 安全/合规
- 对比
- 案例研究
- 集成

## 值得学习的标注示例

### Notion（模式）

- 支持性副标题澄清了承诺。
- 强大的产品视觉帮助用户知道期望什么。
- CTA 微文案降低风险（"免费使用"）。
- 页眉中的固定 CTA 保持操作可用。

### Semrush（模式）

- 成果驱动的标题。
- 单一、专注的转化路径（输入域名/关键词 -> 立即开始）。
- 社会证明标志靠近主视觉区放置以建立信任。
