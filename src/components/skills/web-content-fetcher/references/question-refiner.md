# Question Refiner

## 适用场景

- 用户说"研究一下 X" / "帮我比较 A 和 B" / "调研下行业现状"，但没讲输出形式、地域、时间窗口、受众。
- [deep-research](../../deep-research/SKILL.md) / deep-researcher agent 启动前，把原始问题打磨成可执行 prompt。
- 用户信息散落多轮对话，需要收敛成单一 prompt。
- 不适用：已给出结构化 prompt；简短事实查询；用户明确说"你决定"。

## 核心约束

- 先问再做。澄清没答完不生成 prompt、不开始研究。
- 分 5 维按需问，不机械全问；已清楚的不追问。
- 澄清题带选项（A/B/C/D）或示例，降低回答成本。
- 一次性批量 3-6 题，不挤牙膏。
- 最终 prompt 必须 7 段齐全：TASK / CONTEXT / SPECIFIC QUESTIONS / KEYWORDS / CONSTRAINTS / OUTPUT / FINAL_INSTRUCTIONS。
- 不回答研究问题本身；只生成 prompt。

5 个澄清维度：

1. **核心问题** — 最想解决什么？驱动什么决策？
2. **输出形式** — 执行摘要 / 全报告 / 对比矩阵 / 演示稿？长度？
3. **范围边界** — 地域、时间窗口、行业；要显式排除什么。
4. **来源信任** — 同行评审 / 行业报告 / 官方文档 / 新闻？优先或回避哪些源。
5. **特殊要求** — 受众、合规约束、数据或图表需求。

## 代码模式

```text
交互流程
1. 扫描 5 维，标出空缺
2. 一次批量发 3-6 个澄清题（带选项）
3. 等待答复；不足继续补问
4. 生成 7 段结构化 prompt
5. 交给 deep-research / deep-researcher / wiki-researcher 执行
6. 完成后提示用 citation-validator 做发布前把关
```

完整 7 段 prompt 模板与质量检查见 [references/prompt-template.md](./prompt-template.md)。

## 检查清单

- 是否扫描 5 维，只问空缺的，不机械发完 5 组。
- 每题是否带选项或示例。
- 是否一次性批量发问。
- 是否等用户答完再生成 prompt。
- 7 段是否齐全，字段是否都填了具体内容（不留占位符）。
- FINAL_INSTRUCTIONS 是否强调了引用规范与冲突处理。
- 是否提示下游交给 citation-validator。

## 反模式

### FAIL: 直接开搜

```
用户：研究一下 AI 在医疗的应用
助手：[立刻 WebSearch] → 泛泛结果 → 返工
```

### PASS: 先澄清

```
助手：动手前 3 题：
1. 焦点：A 临床诊断 / B 药物发现 / C 医院运营 / D 全部
2. 输出：A 摘要 2-3 页 / B 全报告 20-30 页 / C 市场预测
3. 范围：地域 + 时间窗？
```

### FAIL: 一次只问一个

```
Q1→答→Q2→答→... 用户疲惫，回答越来越敷衍
```

### PASS: 批量 3-6 题

```
一次发 4 题（焦点 / 输出 / 范围 / 受众），一次回完
```

### FAIL: 开放式追问

```
Q：你想要什么输出？
用户：... 好看的？
```

### PASS: 选项式

```
Q：输出形式：
  A. 执行摘要（2-3 页）
  B. 完整报告（20-30 页）
  C. 对比矩阵表格
  D. 其他，请描述
```

完整交互示例与更多反模式见 [references/patterns.md](./patterns.md)。
