## 代码模式
- 算法选择参考 [决策指南](references/decision-guide.md)。
- 若要比较 Gale-Shapley、Hungarian、层级约束或实体解析方案，按 [算法对比表](references/algorithms.md) 提供的维度对齐。
- 输出建议分成：模型定义 → 约束表 → 评分函数 → 失败样本 → 调整方案。


## 检查清单
- 是否把输入实体、层级关系和约束表述清楚。
- 是否区分了必须满足的约束和可优化目标。
- 是否用样例解释匹配失败或不稳定现象。
- 是否说明求解复杂度和可扩展性限制。

## 反模式

### FAIL: 只谈算法

```
"用 Hungarian 算法分配"
→ 客户："那班级容量限制呢？兄弟姐妹要在一起呢？老师偏好呢？"
→ 算法跑出来一个分配方案 / 半数家长抗议
```

### PASS: 业务约束建模

```md
## 硬约束（必须满足）
- 班级容量 ≤ 35
- 兄弟姐妹同班
- 特殊教育学生 ≤ 2/班

## 软约束（最大化）
- 性别比 ≈ 50:50（权重 3）
- 学业能力分布均衡（权重 5）
- 朋友请求满足（权重 1）

## 算法
满足硬约束 → Hungarian 优化软约束目标函数
```

### FAIL: 不做样本验证

```
权重：性别 3, 能力 5, 朋友 1（拍脑袋设定）
→ 跑出来：能力均衡但全班同性别 → 失败
```

### PASS: 样本验证

```
1. 用上一年真实数据 + 当前模型重跑
2. 与去年人工分配对比，看哪些硬约束被违反
3. 调整权重 → 再跑 → 收敛到合理结果
4. 关键指标（性别比方差 / 能力方差 / 朋友满足率）写文档
```
