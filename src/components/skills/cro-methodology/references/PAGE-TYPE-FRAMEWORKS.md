# page-cro 页面类型框架

不同页面承担的转化职责不同，分析顺序也不同。

## 首页（Homepage）

目标：快速建立认知与信任，导向核心路径。

检查重点：
- 首屏是否在 5 秒内说明“是谁 / 为谁 / 有何价值”
- 主 CTA 是否唯一且可见
- 信任信号是否在首屏或 CTA 附近出现

## 落地页（Landing Page）

目标：承接单一流量意图并触发行动。

检查重点：
- 信息是否围绕单一受众与单一意图
- 是否有与广告承诺一致的连续性
- 表单步骤和字段是否最小化

## 定价页（Pricing）

目标：降低决策不确定性，帮助完成方案选择。

检查重点：
- 套餐差异是否可比较、可扫描
- 常见异议（合同、退款、迁移）是否提前回答
- CTA 是否与用户阶段匹配（试用 / 咨询 / 购买）

## 功能页（Feature）

目标：把“功能”翻译成“结果”。

检查重点：
- 是否先讲结果再讲机制
- 是否有场景化证据（案例、指标、截图）
- 是否明确下一步动作而非停在说明层

## 博客页（Blog）

目标：搜索流量转化为订阅、试用或线索。

检查重点：
- 阅读路径是否有明确过渡到产品价值
- 中段和结尾 CTA 是否与主题强相关
- 导航和外链是否过早分散注意力
