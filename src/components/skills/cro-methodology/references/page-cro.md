
# 页面转化优化（page-cro）

## 适用场景
- 营销页面有流量但转化率低，需要定位阻塞点。
- 对首页、落地页、定价页、功能页或博客做转化审计。
- 需要产出可执行的优化建议和 A/B 测试方案。

## 核心约束
- 先问清页面目标和当前数据，再给建议；没有数据时先列诊断问题。
- 按影响力排序分析，不要从细节开始。
- 每条建议说明预期影响和实施难度。
- 文案建议给出原文对照和替代方案。
- 弹窗问题切 [popup-cro](./popup-cro.md)；实验方法论切 [cro-methodology](../SKILL.md)。

## CRO 分析框架（按影响力降序）

1. **价值主张清晰度** — 首屏 5 秒内回答"这是什么、为谁、为什么选你"。
2. **标题有效性** — 传递结果而非功能；副标题补充证据。
3. **CTA 位置/文案/层级** — 首屏可见、传递行动后价值、主次分明。
4. **视觉层级与可扫描性** — 扫读即可理解主线，无文字墙。
5. **信任信号与社会证明** — CTA 附近放具体信任元素和风险逆转承诺。
6. **异议处理** — 预判购买疑虑，FAQ 针对阻力而非百科。
7. **摩擦点** — 表单最少化、加载速度、移动端、导航泄漏。

## 产出格式

按 Quick Wins / High-Impact Changes / Test Ideas / Copy Alternatives 四类输出，详见 [OUTPUT-FORMAT](./OUTPUT-FORMAT.md)。

## 页面类型框架

按页面类型选择对应框架：首页、落地页、定价页、功能页、博客，详见 [PAGE-TYPE-FRAMEWORKS](./PAGE-TYPE-FRAMEWORKS.md)。

## 诊断问题

分析前主动确认：
1. 页面目标和当前转化率？
2. 流量来源及各来源转化差异？
3. CTA 后续流程和步骤数？
4. 有无热力图/行为数据？用户在哪流失？
5. 之前尝试过什么优化？效果如何？

## 检查清单
- 是否明确了页面单一核心目标。
- 是否按 7 层框架从价值主张开始逐层分析。
- 是否区分了 Quick Wins 和 High-Impact Changes。
- 文案建议是否有原文对照和至少 2 个替代方案。
- 测试建议是否写清假设、指标和样本量。
- 是否考虑了移动端和流量来源影响。

## 反模式

### FAIL: 不问数据直接改
```
"按钮改绿色，标题加感叹号" → 不知道阻塞在哪一层
```

### PASS: 先定位再优化
```
首屏跳出 65% → 价值主张不清晰 → 重写标题 + 加信任信号 → A/B 验证
```

### FAIL: 所有页面同一套建议
```
首页/落地页/定价页用同样模板 → 忽略页面类型和用户意图差异
```

### PASS: 结构优先于文案
```
先修视觉层级 → 再修 CTA 位置 → 最后调文案措辞
```
