# 异议/反异议（O/CO）框架

CRO 的核心技术。不要猜异议——去调研它们，然后系统性反驳。

## O/CO 表

创建两列表：

| 异议 | 反异议 |
|------|--------|
| "太贵了" | ROI 计算器、分期方案、竞品对比 |
| "不信任这家公司" | 客户推荐、奖项、媒体报道、保障 |
| "不适合我的情况" | 同类客户案例、免费试用 |

## 如何发现异议

### 一手来源（直接来自访客）

**退出调研**（Qualaroo、Hotjar 等）
- "什么阻止了你今天注册？"
- "你还有什么疑问？"
- "什么能让这个决定更容易？"

**在线客服记录**
- 搜索"但是""不过""担心"等关键词
- 记录反复出现的犹豫

**客服工单**
- 购前问题暴露异议
- 客服的"标准回复"是经过验证的反异议宝库

**销售通话录音**
- 销售反复听到什么异议？
- 哪些反驳话术有效？

**"推荐给朋友"消息**
- 客户怎么向朋友描述你？
- 他们强调什么？淡化什么？

### 二手来源

**评价（自己和竞品的）**
- 差评 = 未回应的异议
- 好评 = 有效的反异议

**社媒提及**
- Reddit、Twitter 上关于你品类的讨论
- 你的受众讨论替代方案的论坛

**竞品分析**
- 竞品回应了哪些异议？
- 他们强调什么证据？

## 异议类型

### 显性异议
容易回应——访客明确说出了：
- "太贵"
- "没有免费试用"
- "要绑信用卡"

### 隐性异议
难以识别——访客不会承认：
- "我太懒了不想学"
- "不确定我能坚持"
- "好得不像真的"

**隐性异议技巧**：用"只反异不点破"的方式处理。

差："担心你太懒学不会？"
好："让音频替你完成学习。"

### 通用五大异议

大多数产品都会遇到以下某种版本：

1. **信任**："凭什么信你？"
2. **价格**："值不值这个钱？"
3. **适配**："对我这种情况有用吗？"
4. **时机**："为什么现在就行动？"
5. **精力**："需要费多大力气？"

## 反异议技巧

### 应对信任异议
- 具体推荐（含照片、姓名、结果）
- 媒体 Logo（"曾出现在…"）
- 奖项和认证
- 团队资质
- 经营年限、服务客户数
- 退款保障

### 应对价格异议
- ROI 计算器
- 成本对比（vs 竞品、vs 不解决问题）
- 分期方案
- 价格锚定（先展示更贵选项）
- 强调价值而非功能

### 应对适配异议
- 相似场景的案例研究
- 分群落地页
- "专为 [特定受众] 设计"的信息
- FAQ 回应边缘情况
- 免费试用或样品

### 应对时机异议
- 限时优惠（仅用真实稀缺）
- 延迟成本
- 季节相关性
- 紧迫感数据

### 应对精力异议
- "帮你搞定"的表述
- 分步拆解
- 时间预估（"5 分钟设置完成"）
- 引导支持
- "系统"或"流程"式语言

## 放置策略

不要把异议藏在 FAQ 里。在以下位置回应：

- **摩擦点**：信用卡异议 → 支付表单正下方
- **流程中**：阅读时异议自然产生的地方
- **首屏**：针对主要异议
- **多次出现**：不同形式（文字、视频、推荐）

## O/CO 审计问题

1. 访客的 Top 5 异议是什么？
2. 页面上有哪些我们没回应？
3. 哪些回应太弱（没有证据）？
4. 反异议是否放在异议产生的地方？
5. 竞品用了哪些反异议我们没用？
