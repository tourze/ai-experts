## 代码模式

```json
{
  "研究主题": "中国企业知识管理软件市场",
  "分析目标": ["市场规模", "竞争格局", "客户需求", "进入建议"],
  "交付物": "咨询级 Markdown 报告",
  "数据要求": ["官方统计", "公司财报", "产品定价页", "用户访谈摘录"]
}
```

可直接按下面骨架开工：

```markdown
# 执行摘要
## 1. 问题定义
## 2. 研究方法与数据源
## 3. 核心发现
## 4. 战略含义
## 5. 建议与下一步
```

## 反模式

### FAIL: 跳过框架直接写

```
"我先写 30 页报告"
→ 第 20 页发现研究问题定义错了
→ 全部推倒
```

### PASS: 框架先行

```
1. 写 1 页问题定义 + 子问题树
2. 写 1 页方法/数据要求
3. 与用户对齐这 2 页
4. 才开始正文
```

### FAIL: 二手循环引用

```
媒体 A 引用 媒体 B 引用 媒体 C → "市场规模 1000 亿"
→ 追到底，C 也是引用 A
→ 没人知道原始出处
```

### PASS: 一手 + 标 source

```
| 数据 | 来源 | 年份 | 置信度 |
| 市场规模 600 亿 | 中国信通院《XX 白皮书》 | 2024 | 高 |
| 增速 18% | A 公司 ICP 备案 + B 上市公司财报推算 | 2024 | 中 |
```
