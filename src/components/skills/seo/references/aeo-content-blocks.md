# AEO 内容块模板

## 适用场景

- 目标是 Featured Snippet、AI Overview、语音搜索或站内答案卡片。
- 需要把长篇内容重构成可直接提取的答案块。
- 页面已有内容但首屏答案不清晰，AI 抽取到的是铺垫而不是结论。

## 标准块

### Definition Block

```md
## 什么是 [概念]？

[概念] 是 [类别]，用于 [核心用途]。它通常包含 [关键组成]，适合 [典型场景]。
```

要求：第一句 35-60 字，直接回答“是什么”，不要先讲历史。

### Step-by-Step Block

```md
## 如何完成 [任务]？

1. [动作 1]，确认 [检查点]。
2. [动作 2]，产出 [结果]。
3. [动作 3]，验证 [成功信号]。
```

要求：每步用动词开头，并包含可验证结果。

### Comparison Table Block

```md
| 维度 | 方案 A | 方案 B |
| --- | --- | --- |
| 适合场景 | ... | ... |
| 主要优势 | ... | ... |
| 主要限制 | ... | ... |
```

要求：比较维度固定，避免“各说各话”。

### FAQ Block

```md
### [自然语言问题]？

[50-100 词直接回答]。如果存在例外，第二句说明边界。
```

要求：问题用用户会搜索的口吻，不要写内部术语。

## 交付检查

- 前 1-2 句是否能独立回答查询。
- H2/H3 是否只处理一个搜索意图。
- 关键数字是否带来源、年份和口径。
- 表格是否有明确比较维度。
- FAQ 是否能映射到真实长尾查询。
