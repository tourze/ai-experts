# 导航与内链模式

## 适用场景

- 主导航过载、移动端不可用、重要页面点击深度过深。
- 栏目页和详情页缺少互相支持的内链。
- 需要把商业转化路径和搜索爬取路径统一起来。

## 主导航原则

- 保持 5-7 个主入口，优先放产品、方案、价格、客户、资源、文档。
- 公司介绍、招聘、法律、联系方式放 footer。
- 二级菜单按用户任务分组，不按组织架构分组。
- 关键 CTA 在桌面和移动端都应一屏可达。

## 内链模块

| 模块 | 位置 | 作用 |
| --- | --- | --- |
| 相关推荐 | 文章或产品页底部 | 提升主题集群连接 |
| 面包屑 | 所有深层页面 | 暴露层级和返回路径 |
| 相关用例 | 产品/方案页 | 连接功能与场景 |
| 对比/替代方案 | 购买决策页 | 承接商业意图 |
| 热门问题 | 页面中下部 | 承接长尾查询 |

## 点击深度检查

```text
首页 → 核心转化页：≤ 2 次点击
首页 → 核心 SEO 落地页：≤ 3 次点击
首页 → 长尾内容页：≤ 4 次点击，且有主题页入口
```

## 反模式

- 顶部导航塞 12 个以上入口。
- Footer 放大量无分组链接，像链接农场。
- 相关文章只按发布时间推荐，和主题无关。
- 重要页面只能从站内搜索到达。
