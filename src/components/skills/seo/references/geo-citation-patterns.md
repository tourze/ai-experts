# GEO 引用模式

## 适用场景

- 需要提高内容在 ChatGPT、Perplexity、AI Overview 等生成式答案中的引用概率。
- 需要审计品牌是否被 AI 回答提及，以及哪些来源被竞争对手占据。
- 需要把内容从“营销陈述”改成“可被模型引用的证据陈述”。

## 高引用概率块

### 统计引用块

```md
根据 [机构] 在 [年份] 发布的 [报告/调研]，[样本或口径] 显示：[具体数据]。
```

要点：给出机构、年份、样本、口径和原始来源链接。不要只写“研究表明”。

### 专家引言块

```md
“[可验证观点]。” — [姓名]，[职务]，[机构]
```

要点：专家身份应与主题相关；引言必须能支撑页面主张。

### 自包含答案块

```md
[产品/概念] 适合 [场景]，因为 [原因 1] 和 [原因 2]。不适合 [边界场景]。
```

要点：读者不看上下文也能理解这段话。

### 证据三明治块

```md
结论：[明确主张]
证据：
- [来源 A]：[数据或事实]
- [来源 B]：[数据或事实]
- [来源 C]：[数据或事实]
行动：因此应该 [建议]
```

## 反模式

- 没有来源的统计数字。
- 只有品牌自述，没有第三方证据。
- 一段里混多个结论，AI 无法稳定引用。
- 只追求 AI 可读性，牺牲人类读者的判断路径。
