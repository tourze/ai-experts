# Programmatic SEO 12 种模式

批量生成 SEO 页面的标准化策略。核心原则：每页有独特价值、优先用自有数据、子目录优于子域名、质量优于数量。

---

## 模式总览

| 模式 | 关键词格式 | 示例 |
|------|-----------|------|
| Templates | "[类型] 模板" | "简历模板"、"合同模板" |
| Curation | "最好的 [类目]" | "最好的建站工具" |
| Conversions | "[X] 转 [Y]" | "$10 美元转人民币" |
| Comparisons | "[X] vs [Y]" | "Webflow vs WordPress" |
| Examples | "[类型] 示例" | "落地页示例" |
| Locations | "[服务] + [地点]" | "上海律师事务所" |
| Personas | "[产品] + [受众]" | "适合地产的 CRM" |
| Integrations | "[产品 A] + [产品 B] 集成" | "Slack Asana 集成" |
| Glossary | "什么是 [术语]" | "什么是 pSEO" |
| Translations | 多语言内容 | 本地化页面 |
| Directory | "[类目] 工具" | "AI 写作工具" |
| Profiles | "[实体名]" | "Stripe CEO" |

---

## 选择依据

| 你拥有的 | 适合的模式 |
|----------|-----------|
| 自有数据 | Directory、Profiles |
| 产品有集成 | Integrations |
| 设计/创意产品 | Templates、Examples |
| 多细分受众 | Personas |
| 本地业务 | Locations |
| 工具型产品 | Conversions |
| 内容/专业知识 | Glossary、Curation |
| 有竞品格局 | Comparisons |

可叠加使用，如"上海最好的联合办公空间"= Locations + Curation。

## 实施框架

### 1. 关键词模式研究
- 识别重复结构和变量
- 验证聚合搜索量和分布（头部 vs 长尾）
- 确认趋势方向

### 2. 数据源评估
数据防御性层级（从高到低）：
1. 自有数据（你创造的）
2. 产品衍生数据（来自用户行为）
3. UGC 数据（社区贡献）
4. 授权数据（独家获取）
5. 公开数据（任何人都能用，最弱）

### 3. 模板设计
- 标题包含目标关键词
- 每页有独特的介绍文案（不只是换变量）
- 数据驱动的内容区块
- 相关页面内链
- 与搜索意图匹配的 CTA

### 4. 内链架构
Hub-and-spoke 模型：
- Hub：主分类页
- Spoke：各个 programmatic 页面
- 相关 spoke 之间交叉链接
- 所有页面可从主站到达

### 5. 索引策略
- 优先索引高搜索量模式
- 极薄变体 noindex
- 按页面类型分 sitemap
- 管理抓取预算

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 只换城市名的重复内容 | 每个地区页面有本地化独特内容 |
| 多页面竞争同一关键词 | 一个关键词只对应一个页面 |
| 生成无搜索量的页面 | 先验证搜索需求再生成 |
| 数据过时不更新 | 建立数据更新机制 |
| 页面只为搜索引擎存在 | 页面必须对用户有真实价值 |
