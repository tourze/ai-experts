# Citation Validator

## 适用场景

- 研究报告、PRD、白皮书、对外发布内容交付前做"每个事实都有出处"的最后把关。
- 面向高风险决策（医疗 / 法律 / 财务 / 合规），或接手他人研究怀疑存在编造引用。
- [deep-research](../../deep-research/SKILL.md) / [knowledge-synthesis](../../deep-research/references/knowledge-synthesis.md) 闭环的最后一步。
- 不适用：纯主观分析、代码实现、无外部事实依赖的内部讨论。

## 核心约束

- 每条事实声明挂一个可验证来源，四要素齐全：作者/组织、日期、标题、URL/DOI。
- 按 A-E 评级来源；低于 C 要么升级证据，要么降级结论表述。
- 高风险声明必须 chain-of-verification：2-3 个独立源交叉确认。
- URL 必须实际可达或有 archive.org 备份；死链等于无证据。
- 引用存在但原文不支持 = 幻觉，必须显式标出并要求整改。
- 实际打开来源对照原文，不只看引用格式。

来源质量 A-E：

- **A** 同行评审、系统综述、RCT、监管机构原始文件、政府原始统计
- **B** 行业分析师（Gartner/Forrester/IDC）、政府发布、临床指南、成熟媒体调查
- **C** 专家白皮书、公司官方博客/公告、知名技术媒体
- **D** 预印本、会议摘要、无编辑审核的个人博客、社区问答
- **E** 匿名内容、论坛发言、利益冲突、过期资料、断链

## 代码模式

```text
审查流程
1. 抽取声明 — 列所有事实断言（数字/日期/引语/因果/比较）
2. 核对存在 — 每条是否挂了引用
3. 核对完整 — 四要素齐全
4. 核对支持 — WebFetch 打开来源对照原文
5. 评级来源 — A-E
6. 高风险交叉 — chain-of-verification，2-3 独立源
7. 输出报告 — 问题清单 + 整改建议 + 质量分
```

输出骨架、质量分计算、高风险场景要求详见 [references/output-template.md](./output-template.md)。

## 检查清单

- 100% 事实声明是否都挂了引用，而不是抽样。
- 四要素是否齐全；缺一项即不完整。
- 是否 WebFetch 打开关键来源确认支持关系。
- 高风险声明是否有 2-3 源交叉。
- 平均源质量是否达到 B；低于 B 的结论是否已降级表述。
- 无法验证的声明是否显式记为"待补证"。
- 是否检查了 URL 可达性。

## 反模式

### FAIL: 只看格式不看内容

```
"2024 全球 AI 市场 $150B [Gartner, 2024]"
→ 格式齐全，通过
→ 实际原文 $120B，被篡改
```

### PASS: 打开原文对照

```
WebFetch Gartner 报告 → 原文 $120B → 标为数据错误
```

### FAIL: 单一来源当事实

```
"远程办公生产力 +47%" [某博客, 2023]
→ 单博客直接写进结论
```

### PASS: Chain-of-verification

```
交叉搜索 → 命中 Stanford Bloom 2015 原始研究 +13%
博客 47% 来源不明 → 改用原始数据
```

更多反模式（模糊泛称、低质未降级等）见 [references/patterns.md](./patterns.md)。
