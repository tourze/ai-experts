# 团队组成分析

## 适用场景
- pre-seed 到 Series A 阶段的团队设计、关键岗位招聘与预算规划。
- 需要把团队规划与融资节奏联动时，可配合 [fundraise-advisor](../../fundraise-advisor/SKILL.md)；讨论不确定性缓冲时，可配合 [planning-under-uncertainty](../../planning-under-uncertainty/SKILL.md)。

## 核心约束
- 先明确当前阶段的核心瓶颈和里程碑，再决定招谁，不按“理想组织图”反推。
- 人数、薪酬、股权和管理带宽要一起算，不能只看 headcount。
- 组织设计必须服务接下来 12-18 个月的关键目标，而不是为远期梦想预埋过多层级。

## 代码模式
```markdown
| 阶段 | 关键目标 | 必招岗位 | 可延后岗位 | 预算/股权约束 |
| --- | --- | --- | --- | --- |
```

## 检查清单
- [ ] 已说明阶段目标、收入/融资里程碑和当前瓶颈。
- [ ] 招聘优先级与业务风险、交付能力和管理带宽一致。
- [ ] 薪酬、股权与现金消耗模型对得上。
- [ ] 已考虑哪些能力可外包、顾问化或延后招聘。

## 反模式

### FAIL: 大厂照搬

```
Pre-seed 8 人团队配置：
- CTO + VP Eng + Eng Manager + Sr Eng × 2
- VP Product + PM + Designer
→ 3 层管理 / 决策慢 / 烧钱快
→ 6 个月烧光 200w
```

### PASS: 阶段适配

```
Pre-seed (5-7 人)：
- 联创 × 2（CEO + CTO 都做活）
- 工程师 × 3（无 manager，直接 ship）
- 设计/PM 由联创兼
Series A 后（15-25 人）：
- 引入第一个 EM（仍 player coach）
- 拆 Designer 和 PM 角色
→ 每招 1 人前问："这个人 30 天内带来什么里程碑"
```

### FAIL: 只看月薪

```
"招 1 个高级，月薪 4w，一年 48w"
→ 算下来好像可以
→ 漏算：股权 0.5% × 估值 1 亿 = 50w 稀释
→ 福利 / 设备 / 招聘费 ≈ 月薪 25%
→ 真实成本 80w+
```

### PASS: 全口径成本

```
| 岗位 | 月薪 | 股权 % | 福利 25% | 招聘 1 月 | 12 月真实成本 |
| Sr Eng | 4w | 0.3% | 1w | 4w | 64w + 稀释 |
→ 算清后再决策招几个 / 是否外包
```
