# 人才选育用留

## 适用场景
- 快速扩张期的人才策略规划。
- 关键人才流失后的根因诊断。
- 与 [team-composition-analysis](../team-composition-analysis/SKILL.md) 配合：团队分析定结构，选育用留定人才策略。

## 核心约束
- 四个环节闭环：选（招聘配置）-> 育（培训发展）-> 用（绩效管理）-> 留（薪酬激励）。
- 任何一个环节断裂都会导致整体失效——只招不培 = 浪费，只培不用 = 人走。
- 关键是"留住想留的，淘汰该淘汰的"，不是"留住所有人"。
- 环节联动必须检查：招来的人是否得到培养？培养的能力是否被用上？做出贡献的人是否得到回报？留不住的人是否在招聘端补上？

## 不适用场景
- 只需确定团队结构和 headcount：用 [team-composition-analysis](../team-composition-analysis/SKILL.md)。
- 单次招聘决策：选育用留是体系设计，单个岗位招聘不需要全框架。

## 代码模式

本 skill 是分析框架，不要求执行代码。输出时按“输入事实 → 框架拆解 → 关键判断 → 下一步动作”的结构组织，可用表格承载维度、证据、判断和建议。

## 检查清单
- [ ] 四个环节都有诊断。
- [ ] 识别出最薄弱的环节。
- [ ] 关键人才有专门策略。
- [ ] 改进建议具体可执行。

## 反模式

### FAIL: 只关注招聘

```
"人不够 -> 加大招聘" -> 入职后没培训 -> 6个月走了
-> 看起来是"选"的问题，实际是"育"和"留"的问题
```

### PASS: 闭环诊断

```
关键人才流失率 25%（行业 15%）
根因分析：
- 选：招聘质量没问题（入职 3 个月满意度 4.2/5）
- 育：晋升通道模糊（无 IC/管理双通道）<- 薄弱点
- 用：绩效评估偏主观（无量化标准）<- 薄弱点
- 留：薪酬 P50 水平（行业中位数，无竞争力）
-> 优先修"育"（Q1 建晋升通道）和"留"（Q2 调薪至 P75）
```
