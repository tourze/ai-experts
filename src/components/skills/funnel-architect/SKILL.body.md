## 核心约束
- 先明确目标用户、核心承诺和单步转化目标，再设计页面和自动化链路。
- 每一级价值阶梯都要回答“为什么现在值得升级”，不能只靠更多功能堆价格。
- 漏斗设计必须考虑流量来源和成交门槛，不要把所有受众塞进同一条路径。

## 代码模式
```markdown
流量入口 -> 诱饵/预约 -> 核心报价 -> 升级项 -> 留存/复购
```

## 检查清单
- [ ] 入口人群、承诺、CTA 和转化目标明确。
- [ ] 价值阶梯、页面顺序与报价逻辑一致。
- [ ] 已考虑邮件/销售跟进、表单摩擦和异议处理。
- [ ] 每一步都能被指标追踪。

## 反模式

### FAIL: 只改文案

```
落地页 CTA 改了 5 版
"立即开始" → "免费试用" → "马上加入"
→ 转化率仍 1.5%
→ 真问题：注册流程 12 步、需要信用卡、邮件激活慢
```

### PASS: 重做承接链路

```
1. 量化每步流失：访问→注册 60% / 注册→激活 40% / 激活→付费 5%
2. 修最大流失点：注册流程 12 步 → 3 步
3. 邮件激活 → 同步即用
4. 付费门槛 → 14 天免费试用
→ 再迭代文案
```

### FAIL: 没主报价直接 upsell

```
首页：基础版 / 专业版 / 企业版 / 加购 A / 加购 B / 加购 C
→ 用户看花了，关掉页面
```

### PASS: 一个主报价

```
首页只推 1 个主套餐（"专业版"）
其他放定价页二级
upsell 在用户成功使用 3 周后触发
→ 决策路径清晰
```
