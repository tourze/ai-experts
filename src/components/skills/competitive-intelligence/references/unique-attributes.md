# Unique Attributes

## Attribute Test

A useful unique attribute is:

- observable by the customer;
- tied to a real alternative;
- hard enough to copy;
- valuable in a specific use case;
- supported by proof.

## Weak Attribute Patterns

| Weak Claim | Problem |
| --- | --- |
| "AI-powered" | Category feature, not unique. |
| "Easy to use" | Needs task-level proof. |
| "Enterprise-grade" | Too broad without compliance, scale, or governance evidence. |
| "All-in-one" | Can signal bloat instead of value. |

## Strong Attribute Pattern

```text
Attribute:
Alternative lacks:
Customer-visible value:
Proof:
Best-fit customer:
```

## Rule

If an attribute cannot be mapped to a customer value and proof point, do not use it as positioning.
