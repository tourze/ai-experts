# Competitive Scoring Rubric

## Default Dimensions

| Dimension | Weight | Evidence |
| --- | --- | --- |
| Core workflow fit | 25% | Task completion, depth, edge cases. |
| UX and adoption friction | 20% | Steps, onboarding, errors, accessibility. |
| Differentiation | 20% | Hard-to-copy attributes and proof. |
| Pricing and packaging | 15% | Public pricing, limits, buyer fit. |
| GTM and credibility | 10% | Case studies, ecosystem, distribution. |
| Execution momentum | 10% | Release cadence, hiring, roadmap signals. |

## Score Scale

| Score | Meaning |
| --- | --- |
| 1 | Weak or unproven. |
| 2 | Present but limited. |
| 3 | Competitive baseline. |
| 4 | Strong, buyer-visible advantage. |
| 5 | Category-leading and hard to copy. |

## Rules

- Every score needs at least one evidence link or dated observation.
- Use `unknown` instead of inventing a score when evidence is absent.
- Weight dimensions before scoring to avoid post-hoc rationalization.
- Compare competitors against the same buyer scenario.
