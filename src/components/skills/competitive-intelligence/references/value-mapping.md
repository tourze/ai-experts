# Value Mapping

## Mapping Chain

```text
Attribute -> Capability -> Customer outcome -> Business metric -> Proof
```

## Examples

| Attribute | Outcome | Metric |
| --- | --- | --- |
| Realtime sync | Fewer stale decisions. | Report cycle time, error rate. |
| Automated anomaly explanation | Faster issue response. | MTTR, analyst hours saved. |
| Native approval workflow | Reduced compliance rework. | Audit exceptions, approval time. |

## Proof Requirements

- Use customer language, not internal architecture terms.
- Prefer before/after measures where possible.
- Tie value to a named buyer or user.
- State assumptions when the metric is inferred.

## Output Format

```markdown
| Attribute | Customer Value | Proof | Message |
| --- | --- | --- | --- |
```
