## 代码模式

### 1. 收敛输入

优先补齐这些信息：

- 主题
- 数量（Top 5 / 10 / 20）
- 最低 stars
- 排序模式：相关性优先 / 星标优先
- 目标形态：可直接使用的产品 / 可二次开发的框架 / 资料清单

### 2. 设计 5-10 组查询

每组查询至少说明目的，例如：

```text
Query-1: "agent memory framework"
目的：高召回核心主题

Query-2: "long-term memory for ai agents"
目的：补同义词盲区
```

### 3. 召回、去重、过滤

- 每组 query 抓 30-50 个候选。
- 按 `owner/repo` 去重。
- 应用默认硬过滤或用户指定门槛。
- 记录检索时间与配额状态，避免结果不可复现。

### 4. 重排与分类

建议按以下维度综合排序：

- 需求相关性
- 场景适配度
- 活跃度（最近更新时间）
- 工程成熟度（README、示例、License、维护迹象）
- stars

同时给仓库打“角色标签”：

- 通用框架层
- 应用产品层
- 基础设施层
- MCP / 工具服务层
- 目录清单层
- 方法论 / 研究层

### 5. 交付表格

```markdown
| 仓库 | 星标 | 类型 | 项目介绍（是什么 + 为什么推荐） | 其它信息补充 | 链接 |
|---|---:|---|---|---|---|
```
