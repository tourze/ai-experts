# 路演 Deck 审查

## 适用场景
- 融资前审稿、路演彩排、单页重写、叙事主线梳理。
- 开始审查前先阅读 [references/frameworks.md](references/frameworks.md)。
- 需要联动融资策略与市场规模时，可配合 [fundraise-advisor](../fundraise-advisor/SKILL.md) 和 [market-sizing-analysis](../market-sizing-analysis/SKILL.md)。

## 核心约束
- 审查重点是“投资人为什么会相信”，而不是单纯排版好不好看。
- 每一页都要回答它在整条故事里的作用，删不掉的页才值得保留。
- 结论必须指出证据缺口、叙事跳步和风险感知，而不是泛泛说“更精炼”。

## 代码模式
```markdown
| 页码 | 这页在证明什么 | 当前问题 | 建议修改 |
| --- | --- | --- | --- |
```

## 检查清单
- [ ] 已逐页判断风险消除力度与价值放大力度。
- [ ] 叙事顺序覆盖问题、机会、牵引力、团队和资金用途。
- [ ] 数字、市场和竞争口径彼此一致。
- [ ] 已指出最该删、最该补、最该重写的页面。

## 反模式

### FAIL: 视觉反馈

```
"P1 字体改 Inter / 配色用渐变 / 加点图标"
→ 创始人改了一周
→ 投资人："故事我没听懂"
```

### PASS: 叙事审查

```
| 页 | 应证明什么 | 当前问题 | 重写建议 |
| P1 (cover) | 我们是谁 + 一句话价值 | 只有 logo | 加 "We help X do Y in Z" |
| P5 (traction) | 有真实需求 | 列了 50 logo | 改为 ARR 增长曲线 + 净留存率 |
| P9 (ask) | 资金用途 | 模糊 | 改为 12 个月里程碑 + 团队扩招 |
```

### FAIL: 大词堆叠

```
"颠覆性 AI 平台 + 万亿市场 + 10x 增长"
→ 投资人："牵引力数据呢？"
→ "目前 beta 阶段"
→ 评估失败
```

### PASS: 证据驱动

```
P3: 问题
- 客户访谈 30 家，27 家提及"X 流程每月浪费 8 小时"
P4: 解决方案
- 演示视频：8 小时 → 20 分钟
P5: 牵引力
- 6 个月 ARR：$5k → $80k（实数，截图）
- 净留存：125%
```
