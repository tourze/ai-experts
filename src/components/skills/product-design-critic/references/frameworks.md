# 路演 Deck 审查框架

用于快速判断一份 deck 是否具备“可被投资人相信”的结构。

## 逐页检查

| 页类型 | 应证明什么 | 常见缺口 |
| --- | --- | --- |
| Problem | 问题真实且代价高 | 空泛叙述，无证据 |
| Solution | 方案能有效解决问题 | 功能描述多，机制不清 |
| Market | 机会规模合理 | TAM/SAM/SOM 口径混乱 |
| Traction | 已有可复现进展 | 只有 logo，没有指标 |
| Team | 团队有执行优势 | 缺关键能力说明 |
| Ask | 资金用途和里程碑明确 | 只写金额，不写计划 |

## 输出格式建议

按 `P0/P1/P2` 标注问题严重度，并给出：
- 影响对象（投资人会担心什么）
- 建议重写方向
- 可补充证据类型
