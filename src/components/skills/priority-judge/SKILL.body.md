## 代码模式

```text
任务：准备路演材料
清晰度：清楚
截止：本周三 18:00
建议：现在做

任务：重构数据层
清晰度：不清楚
截止：无
建议：暂缓

🔥 现在做
1. 路演材料：周三前必须交付，且路径清楚。

⚡ 然后做
1. 招聘 JD：本周要发，但今天不是阻塞项。

⏸ 暂缓
1. 数据层重构：目标和范围未收敛，先不投入。
```

## 反模式

### FAIL: 全部 P0

```
🔥 现在做：
1. 路演材料
2. 招聘 JD
3. 数据层重构
4. 周报
5. 客户回访
6. ...（共 12 项）
→ 等于没排
```

### PASS: 只锁 1-2 个"现在做"

```
🔥 现在做（≤2）
1. 路演材料（周三 deadline，路径清楚）

⚡ 然后做
2. 招聘 JD（本周内，今天非阻塞）

⏸ 暂缓 / 先澄清
- 数据层重构（范围未收敛 → 先 30 分钟讨论目标）
```

### FAIL: 拍脑袋假精确

```
"今天 9:00-9:23 写邮件，9:23-10:47 改 PR..."
→ 切换成本巨大，第一项 delay 全部崩
```

### PASS: 粗粒度 + 留缓冲

```
上午：路演材料（核心 deck）
下午前半段：路演材料（演练 + 修订）
下午后半段：buffer / 紧急事件
→ 留 30% 时间应对意外
```
