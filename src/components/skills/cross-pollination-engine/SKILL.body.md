## 代码模式

```text
问题：
核心任务：

借鉴源 A（行业）：
- 他们怎么解决：
- 背后的原理：
- 迁移到当前场景：

借鉴源 B（行业）：
- 他们怎么解决：
- 背后的原理：
- 迁移到当前场景：

合成方案：
- 组合后的最小方案

第一步试验：
- 本周能验证的版本
```

## 反模式

### FAIL: 同业互抄

```
问题：付费转化低
借鉴：竞品 A 用免费试用 → 我们也加免费试用
借鉴：竞品 B 用阶梯定价 → 我们也加阶梯定价
→ 全行业越来越像，没有差异
```

### PASS: 跨界远场

```
问题核心任务：降低用户首次承诺成本
借鉴源 A（健身房）：first session 免费但要预约 → 制造 "我已经投入" 感
借鉴源 B（航空业）：里程积分 → 让"未来折扣"变成留存锚
合成：注册即给 30 天积分（30 天后失效），首次完成任务额外 +50%
最小实验：本周给 100 个新用户测一周
```

### FAIL: 借表面不借机制

```
"星巴克的杯子很可爱，我们也做杯子周边"
→ 没看到背后的"第三空间"机制
→ 我们是 SaaS，做杯子完全无意义
```

### PASS: 提取原理

```
"星巴克 = 第三空间（家与公司之外的归属）"
→ 我们的"第三空间"对应什么？
→ 用户在两个核心工作流之外，需要一个轻量探索区
→ 落地：加一个"工作坊"模式，无 KPI 压力
```
