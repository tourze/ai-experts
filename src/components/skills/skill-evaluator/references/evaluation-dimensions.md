# 评估维度（120 分制）

## D1：知识增量（20 分）——核心维度

Skill 是否提供了 Claude 不具备的专家知识？

| 分数 | 标准 |
|------|------|
| 0-5 | 解释基础概念、标准库教程、通用最佳实践——Claude 已知 |
| 6-10 | 有专家知识但被浅显内容稀释 |
| 11-15 | 以专家知识为主，少量冗余 |
| 16-20 | 纯知识增量——每段都值其 token 开销 |

**红旗**（直接 ≤ 5 分）：解释基础概念、标准操作教程、常见库用法、行业标准术语定义。

**绿旗**（高增量）：非显而易见选择的决策树、只有踩坑才知道的权衡、真实经验中的边缘案例、附带非显而易见原因的 NEVER 规则、领域特定思维框架。

**评估三问**：Claude 已知吗？在向 Claude 解释还是为 Claude 解释？E/A/R 比例如何？

---

## D2：思维模式 + 流程（15 分）

是否传递了专家**思维模式**以及**必要的领域特定流程**？

| 类型 | 示例 | 价值 |
|------|------|------|
| 思维模式 | "设计前先问：什么让它难忘？" | 高——塑造决策 |
| 领域流程 | "OOXML：解包→编辑 XML→验证→打包" | 高——Claude 不知道 |
| 通用流程 | "步骤 1：打开，步骤 2：编辑，步骤 3：保存" | 低——Claude 已知 |

| 分数 | 标准 |
|------|------|
| 0-3 | 仅有通用流程 |
| 4-7 | 有领域流程但缺思维框架 |
| 8-11 | 思维模式 + 领域工作流平衡 |
| 12-15 | 专家级：塑造思维并提供 Claude 不会的流程 |

---

## D3：反模式质量（15 分）

专家知识的一半是知道不该做什么。

| 分数 | 标准 |
|------|------|
| 0-3 | 无反模式 |
| 4-7 | 泛泛警告（"避免错误"、"注意边界"） |
| 8-11 | 有具体 NEVER 清单及部分理由 |
| 12-15 | 专家级 NEVER + WHY——只有经验才能教会 |

**检验**：专家读了会说"是的，这是我踩坑才学会的"还是"这谁都知道"？

---

## D4：规范符合度——description（15 分）

| 分数 | 标准 |
|------|------|
| 0-5 | frontmatter 缺失或格式无效 |
| 6-10 | description 模糊或不完整 |
| 11-13 | 有 WHAT 但 WHEN 较弱 |
| 14-15 | WHAT + WHEN + 触发关键词齐全 |

**Description 须回答**：做什么（WHAT）、什么场景（WHEN）、什么关键词触发（KEYWORDS）。

---

## D5：渐进式信息披露（15 分）

```
第 1 层：元数据（始终在内存）—— name + description ~100 token
第 2 层：SKILL.md 正文（触发后加载）—— < 500 行
第 3 层：资源（按需加载）—— scripts/、references/、assets/
```

| 分数 | 标准 |
|------|------|
| 0-5 | 全堆在 SKILL.md（>500 行） |
| 6-10 | 有 references 但加载条件不明 |
| 11-13 | 分层良好，有 MANDATORY 触发 |
| 14-15 | 决策树 + 显式触发 + 禁止加载指引 |

---

## D6：自由度校准（15 分）

| 任务类型 | 自由度 | 原因 |
|---------|--------|------|
| 创意/设计 | 高 | 多种有效方案，差异化是价值 |
| 代码审查 | 中 | 有原则但需判断 |
| 文件格式操作 | 低 | 一个字节错误就损坏文件 |

| 分数 | 标准 |
|------|------|
| 0-5 | 严重错配（创意给僵硬脚本，脆弱给模糊指引） |
| 6-10 | 部分合适 |
| 11-13 | 多数场景校准良好 |
| 14-15 | 完美校准 |

**检验**：Agent 犯错后果大→低自由度；后果小→高自由度。

---

## D7：模式识别（10 分）

| 模式 | 行数 | 特征 | 适用 |
|------|------|------|------|
| Mindset | ~50 | 思考>技法，强 NEVER，高自由度 | 创意任务 |
| Navigation | ~30 | 最小 SKILL.md，路由到子文件 | 多独立场景 |
| Philosophy | ~150 | 哲学→表达，强调工艺 | 艺术/创作 |
| Process | ~200 | 分阶段，检查点，中自由度 | 复杂项目 |
| Tool | ~300 | 决策树，代码示例，低自由度 | 精确操作 |

| 分数 | 标准 |
|------|------|
| 0-3 | 无识别模式 |
| 4-6 | 部分遵循但偏差大 |
| 7-8 | 模式清晰 |
| 9-10 | 精湛运用合适模式 |

---

## D8：实操可用性（15 分）

Agent 能否直接据此行动？

| 分数 | 标准 |
|------|------|
| 0-5 | 混乱、矛盾或未测试 |
| 6-10 | 可用但有明显缺口 |
| 11-13 | 常见场景清晰 |
| 14-15 | 全面覆盖，含边缘情况和错误处理 |

**检查**：决策树清晰否？代码示例可运行否？主方案失败有降级否？边缘情况覆盖否？
