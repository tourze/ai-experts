# 评估协议

## Step 1：第一遍——知识增量扫描

通读 SKILL.md，对每节问：> "基础模型已知这个吗？"

逐节标注：
- **[E] Expert**：基础模型通常不知道——价值增
- **[A] Activation**：基础模型知道但提醒有用——可接受
- **[R] Redundant**：基础模型肯定知道——应删除

计算比例 E:A:R：
- 好 Skill：>70% Expert，<20% Activation，<10% Redundant
- 中等：40-70% Expert，高 Activation
- 差：<40% Expert，高 Redundant

## Step 2：结构分析

- [ ] 校验 frontmatter 有效性
- [ ] 统计 SKILL.md 总行数
- [ ] 列出所有 references 文件及大小
- [ ] 识别 Skill 遵循的模式
- [ ] 检查加载触发条件（如有 references）

## Step 3：逐维度评分

对 8 个维度（详见 `references/evaluation-dimensions.md`）：
1. 找具体证据（引用相关行）
2. 赋分并附一行理由
3. 如果得分低于满分，标注具体改进点

## Step 4：计算总分与等级

```
总分 = D1 + D2 + D3 + D4 + D5 + D6 + D7 + D8
满分 = 120
```

**等级**：

| 等级 | 百分比 | 含义 |
|------|--------|------|
| A | 90%+ (108+) | 优秀——生产级专家 Skill |
| B | 80-89% (96-107) | 良好——小幅改进即可 |
| C | 70-79% (84-95) | 合格——有明确改进路径 |
| D | 60-69% (72-83) | 低于平均——存在显著问题 |
| F | <60% (<72) | 差——需根本性重设计 |
