
# 网络侦察与端口扫描

## 适用场景
- 需要建立资产清单、开放端口和服务版本基线。
- 需要为 [wireshark-analysis](../../wireshark-analysis/SKILL.md) 和 [protocol-reverse-engineering](../../protocol-reverse-engineering/SKILL.md) 提供后续切入点。
- 需要做低噪声扫描或对比多轮结果的变化。

## 核心约束
- 先定义目标：找主机、找端口、识别服务，还是验证特定假设。
- 默认从最小必要扫描开始，不把 `-A -p-` 当起手式。
- 把 TCP、UDP、脚本探测和版本识别拆开执行，方便归因。
- 对生产网络控制速率、并发和重试。

## 代码模式
```bash
nmap -Pn -p 80,443,8080 10.0.0.0/24
nmap -Pn -sV -sC -p 22,80,443 10.0.0.10
nmap -Pn -sU -p 53,161 10.0.0.10
```

## 检查清单
- 确认目标网段、禁扫范围、速率限制和联系人。
- 保留原始 XML/grepable 输出，便于复测对比。
- 把开放端口与真实业务、暴露面和可利用性对应起来。
- 对误报端口做二次验证。

## 反模式

### FAIL: 起手 -A -p- -T5

```bash
nmap -A -p- -T5 10.0.0.0/24
# 65535 端口 × 256 IP × 全脚本
# 触发 IDS 全部告警 / 客户被动应对
# 扫到的服务版本可能因速率被丢包误判
```

### PASS: 分阶段

```bash
# 1. 先发现存活
nmap -sn -PE 10.0.0.0/24
# 2. Top-100 端口快速扫
nmap -Pn -F 10.0.0.10
# 3. 仅对发现的端口做版本 + 默认脚本
nmap -Pn -sV -sC -p 22,80,443 10.0.0.10
# 4. 必要时再 -p- 全端口（限定单 IP）
```

### FAIL: 端口开放即漏洞

```
"22/tcp open ssh → SSH 暴露在公网"
→ 报告 P0
→ 实际：客户的跳板机本就需要 22，bastion + MFA 已配
```

### PASS: 验证业务上下文

```
1. 端口开放是事实
2. 是否预期？→ 询问 / 查文档
3. 是否有补偿控制？（MFA / IP 白名单 / 速率限制）
4. 实际可利用性？（弱密码 / 已知 CVE / 暴露服务版本）
5. 综合后才定级
```
