## 不适用场景
- 已有明确 ICP：如果目标客户已清晰，跳过 STP 直接做定位。
- 纯 B2B 大客户（< 50 个潜在客户）：市场太小无需统计学细分，直接做 ABM（account-based marketing）。

## 检查清单
- [ ] S -> T -> P 按顺序完成，没有跳步。
- [ ] 细分维度至少用了 2 个。
- [ ] 目标市场有明确选择标准和理由。
- [ ] 定位主张是一句话，且有差异化依据。

## 反模式

### FAIL: 跳过 S 直接定位

```
"我们的定位是：最好用的项目管理工具"
-> 给谁用的？什么场景？和谁竞争？
-> 没有细分就没有定位
```

### PASS: 逐步收敛

```
S: 按公司规模 x 行业 x 使用场景细分出 8 个市场
T: 评估后选定"50-200 人互联网公司的研发团队"
   理由：规模大、增速快、竞争弱、我们有产品优势
P: "对于中型互联网研发团队，TaskFlow 是唯一
   内置 CI/CD 集成的项目管理工具，
   不同于 Jira 的复杂配置，我们开箱即用"
-> 每一步都有依据
```
