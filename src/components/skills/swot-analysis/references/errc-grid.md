# ERRC 四动作网格

## 适用场景

- 需要把蓝海战略从概念转成产品、服务、成本或体验动作。
- 需要解释新价值曲线如何形成。
- 需要避免只“增加功能”而不降低复杂度。

## 四动作

| 动作 | 问题 |
| --- | --- |
| 剔除 Eliminate | 行业内长期默认但用户不再需要的因素是什么？ |
| 降低 Reduce | 哪些因素被过度服务、成本过高？ |
| 提高 Raise | 哪些因素应该显著高于行业标准？ |
| 创造 Create | 哪些从未提供的新价值能吸引非顾客？ |

## 输出模板

```md
| 动作 | 因子 | 理由 | 证据 | 风险 |
| --- | --- | --- | --- | --- |
| 剔除 | [因子] | [为什么] | [证据] | [风险] |
```

## 反模式

- 只有提高和创造，没有剔除和降低。
- 把“更多功能”当蓝海。
- 没有说明成本结构和用户牺牲项如何变化。
