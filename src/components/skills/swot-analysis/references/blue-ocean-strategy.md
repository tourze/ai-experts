# 蓝海战略

## 适用场景
- 需要重新定义竞争边界，而不是继续在现有红海里卷价格和功能。
- 需要使用 [references/strategy-canvas.md](./strategy-canvas.md)、[references/errc-grid.md](./errc-grid.md)、[references/non-customers.md](./non-customers.md) 做结构化分析。
- 讨论市场进入顺序时，可配合 [crossing-the-chasm](../../designing-growth-loops/SKILL.md)；讨论定位时，可配合 [obviously-awesome](../../competitive-intelligence/SKILL.md)。

## 核心约束
- 先识别现有竞争因子与用户牺牲项，再谈创新；不要把“多做几个功能”误判为蓝海。
- 每个新价值主张都要同时回答“给用户创造了什么”与“删减了什么成本或复杂度”。
- 输出必须区分假设与证据，尤其是对非顾客、渠道与定价的判断。

## 代码模式
```markdown
| 因子 | 行业现状 | 我们提高/降低/剔除/创造 | 证据 |
| --- | --- | --- | --- |
| 价格透明度 | 低 | 提高 | 访谈 / 数据 |
```

## 检查清单
- [ ] 当前竞争因子和目标非顾客已经列清。
- [ ] 已完成策略画布和 ERRC 四动作梳理。
- [ ] 新价值曲线能解释为什么竞争会变得不重要。
- [ ] 与定位、渠道和落地路径没有冲突。

## 反模式

### FAIL: 只换话术

```
原：CRM 软件
改：AI 驱动的智能客户关系平台
→ 产品/成本/价值曲线完全没变 → 仍在红海
```

### PASS: 真改 ERRC

```
剔除：复杂的销售自动化（80% 用户用不到）
降低：模板数量从 200 → 20
提高：开箱即用速度（设置时间 2 天 → 10 分钟）
创造："周一早晨"摘要（前一周关键变化）
→ 价值曲线明显偏离行业
```

### FAIL: 蓝海 = 高端版

```
原：99/月
"蓝海"：999/月 + 同样功能 + 黑色界面
→ 这是垂直分层，不是新价值空间
```

### PASS: 服务非顾客

```
访谈"为什么不买 CRM"的 3 类非顾客：
- 个体户：太复杂
- 服务团队：不需要 pipeline
- 教育机构：术语不匹配
→ 设计针对其中 1 类的极简产品
```
