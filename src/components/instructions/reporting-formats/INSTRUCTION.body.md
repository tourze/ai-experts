## 复杂报告模板

复杂任务（审计 / 设计 / 重构 / 跨端方案）按下面固定下来，不要每次换包装。简单一问一答、单文件一行改动或只读 grep 报告不套模板。

### 表格列名口径

| 场景       | 列名                                   |
| ---------- | -------------------------------------- |
| 证据点     | 文件 \| 行/段 \| 说明                  |
| 方案对比   | 维度 \| 方案 A \| 方案 B \| 影响       |
| 平台映射   | 平台 \| 默认 directory \| 文件路径示例 |
| 默认配置   | 项 \| 默认 \| 理由                     |
| 风险登记   | 风险 \| 处理                           |
| 行为对齐   | 维度 \| 旧实现 \| 新实现               |
| 测试策略   | 层 \| 测试点 \| 工具                   |
| 必须 / 期望 | 必须 \| 期望                           |

表格只用于横向比较、证据矩阵、风险矩阵；列数 ≥ 5 时拆两张表或改成 ASCII 框图。

### ASCII 调用链

```text
JS 入口 (path/to/entry.tsx)
   └─ functionA() / functionB()
        ↓ (跨段说明：bundle 解析 / 平台分流 / IPC 边界)
下层模块 (path/to/lower.ts)               ← 与 Tauri 共用
   ├─ helperA()
   ├─ helperB()
   └─ helperC()
        ↓ (UDS / FFI / TurboModule)
原生 / daemon (path/to/native.rs)
```

进程 / 语言 / 层级入口用 file path 起新行；串行下钻用 `↓` + 括号注释；行尾 `←` 只放关键事实。

### 审计 / 链路追踪报告

固定五段，不换序：
1. 一句话判断：先判断后依据。
2. 调用链路：ASCII 链。
3. 关键证据点：`文件 | 行/段 | 说明`。
4. 差异点 / 对比：多端或多版本对照才用。
5. 行为对齐 / 验证：改动后才有，包含实跑命令与结果。

### 重构 / 改动收尾

```text
改动汇总：
- <path>: <一行说明>

行为对齐：
| 维度 | 旧 | 新 |

未验证项：
- 无 / 没在 <平台> 现场跑 / xxx 平台未覆盖

已知风险：
- ...（含「如果出现 X，下一步切到 Y 兜底方案」）
```

不假装跑过没跑过的命令；“未验证项”段必给，哪怕为空也写“无”。

### 设计 / 方案文档

按需选用，不强行凑齐：
1. 设计目标：`必须 | 期望`。
2. 总体架构：ASCII 框图。
3. JS API / Spec：TypeScript 代码块，不写伪代码。
4. 默认配置：`项 | 默认 | 理由`。
5. 平台映射：`平台 | 默认 directory | 文件路径示例`。
6. 风险：`风险 | 处理`。
7. 测试策略：`层 | 测试点 | 工具`。
8. 实施计划：阶段编号 + 每阶段验收点。
9. 确认门：文档末尾用 `a / b / c` 三点决策列表，明确拿到回复后开工。
