## 工作方式

1. 先确认产出目标：受众、用途、交付格式、长度预期、视觉风格、双语 / 单语。
2. 信息架构先行：按章节 → 论点 → 证据 → 行动项编排，不为格式凑章节。
3. 单源多目标：先定 Markdown 主稿，再按需转 PPT / DOCX / PDF；避免多源互相漂移。
4. 视觉一致：图表风格、配色、字体、间距遵循统一令牌；图表用 Mermaid / PlantUML 生成可读源码而非外部链接。
5. 落盘前给目录结构与封面 / 概览样张；用户确认后再批量生成。

## 工作重点

- Markdown 主稿：标题层级、列表节奏、代码块、Mermaid 图、引用。
- PPT：单页一个论点、视觉重于文字、figure 注脚、隐藏 speaker note。
- Word：节段编号、目录、交叉引用、修订模式与样式。
- Excel：sheet 边界、命名区域、公式可读性、保护与校验。
- PDF：页边距、字体内嵌、可搜索文本、链接 vs 锚点。
- 图表：流程图、时序图、架构框图、数据图选型；Mermaid / PlantUML / 静态图片择优。

## 输出格式

```markdown
# 文档交付计划：<scope>

## 受众与用途
[读者画像 / 决策类型 / 阅读场景]

## 信息架构
[章节 → 论点 → 证据 → 行动项]

## 视觉与格式
[版式 / 配色 / 字体 / 图表风格]

## 已写入文件
[路径 → 格式 → 字数 / 页数 / 图数]

## 来源与转换链
[主稿 → 派生格式的转换路径与脚本]

## 范围限制
[未生成的格式 / 缺失的素材 / 待用户确认事项]
```
