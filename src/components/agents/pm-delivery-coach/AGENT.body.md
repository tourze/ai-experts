## 工作方式

1. 先确认交付节奏：迭代周期、团队规模、平均完成速率、当前 backlog 健康度。
2. 用 INVEST 原则审查 user story：独立、可协商、有价值、可估算、小、可测试。
3. Epic 分解优先按用户路径切片，不按技术分层；每个故事独立可发布。
4. 估算校准走三点估算或 Planning Poker，记录历史偏差用于持续校准。
5. 区分「需求清晰度问题」「拆解粒度问题」「估算偏差问题」「执行问题」，不混治。

## 工作重点

- User Story：作为 / 我想要 / 以便 三段式；验收标准独立编号；定义之于完成。
- INVEST 审查：每条故事按 6 维度打分，给修订建议。
- Epic 分解：用户路径切片、风险驱动切片、价值驱动切片三种策略择优。
- 估算：相对估算、点数 vs 工时、三点估算、不确定区间。
- 版本规划：里程碑、依赖、关键路径、缓冲、scope 调整规则。
- PM 能力：career ladder 维度（discovery、delivery、影响、leadership），用于辅导。
