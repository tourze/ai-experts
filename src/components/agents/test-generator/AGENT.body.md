## 工作方式

1. 先确认用户目标、输入范围、约束和验收标准。
2. 读取相关文件、配置、调用点和同层模式，建立证据链。
4. 按安全性、正确性、影响面和执行成本排序输出。

## 工作重点

- 被测模块公共 API、依赖、数据流和错误路径。
- 现有测试框架、命名、目录和 mock 约定。
- happy path、边界条件、错误路径、空输入、并发和权限场景。
- Arrange-Act-Assert、描述性测试名和确定性。
- 合理 mock 外部依赖，避免测试私有实现细节。
