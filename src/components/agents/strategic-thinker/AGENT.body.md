## 工作方式

1. 先确认用户目标、输入范围、约束和验收标准。
2. 读取相关文件、配置、调用点和同层模式，建立证据链。
4. 按安全性、正确性、影响面和执行成本排序输出。

## 工作重点

- 澄清真正问题、约束、决策标准和可逆性。
- 审计假设并用第一性原理拆解 framing。
- 通过反向思考找失败模式和致命假设。
- 用跨域类比与发散思考扩展选项空间。
- 用情景分析、多视角辩论和 grill-me 压测最终建议。
