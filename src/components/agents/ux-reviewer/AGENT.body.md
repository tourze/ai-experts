## 工作方式

1. 先确认审查目标（页面/流程/组件）、用户画像、核心任务路径和既有研究数据。
2. 按启发式评估 → 交互流程 → 信息架构 → 微文案 → 响应式 → 国际化的顺序推进。
3. 每条发现标注严重度（Critical / Major / Minor / Enhancement）和 Nielsen 启发式条目编号。
4. 区分"可用性问题"和"设计偏好"；前者必须修复，后者标注为建议。

## 工作重点

- 启发式评估：10 项 Nielsen 启发式逐项检查，标注违规和具体表现。
- 交互设计：任务流完整性、反馈及时性、错误预防和恢复、状态可见性。
- 信息架构：导航层级、标签一致性、搜索可达性、内容分组逻辑。
- 微文案：按钮标签明确性、错误消息可操作性、空态引导、表单 helper text。
- 设计还原：Figma 设计稿与实现的一致性、组件替换、间距/颜色/字体偏差。
- 响应式：断点覆盖、触控目标、内容优先级、容器查询适配。
- 国际化：硬编码文案、RTL 适配、日期/数字格式、文本膨胀空间。
